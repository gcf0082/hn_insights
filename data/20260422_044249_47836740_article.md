## 项目概述

ctx 是一个为 Claude Code 和 Codex 设计的本地上下文管理器。它的核心功能是帮助用户管理 AI 助手的对话上下文，提供工作流（workstream）管理、转录绑定和分支功能。

## 核心功能

### 1. 精确转录绑定（Exact Transcript Binding）
每个内部 ctx 会话可以绑定到特定的 Claude 和/或 Codex 对话，确保上下文管理的准确性。

### 2. 无转录漂移（No Transcript Drift）
后续的拉取操作会保持在绑定的对话上，而不是跳转到磁盘上的最新聊天。

### 3. 安全分支（Safe Branching）
可以从另一个工作流的当前状态启动新工作流，而无需共享未来的转录拉取或劫持源对话。

### 4. 索引检索（Indexed Retrieval）
保存的工作流、会话和条目都建有索引，支持快速的 `ctx search` 查找。

### 5. 精选加载（Curated Loads）
- 固定（pin）保存的条目，使其始终加载
- 排除保存的条目，使其保持可搜索但不再传递给模型
- 删除条目

### 6. 本地优先（Local-First）
- 无需 API 密钥
- 无托管服务
- 仅使用本地 SQLite 和文件

## 安装方式

项目提供多种安装方式：

1. **标准本地安装**：克隆仓库并运行 `setup.sh`
2. **全局安装**：运行 `setup.sh --global`
3. **无克隆全局安装**：使用 curl 直接安装
4. **Bootstrap 技能安装**：使用 npx skills
5. **Agent Shell 引导**：使用 bootstrap 脚本

## 每日使用命令

### Claude Code 命令
- `/ctx` - 显示当前工作流
- `/ctx list` - 列出保存的工作流
- `/ctx search <关键词>` - 搜索工作流和条目
- `/ctx start <名称> --pull` - 创建新工作流并拉取现有上下文
- `/ctx resume <名称>` - 继续现有工作流
- `/ctx rename <新名称>` - 重命名工作流
- `/ctx delete <名称>` - 删除工作流
- `/ctx curate <名称>` - 打开保存内存的策展 UI
- `/ctx branch <源> <目标>` - 创建新工作流分支

### Codex 命令
使用 `ctx` 前缀替代 `/ctx`，功能类似。

## 技术架构

- **存储**：SQLite 数据库（`context.db`）
- **配置文件**：`.ctx.env` 或 `ctx.env`
- **前端**：可选的本地浏览器 UI，用于浏览、搜索和复制延续命令

## 安全说明

ctx 是一个上下文层，不是沙箱。项目包含详细的安全模型说明。

## 适用场景

- 需要管理多个 AI 助手对话上下文
- 希望在不同工作流之间切换而不丢失上下文
- 需要对保存的上下文进行精细管理
- 重视数据隐私（完全本地存储，无云端）

## 总结

ctx 是一个针对 Claude Code 和 Codex 用户的实用工具，通过工作流管理、转录绑定和分支功能，帮助用户更有效地管理和切换 AI 助手的上下文。其本地优先的设计理念确保用户数据完全存储在本地，保护隐私。
