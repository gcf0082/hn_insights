# Godogen 项目洞察报告

## 项目概述

**仓库**: https://github.com/htdt/godogen  
**Stars**: 1k | **Forks**: 71 | **主要语言**: Python (97.2%), Shell (2.8%)  
**许可证**: MIT  
**作者**: htdt (Alex Ermolov)

Godogen 是一个 AI 驱动的游戏开发管道，通过 Claude Code skills 从游戏描述自动构建完整的 Godot 4 项目。

## 核心架构

### 两个 Claude Code Skills 协作

| Skill | 职责 |
|-------|------|
| **godogen** | 编排器，运行规划管道，管理流程和进度 |
| **godot-task** | 任务执行器，在分叉上下文中实现每个游戏功能 |

### 流水线阶段

1. **Visual Target** - 生成参考截图，定义项目视觉方向
2. **Decomposition** - 分解游戏为最小开发任务集
3. **Architecture** - 设计技术架构，生成 Godot 项目骨架
4. **Asset Generation** - 通过 Gemini 生成 2D 艺术，Tripo3D 生成 3D 模型
5. **Task Execution** - 执行任务，生成场景和运行时脚本
6. **Visual QA** - 截图验证，检测 z-fighting、缺失纹理、物理错误
7. **Orchestration** - 整体编排，生成最终演示视频

## 技术亮点

### GDScript 专家系统
- 自定义语言参考文档，解决 LLM 对 GDScript 训练数据不足问题
- 850+ Godot 类 API 文档，惰性两层加载机制
- 引擎 quirks 编码为显式规则，避免常见陷阱

### 视觉 QA 闭环
- 捕获游戏实际截图，用 Gemini Flash 分析
- 静态/动态两种模式检测不同问题
- 最多 3 次 VQA 循环修复问题

### 文档协议
- `reference.png` - 视觉基准
- `STRUCTURE.md` - 架构蓝图
- `PLAN.md` - 任务图
- `ASSETS.md` - 资产清单
- `MEMORY.md` - 项目记忆

## 依赖要求

- Godot 4 (PATH 中)
- Claude Code
- `GOOGLE_API_KEY` - Gemini 图像生成和视觉 QA
- `GOOGLE_API_KEY` (隐式)
- `TRIPO3D_API_KEY` - 3D 模型转换（仅 3D 游戏需要）
- Python 3 with pip

## 使用方式

```bash
./publish.sh ~/my-game          # 使用 teleforge.md
./publish.sh ~/my-game local.md # 使用自定义 CLAUDE.md
```

## 与视频扩散方案的对比

| 维度 | Godogen | 视频扩散方案 |
|------|---------|-------------|
| 输出 | 可编辑 Godot 项目、脚本、资产 | 预测帧 |
| 可修改性 | 完全可编辑 | 黑盒 |
| 视觉质量 | 强（Tripo3D/Gemini）| 强 |
| 动画 | 需额外生成 | 内置 |

## Demo 示例

项目包含 4 个演示提示：
1. **CartoRally** - 俯视角赛车游戏，等高线地形美学
2. **Ultra Realistic Nature Scene** - 超写实自然场景，程序化着色器
3. **Amsterdam Cyclist** - 2D 横版自行车游戏，阿姆斯特丹主题
4. **3D Alpine Snowboard Simulator** - 3D 高山滑雪模拟器

## 洞察总结

Godogen 代表了 AI 辅助游戏开发的成熟方案，通过结构化流水线、视觉 QA 闭环和专业领域知识编码，突破了 LLM 在 GDScript 方面的局限性。文档化协议和分叉上下文设计使系统可恢复、可检查、可调试。值得关注的是其预算感知资产生成策略和经济性思维。
