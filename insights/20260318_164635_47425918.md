# rustunnel 项目洞察报告

## 项目概述

**rustunnel** 是一个用 Rust 编写的开源隧道服务，复制了 ngrok 的核心功能。它通过中继服务器将 NAT/防火墙后的本地服务暴露到公共互联网，支持自托管或使用其托管服务。

## 核心功能

### 1. 隧道服务

| 功能 | 描述 |
|------|------|
| HTTP 隧道 | 将本地 HTTP 服务暴露为公共 HTTPS URL |
| TCP 隧道 | 支持任意 TCP 协议 (如数据库、SSH) |
| 自定义子域名 | 支持指定子域名 |
| 自动重连 | 客户端自动重连机制 |

### 2. 安全特性

- **TLS 终止**: 所有流量通过 TLS 加密
- **认证系统**: Token 认证机制
- **审计日志**: 记录认证尝试、隧道注册等操作
- **速率限制**: 支持每个隧道和 IP 的速率限制

### 3. 管理功能

| 功能 | 描述 |
|------|------|
| 实时仪表盘 | Web UI 查看隧道状态 |
| REST API | 完整的 API 接口 |
| Prometheus 指标 | 监控集成 |
| Token 管理 | 创建、删除 API Token |

### 4. AI 代理集成

rustunnel 提供 MCP (Model Context Protocol) 服务器，允许 AI 代理 (如 Claude) 自动管理隧道：

- `create_tunnel` - 创建隧道并返回公共 URL
- `list_tunnels` - 列出活跃隧道
- `close_tunnel` - 强制关闭隧道
- `get_tunnel_history` - 获取历史记录

## 部署选项

### 1. 自托管

- **系统要求**: Ubuntu 22.04+ (systemd)
- **依赖**: TLS 证书、公网 IP/DNS
- **安装方式**: 
  - 直接编译安装
  - Docker 部署
  - systemd 服务

### 2. 托管服务

公共边缘服务器: **edge.rustunnel.com** (欧洲赫尔辛基)

使用托管服务需要通过 GitHub Issue 申请 Token。

### 3. 客户端安装

| 平台 | 安装方式 |
|------|----------|
| macOS/Linux | Homebrew |
| 多平台 | 预编译二进制 |
| 源码编译 | cargo build |

## 竞争优势

### vs ngrok

| 方面 | rustunnel | ngrok |
|------|-----------|-------|
| 语言 | Rust | Go |
| 许可 | MIT 开源 | 专有 (有免费版) |
| 部署 | 自托管 + 托管 | 仅托管 |
| MCP 集成 | 原生支持 | 无 |
| 仪表盘 | 完整功能 | 基础 |

### 独特优势

1. **开源可控**: 可完全自托管，数据不经过第三方
2. **MCP 集成**: 原生支持 AI 代理自动化
3. **现代化架构**: 使用 Rust + tokio + yamux
4. **完整监控**: Prometheus + Grafana 集成
5. **多协议支持**: HTTP/TCP 双协议支持

## 技术亮点

1. **高性能**: Rust + tokio 异步架构
2. **流复用**: yamux 实现多路隧道复用单个连接
3. **安全认证**: Token 机制 + 审计日志
4. **灵活部署**: 支持 Docker、systemd、直接部署
5. **开发体验**: 交互式设置向导、本地开发配置

## 总结

rustunnel 是一个现代化的开源隧道解决方案，使用 Rust 实现了 ngrok 的核心功能。其独特优势在于：
- 完全开源可自托管
- 原生 AI 代理 (MCP) 集成
- 高性能的 Rust 架构
- 完整的企业级功能 (监控、日志、认证)

适合需要隧道功能但希望保持数据控制和开源的项目，特别是与 AI 代理集成的自动化工作流场景。
