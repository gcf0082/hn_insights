# OpenAI GPT-5.4 mini 与 nano 洞察报告

**发布日期**: 2026年3月17日

## 核心发布内容

OpenAI 于2026年3月17日正式发布了两款小型模型：**GPT-5.4 mini** 和 **GPT-5.4 nano**，这是迄今为止最强大的小型模型，专为高频且对延迟敏感的任务设计。

## 模型亮点

### GPT-5.4 mini

- **性能提升**: 在编程、推理、多模态理解及工具调用等核心维度全面超越上一代 GPT-5 mini
- **速度提升**: 运行速度提升超过2倍
- **基准测试表现**: 在 SWE-Bench Pro 和 OSWorld-Verified 等多项基准测试中，性能接近体积更大的 GPT-5.4 模型
- **适用场景**: 代码精准编辑、代码库导航、前端生成、电脑用户界面截图解析、计算机操作任务

### GPT-5.4 nano

- **定位**: 体积最小、价格最亲民的 GPT-5.4 版本
- **特点**: 专为速度和成本为首要考量的任务设计
- **适用场景**: 文本分类、数据提取、内容排序、简单辅助任务的代码子代理(Subagents)

## 定价与可用性

### GPT-5.4 mini
- **API 定价**: 输入 $0.75/百万token，输出 $4.50/百万token
- **上下文窗口**: 400k tokens
- **可用渠道**: API、Codex、ChatGPT
- **Codex 消耗**: 仅消耗 GPT-5.4 额度的 30%

### GPT-5.4 nano
- **可用渠道**: 仅 API 向开发者开放

## 战略意义

这两款模型的推出旨在填补大模型在实时交互场景中因延迟过高而难以落地的空白，覆盖编程助手、AI代理系统及多模态应用等快速增长的商业市场。

在代码助手、系统截图解析以及实时图像推理等对延迟要求极高的场景中，响应迅速且能可靠调用工具的小型模型往往比大型模型更具优势。

## 总结

GPT-5.4 mini 和 nano 的发布代表了 OpenAI 在小型模型领域的重大突破，以更低的延迟和成本实现了接近旗舰模型的性能，为开发者提供了更高效、更经济的选择。
