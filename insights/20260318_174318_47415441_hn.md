# GPT-5.4 Mini and Nano - HN讨论洞察

**来源**: https://news.ycombinator.com/item?id=47415441  
**日期**: 2026-03-18  
**投票**: 238 points, 143 comments

## 核心要点

### 1. 速度与性能
- GPT-5.4 Mini: ~180-190 tokens/s
- GPT-5.4 Nano: ~200 tokens/s
- 相比GPT-5 Mini (55-60 t/s) 提升约3倍
- 实际用户体验：从15秒等待缩短到2-3秒开始流式输出

### 2. 价格对比
| 模型 | 输入价格 | 输出价格 |
|------|---------|---------|
| GPT-5.4 | $2.5/$15 | >200K: $5/$22.5 |
| GPT-5.4 Mini | $0.75/$4.5 | >200K: $5/$22.5 |
| GPT-5.4 Nano | $0.2/$1.25 | - |
| Claude Opus 4.6 | $5/$25 | - |
| Claude Sonnet 4.6 | $3/$15 | - |
| Gemini 3 Flash | $0.5/$3 | - |

### 3. 主要争议与观点

#### Mini模型的重要性
- Mini版本比SOTA模型更能反映实际进步
- SOTA模型之间的差异已难以察觉
- 小模型在特定场景下已足够使用，成本更低

#### GPT模型 vs Claude
- **GPT优势**: 速度快、价格相对便宜
- **Claude优势**: 
  - 更适合agentic任务
  - 更好地理解指令和上下文
  - 代码质量更高
  - 工具使用更准确
- **GPT问题**: 
  - agentic任务中频繁误解指令
  - 响应过于"对话式"，缺乏执行效率
  - 过度自信，即使错误也坚持自己观点

#### Gemini
- 价格优势明显，适合简单任务
- 缺乏透明度(思考过程不可见)
- 适合低成本、简单自动化任务

### 4. 关键洞察

1. **价格趋势**: 小模型价格并未持续下降，GPT-5.4 Mini比GPT-5 Mini贵约3倍
2. **TTFT(Time to First Token)**: 重要但被忽视的指标，各厂商未充分披露
3. **评估方法**: 通用benchmark已难以准确评估模型能力，repo-specific evals更实用
4. **使用场景**:
   - 代码agent: 推荐Claude
   - 高速简单任务: 推荐Gemini Flash
   - 复杂推理: 推荐Opus 4.6或GPT-5.4
