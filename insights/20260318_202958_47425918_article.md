## 项目概述

Rustunnel 是一个用 Rust 编写的开源隧道服务，复制了 ngrok 的核心功能。它允许将 NAT/防火墙后面的本地服务暴露到公共互联网，通过中继服务器（自托管或托管服务）实现。

## 核心架构

### 组件结构

```
┌──────────────────────────────────────────┐
│           rustunnel-server               │
│  HTTP edge (:80) → HTTPS redirect        │
│  HTTPS edge (:443) → yamux stream        │
│  Control-plane WebSocket (:4040)         │
│  Dashboard UI + REST API (:8443)         │
│  Prometheus metrics (:9090)              │
│  TCP tunnel ports (:20000+)              │
└──────────────────────────────────────────┘
                    │
                    ▼
         ┌─────────────────────┐
         │   rustunnel client   │
         │  (开发者本地机器)     │
         └──────────┬──────────┘
                    │
                    ▼
         ┌─────────────────┐
         │  本地服务 (:3000) │
         └─────────────────┘
```

### 技术栈

- **后端**: Rust (tokio 异步运行时)
- **前端仪表盘**: Next.js + React + TypeScript
- **通信协议**: WebSocket + yamux (多路复用)
- **数据库**: SQLite (本地/测试) / PostgreSQL (生产)
- **监控**: Prometheus metrics
- **部署**: Docker, systemd

## 核心功能

### 1. HTTP 隧道
- 将本地 HTTP 服务暴露为公共 HTTPS URL
- 支持自定义子域名
- HTTP 到 HTTPS 自动重定向

### 2. TCP 隧道
- 支持暴露任意 TCP 服务 (如数据库)
- 每个活动 TCP 隧道占用一个端口

### 3. TLS 终止
- 支持 Let's Encrypt 自动证书
- 支持 ACME 协议自动续期
- Cloudflare DNS 验证集成

### 4. 控制平面
- WebSocket 连接管理
- 认证和授权
- 令牌管理 API

### 5. 仪表盘与 API
- 实时 Web UI
- RESTful API (OpenAPI 3.0)
- 捕获和重放 HTTP 请求
- 隧道历史记录

### 6. AI 代理集成 (MCP 服务器)
- 提供 Model Context Protocol 接口
- 可被 AI 代理调用
- 工具: create_tunnel, list_tunnels, close_tunnel 等

### 7. 监控
- Prometheus 指标端点
- Grafana 集成支持
- 审计日志

## 部署选项

### 1. 托管服务
- 公共边缘服务器: `edge.rustunnel.com`
- 需要申请认证令牌

### 2. 自托管 (生产)
- Ubuntu 22.04+ with systemd
- Let's Encrypt 证书
- Wildcard DNS 配置

### 3. Docker 部署
- 多阶段构建
- 支持本地开发 (自签名证书)
- 支持生产部署

## 安全特性

- TLS 加密传输
- 认证令牌管理
- 速率限制 (per-tunnel 和 per-IP)
- 审计日志

## 性能特性

- yamux 流多路复用
- 异步 I/O (tokio)
- 连接数和隧道数限制
- 请求体大小限制

## 配置项

### 服务端关键配置
- `domain`: 隧道 URL 基础域名
- `http_port`, `https_port`: 隧道入口端口
- `control_port`: WebSocket 控制平面端口
- `tls`: 证书路径、ACME 配置
- `auth`: 认证设置
- `limits`: 速率限制、连接限制

### 客户端配置
- `server`: 隧道服务器地址
- `auth_token`: 认证令牌
- `tunnels`: 命名隧道配置

## 开发指南

### 构建要求
- Rust 1.76+
- pkg-config, libssl-dev
- Node.js 18+ (仪表盘重建)

### 测试
- 需要 PostgreSQL 实例
- `make db-start` 启动测试数据库
- `make test` 运行完整测试套件

## 竞争优势 (vs ngrok)

1. **开源**: 完全透明，可自托管
2. **Rust 性能**: 内存安全、高性能
3. **MCP 集成**: 原生 AI 代理支持
4. **灵活部署**: Docker、systemd、托管服务

Rustunnel 是一个成熟的 ngrok 开源替代方案，采用现代 Rust 技术栈构建。它提供了完整的功能集，包括 HTTP/TCP 隧道、TLS 加密、实时仪表盘、AI 代理集成和全面的监控能力。项目结构清晰，文档完善，支持多种部署方式，适合需要自托管隧道服务的开发者和团队。
