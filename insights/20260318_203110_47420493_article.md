# Agent-Based Task Execution - OnPrem.LLM 洞察报告

## 来源信息
- **URL**: https://amaiya.github.io/onprem/examples_agent.html
- **抓取时间**: 2026-03-18

## 核心内容总结

### 1. 产品定位
OnPrem.LLM 的 `AgentExecutor` 是一个基于 PatchPal（编码代理）的自主任务执行框架，支持云端和本地模型（需支持 tool-calling）。

### 2. 9个内置工具
默认提供完整的文件操作和网络工具集：
- `read_file`, `read_lines`, `edit_file`, `write_file` - 文件操作
- `grep`, `find` - 搜索工具
- `run_shell` - 执行 shell 命令
- `web_search`, `web_fetch` - 网络工具

### 3. 支持的模型
**云端**: OpenAI GPT 系列、Anthropic Claude、Google Gemini 等
**本地**: Ollama (llama3.1)、vLLM、llama.cpp

### 4. 关键特性

#### 沙箱执行 (`sandbox=True`)
- 在 Docker/Podman 容器中隔离运行
- 防止对主机系统的意外修改
- 代价：容器启动约 5-10 秒延迟

#### 工作目录限制
- 默认限制在 `working_dir` 范围内
- 防止文件访问越界

#### 自定义工具
- 通过 Python 函数定义任意自定义工具
- 示例：金融分析工具（股票价格、波动性分析）

### 5. 实践案例

| 案例 | 场景 | 使用的工具 |
|------|------|-----------|
| 计算器模块 | 创建 Python 模块+测试 | 文件操作、shell |
| 网络研究 | 生成量子计算报告 | web_search, web_fetch, write_file |
| 股票分析 | 对比 AAPL vs MSFT | 自定义金融工具 |

### 6. 本地模型注意事项
- 默认上下文窗口太小，建议至少 8192 tokens
- 推理模型可能需要 32K-64K tokens
- 本地模型 + 沙箱需配置网络：Linux 用 `network='host'`

## 关键洞察

1. **安全设计优先**: 工作目录限制 + 沙箱隔离 + 可选的 shell 禁用
2. **灵活性**: 工具可按需启用/禁用
3. **本地部署友好**: 支持 Ollama/vLLM/llama.cpp 等开源模型
4. **成本可控**: token 消耗透明显示（如计算器任务仅 $0.0547）
5. **扩展性强**: 自定义工具机制支持任意业务逻辑

## 技术栈依赖
- **核心**: onprem, patchpal
- **可选**: yfinance (金融工具), docker/podman (沙箱)
