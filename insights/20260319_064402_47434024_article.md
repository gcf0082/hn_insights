# 洞察报告：cook — AI编程工作流循环工具

**来源**: https://rjcorwin.github.io/cook/  
**生成时间**: 2026-03-19

---

## 核心概述

Cook 是一个为 Claude Code、Codex 和 OpenCode 设计的**工作流循环框架**，通过组合操作符实现自动化迭代、评审和并行实验。

---

## 主要洞察

### 1. 三层操作符体系

Cook 的设计基于三类原语：
- **Work（工作）**: 单次代理调用，是核心执行单元
- **Loop Operators（循环操作符）**: 为工作添加迭代能力：`xN`、`review`、`ralph`
- **Composition（组合操作符）**: 并行运行多个分支并合并结果：`vN`、`vs`、`pick`

### 2. 循环操作符

| 操作符 | 功能 |
|--------|------|
| `xN` | 顺序重复 N 次，每次可见前一次输出 |
| `review` | 添加评审→门控循环，审查质量后决定 DONE 或 ITERATE |
| `ralph` | 任务列表推进器，外层门控判断任务完成或进入下一项 |

### 3. 组合操作符实现并行实验

- `vN`：N 个相同任务并行运行，取最优结果
- `vs`：两个不同方案并行运行，可自由组合各自的循环操作符

这支持了**版本竞赛**模式（race），是快速探索方案空间的利器。

### 4. 灵活的配置与隔离

- 支持为不同步骤配置不同代理/模型
- 支持 Docker 沙箱隔离执行环境
- 支持自定义评审和门控提示词

### 5. 与 OpenCode 的关系

Cook 支持 OpenCode，但**仅在 Docker 模式下**可用。这意味着：
- OpenCode 用户需要启用 Docker 沙箱
- 这确保了执行环境的隔离和一致性

---

## 关键应用场景

1. **代码迭代优化**: `cook "Add dark mode" x3 review` — 3次迭代加评审
2. **方案对比选型**: `cook "Auth with JWT" vs "Auth with sessions" pick "best security"`
3. **质量门控开发**: 设定明确的质量标准（WCAG、无高危问题等）
4. **批量任务执行**: 使用 ralph 推进任务列表

---

## 技术特点

- **左到右组合**: 操作符从左到右依次包装前面的内容
- **基于 Git Worktree**: 并行分支使用独立的 git worktree 实现隔离
- **可扩展配置**: 通过 `cook init` 生成项目级配置文件

---

## 总结

Cook 将 AI 编程从**单次调用**提升到**可控迭代**和**并行实验**的层面。其模块化的操作符设计允许开发者根据需求灵活组合循环和并行策略，特别适合需要高质量输出的复杂项目。
