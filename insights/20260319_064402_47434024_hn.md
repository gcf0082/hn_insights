## 项目概述

Cook 是一个用于编排 Claude Code、Codex 和 OpenCode 的 CLI 工具，专注于工作流循环。主要功能包括：

- **循环操作器**: `xN`(重复)、`review`(评审)、`ralph`(任务列表)
- **组合操作器**: `vN`(多版本竞速)、`vs`(对比方案)
- **解析器**: `pick`(选最佳)、`merge`(合并)、`compare`(对比)

## 社区洞察

### 1. 核心价值定位

社区普遍认为 Cook 填补了 Claude Code 在**可重复的多步骤工作流**方面的空白。原始 CLI 适合单次交互会话，但缺乏复杂工作流的原生支持。

### 2. 开发者态度

**积极方面**:
- 界面设计简洁优雅，skill markdown 仅 180 行，易于上手
- 多人已构建类似工具，说明存在真实需求
- 支持自定义 agent 和 model 组合(如 Opus 做 review)

**顾虑方面**:
- 部分开发者担心失去对代码的直接控制
- 质疑是否可以用简单 bash 脚本或 Makefile 实现相同功能
- `$200/月` 计划的 token 消耗问题(现已有改善)

### 3. 技术实现讨论

- **子 agent 行为差异**: 主 agent 和子 agent 的思考/努力模式存在差异
- **用户输入处理**: 自动化场景下权限检查和交互的处理方式
- **Docker 沙箱**: OpenCode 仅在 Docker 模式下支持

### 4. 竞争观点

- 有人推荐类似工具: [ossature.dev](https://ossature.dev) (使用 .smd spec markdown)
- Claude Code 本身通过 `skills` 功能可实现部分能力
- Bash 脚本作为替代方案

### 5. 生态观察

这是一个新兴的工具类别，反映了 AI 编程助手从"单次交互"向"自动化工作流"演进的趋势。
