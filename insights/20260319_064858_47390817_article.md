# Chrome DevTools MCP 文章洞察

**来源**: https://developer.chrome.com/blog/chrome-devtools-mcp-debug-your-browser-session  
**发布日期**: 2025年12月11日

## 核心功能

Chrome DevTools MCP Server 新增了直接连接活跃浏览器会话的能力，让编码代理能够：

1. **复用现有浏览器会话** - 代理可直接访问已登录状态的浏览器，无需重新登录
2. **访问活跃调试会话** - 可对接DevTools UI中的调试会话，选择Elements或Network面板中的元素/请求，让编码代理分析

## 工作原理

- 新功能需要 Chrome M144 (Beta) 支持
- 通过 `--autoConnect` 选项，MCP服务器可请求远程调试连接
- 为防止滥用，每次请求都会弹出用户授权对话框
- 调试会话激活时，Chrome顶部显示 "Chrome is being controlled by automated test software" 提示

## 配置步骤

1. 在 Chrome 地址栏访问 `chrome://inspect/#remote-debugging` 启用远程调试
2. 配置 MCP 服务器添加 `--autoConnect --channel=beta` 参数
3. 用户启动Chrome后，运行编码代理命令即可

## 应用场景

- **无缝切换调试** - 人工调试发现问题时，可直接选中文本让编码代理修复
- **Elements面板集成** - 选择页面元素，请求编码代理调查问题
- **Network面板集成** - 选择网络请求，让编码代理分析

## 技术意义

这是一项提升AI辅助调试能力的重要更新，将人工调试与AI辅助修复无缝连接，未来计划逐步暴露更多面板数据给编码代理。
