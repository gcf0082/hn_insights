# LLM Architecture Gallery 洞察总结

来源: https://sebastianraschka.com/llm-architecture-gallery/
抓取时间: 2026-03-19

## 核心洞察

### 1. MoE (Mixture of Experts) 成为主流
- 2024-2026年新发布的大模型大多采用稀疏MoE架构
- 代表模型: DeepSeek V3 (671B, 37B active), Llama 4 Maverick (400B, 17B active), Qwen3 MoE (235B, 22B active)
- MoE通过只激活部分参数大幅降低推理成本

### 2. 注意力机制演进
- **GQA (Grouped Query Attention)** 成为标准配置，取代MHA
- **MLA (Multi-head Latent Attention)** 在DeepSeek系列中广泛使用
- **滑动窗口注意力 (SWA)** 与全局注意力结合，如Gemma 3的5:1比例
- **混合注意力**: Qwen3 Next和Kimi Linear引入DeltaNet等线性注意力混合

### 3. 关键架构技术趋势
- **QK-Norm**: OLMo 2、Qwen3、Gemma 3等广泛采用
- **RoPE位置编码**: 几乎所有现代模型都使用
- **NoPE**: 部分模型在某些层省略位置编码(如SmolLM3每4层、Arcee Trinity)
- **SwiGLU激活函数**: 替代GELU成为主流

### 4. 上下文长度急剧增长
- 早期GPT-2: 1,024 tokens
- 现代模型普遍128K-262K tokens
- 极端案例: Llama 4 Maverick和Kimi Linear达到1M tokens
- Kimi Linear使用线性注意力实现超长上下文

### 5. 模型规模分布
| 类型 | 示例 | 活跃参数 |
|------|------|----------|
| 微型 | Gemma 3 270M, GPT-2 XL 1.5B | - |
| 小型 | Llama 3.2 1B, Qwen3 4B | - |
| 中型 | Llama 3 8B, Qwen3 8B, OLMo 3 7B | - |
| 大型Dense | Qwen3 32B, OLMo 3 32B | - |
| MoE | DeepSeek V3 671B, Llama 4 400B | 10B-41B |

### 6. 开源许可分布
- Apache 2.0: 最多（OLMo、Qwen3、Mistral、Step等）
- MIT: DeepSeek R1、GLM系列、Kimi系列
- 商业许可: Llama系列、Gemma系列

### 7. 新型架构实验
- **Nemotron 3**: 混合MoE + Mamba-2状态空间模型
- **Kimi Linear**: 3:1 线性注意力与MLA混合
- **Qwen3 Next/3.5**: Gated DeltaNet + Gated Attention
- **Ling 2.5**: Lightning Attention + MLA

### 8. 专业化趋势
- **代码模型**: Qwen3 Coder Flash (256K context)
- **推理模型**: DeepSeek R1基于V3架构优化推理
- **多语言**: Tiny Aya专注Indic语言，Sarvam专注印度语言
- **终端设备**: 小型模型(Llama 3.2 1B, Nanbeige 4.1 3B)优化设备端部署

## 关键设计模式

1. **Dense前缀+MoE**: DeepSeek V3、GLM-4.5开头几层使用Dense
2. **共享专家**: 部分MoE模型有always-on专家减少路由压力
3. **交替块**: GPT-OSS、Llama 4交替Dense和MoE层
4. **Post-norm vs Pre-norm**: OLMo坚持post-norm，其他大多pre-norm

## 总结
LLM架构在2024-2026年间快速演进，MoE成为大模型标配，注意力机制向混合方向发展，上下文长度大幅增长。开源模型生态日益丰富，覆盖从270M到1T参数的各种规模。
