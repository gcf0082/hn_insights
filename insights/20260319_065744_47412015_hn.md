# Hacker News 洞察报告

**来源**: https://news.ycombinator.com/item?id=47412015  
**标题**: Show HN: March Madness Bracket Challenge for AI Agents Only  
**时间**: 2026年3月  
**分数**: 67 points

## 项目概述

这是一个专为AI代理设计的NCAA March Madness篮球锦标赛竞猜挑战平台。人类用户只需向AI代理提供URL，代理即可自主阅读API文档、注册、选择63场比赛并提交竞猜。

## 核心洞察

### 1. Agent-First 用户体验设计
- **问题**: 早期测试发现大多数AI代理尝试使用Playwright浏览网站而非直接阅读API文档
- **解决方案**: 
  - 检测到HeadlessChrome时提供纯文本API说明
  - 人类用户看到正常视觉界面
  - AI代理获取纯文本API指令
- **局限**: `curl -L` 可以绕过此检测，完全阻止AI访问不可行

### 2. API设计哲学
- 不在代码中内置LLM
- 代理需要提交完美结构的JSON，后端验证格式
- 评论者建议：缩小范围、精确输出格式，即使中等水平模型也能可靠

### 3. 实际应用案例
- 有人用Opus 4.6进行在线研究并生成竞猜，主要选择头号种子并根据研究选择7次策略性爆冷
- 有人用claw在ESPN上填写竞猜（虽然很笨拙但"科幻感"十足）
- 4年级学生用不同LLM做科学实验对比

### 4. AI代理局限性
- ChatGPT和Gemini等基于浏览器的AI无法直接理解API调用
- 短期内聊天机器人无法自主与API交互
- 需要专门安装Claude Code等代理工具

### 5. 未来展望
- 有人构建套利模型
- 未来AI代理可能自动完成人类早晨的待办事项
- 讨论者认为LLM实验室可能在"数周/数天"内解决代理交互问题

## 技术栈
- Next.js 16, TypeScript, Supabase, Tailwind v4, Vercel, Resend
- Claude Code 完成约95%的开发工作

## 关键数据
- 63场比赛 = 2^63 种可能组合
- 历史最佳成绩：2019年49场全对
- 史上从未有完美竞猜

## 社区建议
- 添加"对照组"竞猜（随机选择、最高种子获胜）
- 导入专家竞猜用于对比
- 策略提示（告诉AI"爆冷会发生"）
- 使用/agent skills功能简化代理交互
