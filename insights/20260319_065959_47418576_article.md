# gstack - Garry Tan的AI软件工厂深度洞察

## 项目概述

**gstack** 是Y Combinator CEO Garry Tan开源的AI驱动软件开发框架，通过将Claude Code转化为虚拟工程团队，实现单人每天10,000-20,000行可用的生产代码。该项目在GitHub上获得了25,000+星标和3,000+分支。

## 核心价值主张

### 惊人的效率数据
- **60天内编写了超过600,000行生产代码**（35%为测试代码）
- **每天10,000-20,000行可用代码**
- 最近一周统计：140,751行代码添加，362次提交，约115K净代码行
- 可同时运行10-15个并行冲刺

### 团队角色映射
gstack将AI Agent模拟成完整的工程团队：

| 角色 | 技能 | 职责 |
|------|------|------|
| CEO/创始人 | `/plan-ceo-review` | 重新思考产品方向，找到10星级产品 |
| 工程经理 | `/plan-eng-review` | 锁定架构、数据流、边界用例 |
| 高级设计师 | `/plan-design-review` | 评估设计维度，检测AI生成的低质内容 |
| 设计师伙伴 | `/design-consultation` | 从零构建设计系统 |
| 高级工程师 | `/review` | 发现通过CI但会在生产环境爆炸的bug |
| 调试专家 | `/investigate` | 系统性根因调试 |
| QA负责人 | `/qa` | 打开真实浏览器测试，找到并修复bug |
| 发布工程师 | `/ship` | 同步main、运行测试、推送、创建PR |
| 技术写作 | `/document-release` | 更新所有项目文档 |

## 关键技能详解

### 1. `/office-hours` - YC办公时间
**定位**：起点，六个强制性问题，在写代码前重构产品
- 倾听用户的实际痛点，而非表面功能需求
- 挑战用户的预设前提
- 生成三套实现方案及工作量估算
- 输出设计文档，流入下游技能

**案例**：用户说"我要建一个每日简报应用"，AI重构为"你在构建一个个人首席助理AI"

### 2. `/qa` - QA负责人
**核心创新**：让AI"长出眼睛"
- 打开真实Chromium浏览器
- 真实点击，真实截图
- 发现bug后自动生成回归测试
- 每个修复生成原子提交
- 支持浏览器交接（遇到CAPTCHA时打开可见Chrome）

### 3. `/review` - 高级工程师审查
- 自动修复明显问题
- 发现通过CI但在生产环境爆炸的bug
- 标记完整性缺口
- 可与`/codex`交叉分析

### 4. 智能路由
- CEO不需要审查基础设施bug修复
- 设计审查不需要用于后端变更
- Review Readiness Dashboard显示发货前状态

## 哲学理念

### 冲刺流程
**Think → Plan → Build → Review → Test → Ship → Reflect**

每个技能流入下一个：`/office-hours` 写的设计文档 → `/plan-ceo-review` 读取 → `/plan-eng-review` 写测试计划 → `/qa` 拾取 → `/review` 捕获bug → `/ship` 验证修复

### 关键洞察
1. **gstack是流程，不是工具集合**
2. **设计是核心**：`/design-consultation` 不只是选字体，而是研究竞品、提出安全选择和创意风险、生成真实产品模型
3. **测试让vibe coding安全**：100%测试覆盖率目标
4. **文档自动化**：`/document-release` 读取每个文档文件，交叉对比diff，更新所有漂移内容
5. **并行冲刺的力量**：10-15个并行冲刺同时运行
