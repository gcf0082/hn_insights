# Claude Code Channels 洞察总结

## 核心功能
- **Channels** 是一个研究预览功能，允许通过 MCP 服务器向运行中的 Claude Code 会话推送消息、警报和 webhook
- 支持双向通信：Claude 可以读取事件并通过同一通道回复
- 事件仅在会话打开时到达，支持后台进程或持久终端运行

## 支持的平台
1. **Telegram** - 通过 BotFather 创建机器人，配置 token 后即可使用
2. **Discord** - 需要启用 Message Content Intent，通过 OAuth2 添加到服务器
3. **Fakechat** - 本地演示工具，无需认证，适合快速测试

## 安全机制
- 每个通道维护发送者白名单，仅白名单中的 ID 可以推送消息
- 配对流程：用户向机器人发送消息 → 获取配对码 → 在 Claude Code 中批准
- `.mcp.json` 中的服务器必须同时在 `--channels` 中命名才能推送消息

## 企业控制
- **Pro/Max 用户**：通道可用，用户通过 `--channels` 选择启用
- **Team/Enterprise**：默认禁用，需管理员在 claude.ai Admin settings 中显式启用
- 设置项：`channelsEnabled`

## 技术要求
- Claude Code v2.1.80 或更高版本
- 需要 claude.ai 登录（控制台和 API key 不支持）
- 插件需要 Bun 运行时
- 研究预览期仅接受 Anthropic 维护的允许列表中的插件

## 使用场景
- 转发 CI 结果
- 聊天消息桥接
- 监控事件通知
- 远程控制本地会话
