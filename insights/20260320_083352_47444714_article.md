# 洞察报告：no-ai-in-nodejs-core

## 概述

这是一个向Node.js技术指导委员会(TSC)提交的请愿书，呼吁禁止在Node.js核心代码中使用LLM（大型语言模型）辅助的代码贡献。

## 核心主张

请愿书要求Node.js TSC对"Is AI-assisted development allowed?"议题投反对票，拒绝接受LLM辅助重写的Node.js内部代码。

## 背景事件

- **争议PR**：2026年1月，一个包含19,000行代码的PR (#61478)被提交，目标是实现Node.js虚拟文件系统
- **作者身份**：提交者mcollina是Node.js的长期贡献者
- **关键争议点**：作者声明使用大量Claude Code tokens创建此PR，但已审查所有更改
- **DCO问题**：虽然OpenJS基金会的法律意见认为LLM辅助更改不违反开发者来源证书(DCO)，但社区认为这只是问题的一部分

## 主要论点

1. **关键基础设施**：Node.js是运行在数百万服务器上的关键基础设施
2. **代码质量**：核心代码多年来由开发者手工精心编写，不应被稀释
3. **可复现性**：生成的代码应该让审查者无需付费订阅LLM工具即可复现
4. **信任基础**：接受LLM更改会破坏公共贡献的信誉基石
