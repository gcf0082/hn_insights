# Mistral Small 4 产品洞察

## 基本信息

- **发布时间**: 2026年3月
- **许可协议**: Apache 2.0（完全开源）
- **定位**: 统一指令、推理和多模态能力的多功能模型

## 核心架构

| 参数 | 规格 |
|------|------|
| 总参数量 | 119B |
| 活跃参数/Token | 6B（嵌入层和输出层共8B） |
| MoE专家数 | 128个，每Token激活4个 |
| 上下文窗口 | 256k |
| 输入模态 | 文本 + 图像（原生多模态） |

## 核心亮点

1. **三大能力统一**: 首次将 Magistral（推理）、Pixtral（多模态）、Devstral（代码代理）能力整合到单一模型中
2. **可配置推理强度**: 通过 `reasoning_effort` 参数动态切换：
   - `"none"`: 快速响应，适合日常任务
   - `"high"`: 深度推理，适合复杂问题
3. **性能大幅提升**（对比 Mistral Small 3）:
   - 端到端延迟降低 40%
   - 吞吐量提升 3倍

## 性能表现

在 LCR、LiveCodeBench、AIME 2025 三个基准测试中：
- 与 GPT-OSS 120B 性能相当或更优
- 输出长度显著更短（效率高 3-4 倍）
- 更短输出意味着更低延迟、更低推理成本、更好用户体验

## 目标用户

| 用户类型 | 适用场景 |
|----------|----------|
| 开发者 | 代码自动化、代码库探索、代码代理工作流 |
| 企业 | 通用聊天助手、文档理解、多模态分析 |
| 研究人员 | 数学推理、复杂推理任务 |

- **API/平台**: Mistral API、AI Studio
- **开源**: Hugging Face
- **推理优化**: vLLM、SGLang、llama.cpp、Transformers
- **硬件要求**: 最低 4x NVIDIA HGX H100 / 2x HGX H200 / 1x DGX B200
- **NVIDIA NIM**: 生产部署开箱即用
- **NVIDIA NeMo**: 支持领域定制微调
