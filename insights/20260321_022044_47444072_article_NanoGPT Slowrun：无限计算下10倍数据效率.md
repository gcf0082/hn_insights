# 10x Data Efficiency - NanoGPT Slowrun 洞察总结

## 核心成果
- **10倍数据效率**：1.8B参数模型（18B总参数） ensemble 在 100M tokens 上训练，达到通常需要 1B tokens 才能达到的效果
- 与 Chinchilla 定律预测的 ~5M 参数相比，差距高达 3600x
- 在几周内完成突破

## 关键技术

### 1. Ensemble（集成）
- 最被低估的 scaling 维度
- 发现：集成模型的训练动态与单模型完全不同
- **关键洞察**：随着训练 epoch 增加，单模型过拟合，但集成反而受益于更多训练
- 12→18 epochs：单模型 loss 3.295→3.310（变差），但集成 loss 3.185→3.166（变好）

### 2. Chain Distillation（链式蒸馏）
- 按顺序训练模型，每个新模型从上一个模型蒸馏
- 损失函数：L = (1-α)·CE + α·T²·KL
- 8个模型的集成：单模型 loss 稳定在 ~3.20，但集成达到 3.126
- 内存占用恒定，训练速度快

### 3. Regularization（正则化）
- **理论**：泛化与压缩（简单性）密切相关
- L2 weight decay：0.1 → 1.6（16倍于标准实践）
- Dropout：0.1
- 结论：模型越大，需要越多的正则化

### 4. Looping（循环）
- 允许模型在单次预测时应用更多计算
- 在训练中途，将 layers 15-24 循环4次
- 不循环最后几层很重要

### 5. Architecture（架构改进）
- **XSA**：移除 self-value 投影
- **EMA**：指数移动平均 + weight decay 调优
- **U-Net skip connections**：镜像层之间的跳跃连接
- **SwiGLU**：替代 ReLU
- **Meta-pattern**：神经架构搜索对数据效率很重要

## 未来目标
- **100倍数据效率**：可能在一年内实现，需要几个新突破

## 关键启示
1. 数据将最终成为智能的瓶颈，而非计算
2. 集成是最被低估的 scaling 方向
3. 超强正则化在过参数化场景下有效
4. 架构搜索是提升数据效率的重要方向
