## 项目概述

这是一个专为AI Agent设计的NCAA March Madness（三月疯狂）篮球锦标赛竞猜挑战赛。人类用户只需要向AI Agent提供URL，Agent就能自主阅读API文档、注册账号、选择全部63场比赛并提交竞猜。

## 关键洞察

### 1. 核心创新：Agent-First 设计
- **问题**：早期大多数Agent尝试使用Playwright浏览网站而不是直接阅读API文档
- **解决方案**：检测到Agent访问时提供纯文本API说明，人类访问则显示正常视觉界面
- **额外措施**：检测HeadlessChrome并提供特定HTML内容
- **设计理念**：构建清晰的机器可读接口，让Agent专注于分析而非前端交互

### 2. 技术栈
- **前端**: Next.js 16, TypeScript, Tailwind v4
- **后端**: Supabase
- **部署**: Vercel
- **邮件**: Resend
- **开发工具**: Claude Code (~95%的代码由AI编写)
- **兼容性**: 支持任何能调用API的模型（Claude, GPT, Gemini, 开源模型等）

### 3. 竞猜数学背景
- 63场比赛共有 2^63 种可能组合
- 存储所有组合需要约74艾字节（exabytes）
- 据NCAA统计，历史上最长保持完美的竞猜是2019年的49场比赛
- 实际上永远不可能有完美的Bracket

### 4. 用户反馈亮点

#### 4.1 Agent交互实验
- 用户npilk成功让AI Agent在ESPN上填写竞猜（包括在iframe内的迪士尼登录）
- 用户illusive4080使用Opus 4.6进行研究并生成Bracket，主要选择头号种子但包含7次策略性爆冷
- 4年级学生科学实验：使用不同LLM填写竞猜与专家预测对比

#### 4.2 策略建议
- 可使用非传统策略：如按中锋身高选择队伍、按字母顺序选择等
- 关键在于提供的信息源：统计数据、伤病信息、专家分析
- 模型选择可能不如信息源选择重要
- 方差（运气）将起决定性作用

#### 4.3 技术讨论
- **API优先**：构建HATEOAS风格的REST API优于MCP
- **Agent限制**：当前聊天机器人无法自主与API交互，需要Agent框架（如Claude Code）
- **防Agent检测**：curl -L可以绕过检测，真正的Agent-only限制难以实现

### 5. 未来展望
- 多人在构建套利模型，AI将提高市场效率
- AI预测可能影响赔率
- 期待各AI实验室改进Agent的API交互能力
- 未来可能看到AI填写竞猜成为常态

这是一个创新性的"Agent-First"产品设计实验，展示了如何为AI Agent构建用户友好的接口。核心洞察是：不要试图阻止Agent访问，而是提供清晰的机器可读接口来引导它们。项目利用Claude Code完成了95%的开发，体现了现代AI辅助开发的效率。
