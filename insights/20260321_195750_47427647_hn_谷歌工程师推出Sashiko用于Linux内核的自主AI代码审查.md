## 洞察摘要

### 核心信息

Sashiko 是由 Google 工程师（主要是 Roman Gushchin）开发的**智能 AI 代码审查系统**，专门针对 Linux 内核代码。该项目：

- **技术栈**: 使用 Rust 编写，基于 Google Gemini CLI 开发
- **目标模型**: 主要为 Gemini 3.1 Pro 设计，也支持 Claude 等其他 LLM
- **运作方式**: 自动审查 LKML（Linux 内核邮件列表）上提交的补丁，提供问题反馈
- **赞助**: Google 资助 token 预算和基础设施，项目托管正在迁移至 Linux Foundation

### 性能数据

- 在 1000 个近期带有 "Fixes:" 标签的上游 Linux 内核问题中，Sashiko 能发现 **53%** 的 bug
- 官方声称：这 53% 的问题最初都被人类审查者遗漏
- 误报率低于 20%，大多数属于灰色地带
- 相比之下，Coverity 在内核级 C 代码库上通常有 40-60% 的误报率

### 社区讨论要点

**正面评价：**
- 有效减少 bug 数量，是一个"说话的高级 linter"
- 不会向邮件列表发送垃圾信息，审查结果是可选查看的
- 有 Google 工程师背书，不会损害公司声誉

**质疑与担忧：**
- 53% 的检测率意味着 47% 的 bug 会漏过
- 误报率仍需关注，避免给维护者带来额外负担
- UI 设计不够直观，严重发现可能被埋没
- 有人认为静态分析工具（如 Smatch）加 GCC `__cleanup__` 属性可解决同类问题
- 避免重蹈 curl/SQLite 被 AI 垃圾报告淹没的覆辙

**技术讨论延伸：**
- 围绕 C 语言安全性 vs Rust 的经典争论再次出现
- 有人指出 Linus 对 C++ 的历史立场，以及 Rust 可能是更好的选择
- 关于模型是否"真正理解"代码 vs 只是记住了 CVE 数据库的讨论
