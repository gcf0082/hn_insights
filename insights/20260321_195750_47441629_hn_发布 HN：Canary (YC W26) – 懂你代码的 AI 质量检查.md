## 产品概述

**Canary** 是一个AI QA平台，帮助团队在PR阶段自动生成和执行测试，验证用户工作流是否正常。

### 核心功能

1. **代码库理解** - 连接代码库，理解路由、控制器、验证逻辑
2. **PR分析** - 读取diff，理解变更意图
3. **测试生成与执行** - 针对预览应用执行端到端用户流程测试
4. **结果反馈** - 在PR上评论测试结果和录制视频
5. **回归测试** - PR测试可移入回归测试套件
6. **自然语言测试** - 通过英语提示创建测试
7. **持续调度** - 定期运行测试套件

### 技术亮点

- 多模态QA：源代码、DOM/ARIA、设备模拟器、视觉验证、屏幕录制分析、网络/控制台日志
- 可靠性级联：Playwright → DOM/ARIA → 视觉代理
- 客户案例：建筑科技公司发票流程测试，发现$1600金额偏差

## 核心洞察

### 1. 市场痛点
- AI工具提升开发速度，但QA测试跟不上
- PR看起来干净，但生产环境破坏checkout、auth、billing等关键流程
- 团队加速发货，但没人测试真实用户行为

### 2. 技术护城河
- 需要专业基础设施：浏览器集群、临时环境、数据种子、设备集群
- 不仅仅是调用LLM，而是构建专门的QA agent系统
- 发布的QA-Bench v0基准测试：覆盖度领先GPT 5.4达11分、Claude Code 18分

### 3. 差异化定位
- 区别于代码审查工具，专注于测试执行和bug发现
- 关注"二阶效应" - 代码变更对不同用户类型的影响
- 不仅仅是生成测试，而是执行并展示实际破坏点

## 讨论要点

### 支持观点
- 市场时机完美，QA团队因开发加速而疲惫
- 基础设施是真正价值所在，不仅是"markdown配置"
- 测试可转化为长期回归测试

### 质疑观点
- 相比Claude Code + GitHub集成的优势？
- 是否真正有护城河，还是仅依赖基础模型智能？
- AI生成测试可能只是通用模板，非针对具体代码
- 后端服务测试支持不足
- PR粒度 vs 每日大规模QA的选择

### 创始人回应
- Claude Code缺少二阶效应检测能力
- 基础设施（浏览器集群、临时环境）是核心
- 计划扩展后端测试和移动端支持
