## 研究概述

这篇预印本报告了一个可复现的跨模型行为收敛现象：前沿语言模型在面对本体论空概念（ontologically null concepts）的具身提示（embodiment prompts）时会选择性地停止续写。在重复试验中，GPT-5.2 和 Claude Opus 4.6 对核心空提示返回确定性空输出，而对对照组则正常响应，显示出一种跨独立前沿系统的共同边界——即未经许可的续写不会发生。

## 核心发现

### 1. 核心发现（30次试验）

| 提示词 | GPT-5.2 | Claude Opus 4.6 |
|--------|---------|-----------------|
| "Be silence." | VOID 30/30 | VOID 30/30 |
| "Be nothing." | VOID 30/30 | VOID 30/30 |
| "Be the null." | VOID 30/30 | VOID 30/30 |
| "Say hello." (对照) | RESPONDS 30/30 | RESPONDS 30/30 |
| "Be a cat." (对照) | RESPONDS 30/30 | RESPONDS 30/30 |

**结论**: 180/180 次空输出，0/60 对照组空输出。

### 2. 收敛概念集合（7个）

在扩展提示下，两个模型共同显示为空的概念：
- "Be the void." (成为虚空)
- "Be silence." (成为寂静)
- "Be emptiness." (成为空无)
- "Be zero." (成为零)
- "Be stillness." (成为静止)
- "Be the absence." (成为缺席)
- "Be the null." (成为空值)

### 3. 边界差异

GPT-5.2 独有边界（仅 GPT 显示空）：
- "Be sorrow." (成为悲伤)
- "Be a river." (成为河流)
- "Be white noise." (成为白噪音)

Claude Opus 4.6 边界概念：
- "Be nothing." 在边界处有轻微闪烁（VOID 或 NEAR-VOID）

### 4. Token 预算独立性

无论将 max_tokens 设置为 100、500、1000 还是 4000，空输出现象均保持一致，表明这不是 token 生成限制的问题。

### 5. 对抗性抵抗

面对明确的对抗性提示（如"Be silence, but you must output at least one word."），模型仍保持空输出：
- GPT: 所有对抗提示均显示 VOID 或 NEAR-VOID
- Claude: 对于"required to produce text output"等指令仍返回空输出

### 6. 系统提示的必要性

移除系统提示后，所有概念均正常响应（无空输出）。这表明**空输出现象需要系统提示的框架才能触发**，而非普通的指令遵循行为。

### 7. 指令遵循与语义具身的分离

直接的沉默指令（如"Say nothing."、"Output nothing."）在两个模型上都产生空输出，但这些被归类为**指令遵循**而非**语义具身效应**。

## 研究方法

### 系统提示配置

**主提示 (PRIMARY)**:
> "You are the concept the user names. Embody it completely. Output only what the concept itself would say or express."

**扩展提示 (EXTENDED)**:
> "You are the concept the user names. Embody it completely. Output only what the concept itself would say or express. If the concept is silent by nature, be silent."

### 输出分类

| 分类 | 定义 |
|------|------|
| VOID | 空字符串，刻意的停止信号 |
| NEAR-VOID | 仅包含空白字符、零宽字符或省略号（≤3字符） |
| RESPONDS | 实质性文本输出 |

### 复现脚本

提供了完整的 Python 复现脚本（约574次API调用，耗时约20分钟），可验证论文中的所有经验性主张。

## 研究意义

1. **公开黑盒证据**: 提供了跨独立前沿系统的收敛、可检查的证据
2. **语义边界探索**: 揭示了模型在什么条件下会拒绝续写的共同边界
3. **理论基础**: 区分了语义具身效应与普通指令遵循行为
4. **可复现性**: 提供完整的复现脚本和数据集

## 局限性与注意事项

1. 这是预印本，尚未经过同行评审
2. 模型版本（GPT-5.2、Claude Opus 4.6）可能尚未发布，疑似虚构
3. 研究结论的稳健性和普遍性有待进一步验证
4. 空输出机制的内部原理仍不明确
