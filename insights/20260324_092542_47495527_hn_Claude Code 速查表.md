## 帖子概述

用户phasE89创建了一个Claude Code CLI快捷方式速查表网站（https://cc.storyfox.cz），该速查表涵盖了以下内容：

- 键盘快捷键
- 斜杠命令（Slash Commands）
- 工作流
- 技能系统（Skills）
- Memory/CLAUDE.md功能
- MCP设置
- CLI标志和配置文件

该速查表支持Mac/Windows自动检测，显示当前版本号，并有一个可关闭的最近更新日志。每天自动检查更新并标记新功能。

## 社区讨论要点

### 1. 速查表内容完整性讨论

- **缺失功能**: 有用户指出速查表缺少 `/keybinds` 命令，该命令可以查看所有键盘绑定
- **环境变量**: 用户提到还有更多环境变量未被列出，推荐参考官方文档 https://code.claude.com/docs/en/env-vars
- **快捷键修正**: 有用户指出CMD+V粘贴图片是错误的，Mac上应该是Ctrl+V

### 2. CLI vs GUI 的用户体验辩论

**支持CLI的观点**:
- Claude Code在终端中大受欢迎，因为许多开发者大部分时间都在终端中工作
- CLI工具是日常使用的高频工具
- Claude Code的帮助系统其实很易上手，功能在几分钟内即可发现

**支持GUI的观点**:
- TUI（终端用户界面）不适合代理操作，太受限且太线性
- 有人认为"需要速查表存在"本身就是一个UX问题
- Codex、Cursor等工具的UI是必要的

### 3. Claude Code 与竞品对比

- 有用户认为Claude Code在CLI方面远超Codex
- 讨论中提到OpenAI最近收购的一些人称Codex是"未来"
- 有人用Claude Code创建了复杂的自进化代理系统

### 4. 关于打印和移动端

- 支持Ctrl+P打印
- 支持移动端使用
- 但有用户质疑每天更新的内容是否适合打印

## 技术细节

### 主要Slash命令

| 命令 | 功能 |
|------|------|
| /help | 显示帮助和可用命令 |
| /clear | 清除对话历史 |
| /compact | 压缩对话以节省上下文 |
| /cost | 显示token使用量和费用 |
| /doctor | 检查安装和诊断问题 |
| /model | 切换模型 |
| /keybinds | 查看键盘绑定 |
| /init | 创建CLAUDE.md文件 |

### 主要键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| Ctrl+C | 取消当前操作 |
| Ctrl+L | 清屏 |
| Ctrl+D | 退出 |
| Escape | 取消生成 |
| Shift+Tab | 切换权限模式 |
| Tab | 自动补全 |

### 主要CLI标志

```
claude              # 启动交互式会话
claude -p "prompt"  # 管道模式，单次响应
claude -c           # 继续上次会话
claude -r "id"      # 恢复指定会话
```
