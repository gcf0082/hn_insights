## 洞察摘要

本文作者分享了他首次使用 AI 工具（Claude Code）创建 Pull Request 的经历，以及随之而来的复杂情感。文章探讨了 AI 编程对开发者身份认同和职业价值观的深刻影响。

## 核心内容

### 第一次 AI 辅助编程的经历

作者使用 Claude Code 为 Chroma（Hugo 的默认语法高亮器）创建了他的第一个 AI 辅助 Pull Request。这个 PR 的目的是为他博客中的 ERB 代码片段添加语法高亮支持。尽管这个 PR 最终被维护者批准并合并，但作者内心却充满了矛盾和不安。

### 冒名顶替综合症的加剧

作者坦承自己“没有学到任何东西”，感觉像是“在墙上扔垃圾（slop）”，甚至觉得自己是个骗子。他引用了 Ori Bernstein 的话：“使用 LLM 写代码就像雇一个任务兔子来帮我拼拼图一样有趣”——但这种“有趣”的感觉已经完全消失了。

### 现实的妥协

尽管内心挣扎，作者也承认：如果没有 AI 工具，他根本没有能力或精力独立完成这个 PR。工作一天后，他的大脑已经疲惫不堪，根本无法花费额外时间去学习代码库并提交 PR。这反映了许多开发者面临的现实困境。

### 行业层面的担忧

在工作中使用 AI 工具交付了具有真正客户影响的修复和改进后，作者仍然感到“内心空洞”。他同意 Xe Iaso 的观点：“每当 Claude 帮我做事时，我对结果毫无感觉。” 更令人担忧的是，现在使用 AI 已成为工作中的正常预期，并被纳入绩效评估，这让他预感到这种“欺骗感”只会与日俱增。

### 身份认同的危机

作者反思自己可能过度地将自我身份与职业紧密联系。他一直努力交付优质工作、持续学习、深入理解底层系统、注重代码工艺——而 AI 工具恰恰对这些价值观毫不在意。最终他意识到：股东们关心的只是交付功能、获得客户和盈利，而非软件是如何被构建的。

## 关键引用

> “使用 LLM 写代码就像雇一个任务兔子来帮我拼拼图一样有趣。”

> “每当 Claude 帮我做事时，我对结果毫无感觉。”

## 结论

这篇文章揭示了 AI 编程工具在提升效率的同时，也给开发者带来了深层的心理和职业认同挑战。当技术的进步与个人价值感发生冲突时，如何找到平衡点，成为每个开发者需要思考的问题。
