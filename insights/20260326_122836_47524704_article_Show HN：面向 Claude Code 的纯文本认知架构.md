## 什么是 Cog？

Cog 是一个为 Claude Code 设计的认知架构，旨在赋予 AI 代理持久记忆、自我反思和远见能力。它是 AI 代理连续意识的第一层，让 Claude 可以跨越会话记住用户的偏好、项目历史和长期目标。

与传统需要服务器或运行时的复杂代理框架不同，Cog 完全通过纯文本 Markdown 文件实现，无需任何依赖。

---

## 核心特性

### 1. 持久记忆系统

Cog 采用三层记忆架构：

- **热记忆**（Hot Memory）：`memory/hot-memory.md` — 始终加载，不超过 50 行，记录当前优先级和活跃情境
- **温记忆**（Warm Memory）：`memory/personal/`、`memory/work/` — 领域特定文件，相关技能激活时加载
- **冰川记忆**（Glacier Memory）：`memory/glacier/` — 归档内容，通过 YAML 头信息检索

### 2. 自我反思能力

系统定期运行反思流程，分析近期工作、提炼模式、检测主题，并决定哪些信息值得保留、哪些应该遗忘。

### 3. 远见与场景模拟

每晚运行「REM 睡眠」管道：整合当日对话、提取模式、修剪过时内容、重写规则以优化次日表现。

### 4. 技能系统

内置 11 个技能 + 从清单自动生成的领域技能，支持路由和域管理。

---

## 为什么选择纯文本架构？

1. **Unix 哲学**：使用熟悉的命令行工具操作
2. **完全透明**：每个决策可见、每条规则可编辑、每项变更在 Git 日志中
3. **零锁定**：停止使用 Cog 后记忆仍可访问
4. **完全可观测**：可以观察 Claude 如何组织知识，以及自我改进是否成功

## 快速开始

```bash
git clone https://github.com/marciopuga/cog
cd cog
```

在 Claude Code 中打开项目，然后输入：

```
/setup
```

Cog 会询问用户的生活和工作情况——公司、副业、想追踪的内容——并据此生成路由表、技能文件、记忆目录结构和域清单。

---

## 适用场景

- **反思性开发**：分析编码习惯和决策模式
- **项目管理**：维护决策、行动项目和项目进度
- **个人知识管理**：追踪洞见、决策和进展
- **团队知识共享**：新开发者加入时，Claude 可以引用整个项目历史

## 结论

Cog 是一个实验性的开源项目，展示了如何通过纯文本文件实现 AI 的持久记忆和自我进化能力。它不需要服务器、不需要运行时，只依赖 Claude Code 本身和文件系统。对于希望在不同会话间保持上下文连续性的开发者来说，这是一个值得尝试的解决方案。
