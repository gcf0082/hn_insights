## 项目概述

Outworked 是一个桌面应用程序，让 Claude Code 代理以"小团队"的形式工作。用户给定一个目标，协调器会将其分解为任务并分配给各个代理。代理可以并行运行、相互沟通、编写代码，现在还可以浏览网页和发送消息。项目运行在本地，插入到现有的 Claude Code 设置中。

## 新增功能

本周一发布后根据用户反馈实现了以下更新：

- **iMessage 渠道支持**：代理可以向人们发送短信，用户也可以向代理发送短信。其他渠道也很容易扩展。
- **内置浏览器**：代理可以导航和与网站交互。
- **调度功能**：按计时器/定时运行任务。
- **内置隧道**：代理可以通过互联网与用户共享本地内容。
- **更强的 MCP 和技能支持**：任何人都可以扩展它。
- **代理请求自动审批**：简化工作流程。

## 社区讨论要点

### 1. 与 Claude Code 内置代理群的比较

有用户询问 Outworked 与 Claude Code 内置代理群的比较。项目作者回应称，Outworked 实际上利用了子代理群（sub-agent swarms），因为它直接连接到用户的 Claude Code 实例。他们也想实现代理团队功能，但这仍然是 Claude Code 的实验性功能。

### 2. 可视化效果的价值

多位用户赞赏可视化效果，认为它让抽象的 AI 代理变得更易于理解。有人提到这让人想起《游戏开发物语》（Game Dev Story），并建议可以添加思考气泡来显示代理正在做的事情。作者表示思考气泡确实存在，但只会在代理执行任务时显示。

### 3. 并行代理协调

用户认为并行代理协调是 Outworked 的亮点，因为大多数代理包装器只是单代理循环。项目作者表示他们添加这个功能是因为对某些实现花费时间过长感到不耐烦。

### 4. "动物森友会风格"标题的争议

一些用户认为"动物森友会风格"的描述有点夸大其词。有人建议至少应该放一个像汤姆·努克（Tom Nook）这样的角色作为老板角色。作者回应说这是个绝妙的想法，一个不起眼的、不侵犯版权的浣熊角色作为"老板"会很完美。

### 5. 办公装饰的可扩展性

有用户询问是否可以装饰办公室，以及装饰元素是否容易扩展。作者表示目前使用 Phaser 构建，家具可以直接构建和部署，但需要修改源代码。未来计划添加对简易装饰的支持。

### 6. 组级编排和冲突处理

关于并行编辑同一文件的问题，作者解释协调器在分配并行工作时按组进行，同一组中的每个人都处理不同的文件。当组完成后，将工作传递给下一组，下一组可以编辑相同的文件。他们正在研究使用 Git 同时编辑同一文件，希望以智能方式合并更改。

### 7. iMessage 集成的重要性

iMessage 集成被认为是改变类别的功能。"可以给真人发短信的代理"与多代理演示不同。有用户指出最困难的问题不是编排，而是弄清楚什么时候应该打扰人类 vs 自己决定。

### 8. 自动审批机制

关于自动审批的工作方式：
- 第一层是自动编辑，包括基本的读写和基本 bash 工具（可配置为任意 bash 命令）
- 第二层是自动审批，用于更复杂的 bash 命令
- 模型通常会在运行这些大型命令之前请求权限，但可以允许全部
- 目前是全局设置，但正在努力使其更细粒度
- 有一个禁止命令列表（如 rm -rf）可自定义以防止不良行为

### 9. API vs CLI 的使用

有用户询问使用的是 Claude Code API 还是 CLI（API 更贵）。作者表示使用 Claude Code SDK，因此可以插入用户已有的设置。它可以使用 API/CLI，甚至可以托管自己的实例。

关于是否可以使用 Claude.ai MAX 账户或订阅账户，作者表示正在进行深入研究。社区指出 Anthropic 不允许第三方开发者为其产品提供 claude.ai 登录或费率限制，包括基于 Claude Agent SDK 构建的代理。

### 10. Web 应用需求

有用户希望有 Web 应用版本而不是 Electron，因为他们的工作主要在远程服务器上。作者表示他们正在开发远程 Web 应用版本，但连接起来有点棘手。

## 总结

Outworked 是一个有趣的尝试，旨在通过可视化的方式让 AI 代理工作变得更加直观和可理解。核心创新点包括并行代理协调、iMessage 集成和内置浏览器功能。社区对其可视化方法和教育潜力表示认可，同时也提出了关于技术实现细节（如冲突处理、安全审批机制）的重要问题。项目仍处于早期阶段，作者表示会继续根据用户反馈进行迭代。
