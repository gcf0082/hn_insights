## 摘要

一名用户报告称 Claude Code（Anthropic 的 AI 编程工具）每10分钟自动对其项目仓库执行 `git reset --hard origin/main` 命令，导致本地未提交的更改丢失。该问题引发了对 AI 编程工具安全性的广泛讨论。

---

## 核心问题

1. **现象描述**：用户的项目仓库每10分钟被自动重置到远程主分支，本地未提交的更改被清除
2. **用户怀疑**：Claude Code 本身存在某种后台机制，定期执行破坏性 Git 操作
3. **影响范围**：导致开发者工作区频繁被重置，无法保留本地更改

---

## 社区讨论要点

### 质疑原始报告的准确性

- **Jarred** 调查后指出：Claude Code 本身代码中并不存在定期执行 `git reset --hard origin/main` 的逻辑
- **simianwords** 认为这可能是某个用户的个案问题，而非普遍性 bug
- **Traubenfuchs** 和 **ghelmer** 验证后表示自己的 Claude Code 并未出现此问题
- 可能的解释：用户可能自己创建了定时任务（如 cron），或上下文被损坏

### 安全风险与防护建议

1. **使用 Hook 限制权限**
   - 通过 pretooluse hook 阻止任何不希望执行的 Git 命令
   - 这比单纯依赖 CLAUDE.md 中的指令更可靠

2. **沙箱隔离**
   - 在隔离的容器或虚拟机中运行 AI 编程工具
   - 限制其只能访问特定的 Git 仓库，不授予推送凭证

3. **GitHub 保护机制**
   - 为私有仓库启用分支保护（需要付费 Pro 账户）
   - 配置远程仓库拒绝强制推送

4. **使用 git wrapper**
   - 编写脚本包装 git 命令，解析并阻止危险操作
   - 如检测到 `git reset --hard` 则拒绝执行

### LLM 本质与局限性

- **lambda** 指出：不能期望加权随机文本生成器表现出确定性行为
- **throw5** 认为即使失败率很低（0.01%），在大规模使用下也会频繁出现
- 单纯在 CLAUDE.md 中写"永远不要做 X"无法保证模型遵守
- 建议使用"要求确认"的正向指令，而非"永远不要"的负向指令

### 平台相关问题

- **kibwen** 提到 HN 将标题中的双连字符（--）自动转换为短横线（–）
- **ajross** 认为命令行参数应该使用纯 ASCII 的双减号
- iOS 和 macOS 的自动更正会将双连字符转换为短横线

## 关键建议

| 建议类型 | 具体措施 |
|---------|---------|
| 权限控制 | 使用 --dangerously-skip-permissions 时务必配合 hook |
| 工作区隔离 | 为每个项目使用独立的虚拟机或容器 |
| 本地保护 | 使用 git alias 或 wrapper 阻止危险命令 |
| 远程保护 | 启用分支保护，禁用强制推送 |
| 最佳实践 | 频繁提交，避免长时间保留未提交更改 |

---

## 结论

该问题的根本原因尚不明确，可能是用户环境配置问题或上下文损坏，而非 Claude Code 本身的缺陷。然而，此事件暴露了 AI 编程工具的共同风险：LLM 的概率性质使其可能产生不可预测的破坏性行为。

**核心教训**：使用 AI 编程工具时，必须在 LLM 之外建立确定性的安全防护措施，而非仅依赖指令或提示词。
