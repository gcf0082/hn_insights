# Pneuma：意图驱动的软件生成平台

**洞察链接：** https://pneuma.computer  
**发布日期：** 2026年3月29日  
**来源：** Pneuma 官方博客

---

## 核心概述

Pneuma 是由 Samix Technology 开发的一款革命性软件平台，其核心理念是"软件从意图中具现化"（software that materializes from intent）。用户无需编写代码，只需描述想要的功能，系统便会实时生成、编译并执行相应的程序。这一模式彻底颠覆了传统的软件开发流程，消除了对应用商店、包管理器的依赖，实现了"言出法随"的编程体验。

---

## 技术架构详解

### 意图分类（Intent Classification）

在生成代码之前，Pneuma 首先需要理解用户的意图。系统采用两层分类架构：

- **本地模式匹配（T0）**：处理常见命令如退出、列出代理、打开帮助等，无需网络请求，确保即时响应
- **AI 分类（Claude）**：复杂意图交由 Claude 模型分类，返回 `create`（创建）、`edit`（编辑）、`delete`（删除）、`close`（关闭）或 `navigate`（导航）等指令

这种设计既保证了简单交互的即时性，又将 AI 推理保留给真正需要语义理解的场景。

### 代码生成（Code Generation）

代码生成是 Pneuma 的核心环节。系统将用户意图连同屏幕尺寸信息发送给 Claude Sonnet，生成完整的 Rust 程序。系统提示词长达 600 多行，详细定义了：

- **系统调用 ABI**：绘图、输入、网络、存储、进程间通信、时间、音频等接口
- **内存和执行约束**：`#![no_std]`、`#![no_main]`、无分配器
- **常见任务模式**：文本渲染、数字格式化、状态管理
- **LLM 常见错误目录**：包含已知的错误模式及修正方案

生成的代码是完全自包含的 Rust 程序，声明 `extern "C"` 导入所需的宿主函数，定义 `pneuma_main()` 入口点，使用 `static mut` 全局变量管理状态。典型代理（如时钟）仅需 40-80 行代码，复杂仪表盘可能需要 200-400 行。生成时间通常在 3-8 秒内完成。

### 编译（Compilation）

生成的源代码通过以下预处理流程后由 `rustc` 编译为 WebAssembly：

1. **常见错误自动修复**：自动修补 LLM 常犯的错误（如 `.sqrt()` 方法调用改为自由函数，整数/浮点数取模不匹配等）
2. **数学函数注入**：`sin`、`cos`、`sqrt`、`abs` 等数学函数作为 WASM 原生导入注入

编译命令示例：
```bash
rustc --target wasm32-unknown-unknown \
      --edition 2021 -O \
      -C debuginfo=1 \
      -C link-args=--export=__stack_pointer \
      --crate-type cdylib \
      agent.rs -o agent.wasm
```

编译通常在 1 秒内完成，输出 50-200KB 的 WASM 模块。

### 执行（Execution）

每个代理在 Wasmtime 沙箱中独立的 OS 线程上运行。沙箱强制执行硬性限制：

- **内存限制**：每个代理 64MB
- **燃料调度**：每帧约 10 亿燃料单位（约 16ms 计算时间@60FPS）
- **基于能力的安全权限**：代理声明所需能力（网络、存储、音频、IPC），用户批准或拒绝

执行模型是协作式但被强制的。代理的 `pneuma_main()` 在无限循环中运行，调用 `pneuma_present()` 标记每一帧，`pneuma_sleep()` 让出时间。

### 渲染（Rendering）

代理不直接渲染到屏幕，而是发出绘图命令（`clear`、`rect`、`text`、`circle`、`line`），这些命令累积在共享缓冲区中。主线程的合成器每帧收集这些缓冲区并使用 `wgpu`（Vulkan/Metal/DX12）合成到 GPU。

## 错误修正机制

Pneuma 设计充分考虑了 LLM 代码生成的局限性：

- **编译失败**：错误信息和源代码发送回 Claude，最多 3 次重试（第 1-2 次使用 Sonnet，第 3 次升级到 Opus）
- **运行时错误**：WASM 代理陷阱（panic、越界访问、整数溢出）被捕获，发送回修正管道
- **用户代理**：2 次崩溃恢复机会
- **系统代理**（网络、文件系统、加密、音频）：自动无限重试

---

## 代理生态系统

Pneuma 包含一组不可变的系统代理（原生 Rust 编写）：

- **合成器**：管理窗口布局和代理区域
- **状态栏**：显示层级、代理数量、系统状态
- **停靠栏**：启动和管理代理的侧边面板
- **提示框**：文本输入覆盖层
- **网络代理**：通过 IPC 处理所有 HTTP 和 TCP/TLS 请求
- **文件系统代理**：由嵌入式数据库支持的受限文件存储
- **加密代理**：SHA-256、HMAC、base64
- **音频代理**：通过 `rodio` 播放音调和音符

用户代理不直接发起网络请求，而是向网络代理发送 IPC 消息，由网络代理代为执行。所有 I/O 都通过能力检查的消息传递进行。

## 核心理念

Pneuma 的设计理念是：**代码生成足够快速和可靠，可以替代预构建软件**。

- 无需天气应用——需要的是天气信息，显示它的程序可以实时编写
- 无需待办事项应用——需要的是列表，列表代码可以按需生成

Pneuma 中每个程序默认是临时的，按需持久化。状态存储在受限的键值存储中，源代码存储在服务器上，编译后的 WASM 存在于内存中。关闭代理即消失，从商店重新打开时重新生成。软件是意图的函数，而非需要管理的产物。

---

**总结：** Pneuma 代表了软件开发范式的一次根本性转变——从"构建和安装"到"描述即实现"。它不是简单的 LLM 封装，而是一个完整的运行时环境，更接近操作系统而非聊天机器人。桌面应用纯 Rust 实现（`wgpu` 渲染、`winit` 窗口管理、`wasmtime` 沙箱执行、`sled` 存储），核心路径无 Electron、无浏览器引擎、无 JavaScript。
