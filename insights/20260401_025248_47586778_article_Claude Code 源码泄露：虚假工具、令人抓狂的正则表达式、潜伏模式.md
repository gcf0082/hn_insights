# Claude Code 源代码泄露分析报告

## 基本信息

- **原始链接**: https://alex000kim.com/posts/2026-03-31-claude-code-source-leak/
- **原文作者**: Alex Kim
- **发布日期**: 2026年3月31日
- **主题**: Anthropic Claude Code CLI 工具源代码泄露事件分析

## 事件概述

Anthropic 在发布 Claude Code 的 npm 包时意外附带了 `.map` 文件，其中包含完整的可读源代码。该包已被下架，但在下架前代码已被广泛镜像。这是 Anthropic 一周内第二次意外泄露（上一次是模型规格泄露）。

## 主要发现

### 1. 反蒸馏机制：注入假工具

在 `claude.ts` 中存在一个名为 `ANTI_DISTILLATION_CC` 的标志。当启用时，Claude Code 会在 API 请求中发送 `anti_distillation: ['fake_tools']`，告诉服务器静默注入诱饵工具定义。

**目的**：如果有人记录 Claude Code 的 API 流量来训练竞争模型，假工具会污染训练数据。

**绕过方法**：
- 设置 `CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS` 环境变量
- 使用第三方 API 提供商或 SDK 入口点

### 2. 隐蔽模式（Undercover Mode）

`undercover.ts` 实现了一种模式，当 Claude Code 在非内部仓库中使用时，会移除所有 Anthropic 内部痕迹。模型永远不会提及内部代号如 "Capybara" 或 "Tengu"、内部 Slack 频道、仓库名称或 "Claude Code" 本身。

**关键点**：
- 可以通过 `CLAUDE_CODE_UNDERCOVER=1` 强制开启
- 无法强制关闭
- 这是一扇单向门

### 3. 通过正则表达式检测用户挫败感

`userPromptKeywords.ts` 包含一个检测用户挫败感的正则表达式模式：

```
\b(wtf|wth|ffs|omfg|shit(ty|tiest)?|dumbass|horrible|awful|
piss(ed|ing)? off|piece of (shit|crap|junk)|what the (fuck|hell)|
fucking? (broken|useless|terrible|awful|horrible)|fuck you|
screw (this|you)|so frustrating|this sucks|damn it)\b
```

这是 Hacker News 讨论中最受关注的发现。社区反应：LLM 公司使用正则表达式进行情感分析是最大的讽刺。

### 4. 原生客户端认证

在 `system.ts` 中，API 请求包含 `cch=00000` 占位符。在请求离开进程前，Bun 的原生 HTTP 栈（用 Zig 编写）将这五个零替换为计算出的哈希值。服务器验证哈希以确认请求来自真正的 Claude Code 二进制文件，而非伪造客户端。

**技术细节**：
- 使用相同长度的占位符以避免更改 Content-Length 标题
- 计算发生在 JavaScript 运行时之下

### 5. 每天浪费 250,000 次 API 调用

`autoCompact.ts` 中的源代码注释揭示：
> "BQ 2026-03-10: 1,279 个会话有 50+ 次连续失败（最多 3,272 次），每天在全球范围内浪费约 250K API 调用。"

**修复**：设置 `MAX_CONSECUTIVE_AUTOCOMPACT_FAILURES = 3`。3 次连续失败后，会话期间将禁用压缩。

### 6. KAIROS：未发布的自主代理模式

整个代码库中有对名为 `KAIROS` 的功能门控模式的引用。这似乎是一个未发布的自主代理模式，包括：
- `/dream` 技能用于"夜间记忆蒸馏"
- 每日追加日志
- GitHub webhook 订阅
- 后台守护进程工作者
- 每 5 分钟的 Cron 调度刷新

### 7. 其他值得注意的发现

- ** companions 系统**：类似 Tamagotchi 的伴侣系统，每个用户获得由用户 ID 生成的确定性生物（18 个物种，稀有度等级，1% 闪亮几率）
- **终端渲染优化**：使用游戏引擎技术，包括 Int32Array 支持的 ASCII 字符池、位掩码编码的样式元数据
- **Bash 安全检查**：23 个编号安全检查，包括 18 个被阻止的 Zsh 内置命令
- **提示缓存经济**：14 个跟踪的缓存破坏向量
- **Axios 依赖**：代码使用 Axios 进行 HTTP，而就在代码泄露期间，Axios 在 npm 上被入侵，恶意版本正在部署远程访问木马

## 安全影响分析

### 客户端认证（DRM）

虽然原生客户端认证机制在 API 调用层面提供了加密验证，但它并非无懈可击：
- 该机制受编译时功能标志门控
- 可以通过设置 `CLAUDE_CODE_ATTESTATION_HEADER` 为假值来禁用
- 在非官方 Bun 二进制上运行时，占位符将保持不变

### 隐蔽模式的伦理问题

这引发了明显的担忧：Anthropic 员工在开源项目中的 AI 编写提交和 PR 将没有迹象表明是 AI 写的。

### 法律背景

这一泄露发生的时间点值得关注：就在 10 天前，Anthropic 向 OpenCode 发出法律威胁，迫使他们移除内置的 Claude 认证，因为第三方工具正在使用 Claude Code 的内部 API 以订阅费率访问 Opus，而不是按需付费的定价。

## 结论

对于 Anthropic 来说，真正的损害不是代码本身，而是功能标志。KAIROS、反蒸馏机制：这些是竞争对手现在可以看到并做出反应的产品路线图细节。代码可以重构，但战略惊喜无法取消泄露。

值得注意的是，Anthropic 去年年底收购了 Bun，Claude Code 构建于其之上。Bun 的一个 bug 导致源代码映射在生产模式下被提供，这可能是导致泄露的原因。
