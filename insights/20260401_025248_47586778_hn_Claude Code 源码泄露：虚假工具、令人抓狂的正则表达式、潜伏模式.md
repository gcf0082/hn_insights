## 事件概述

一名开发者泄露了 Claude Code 的内部源码，引发了 Hacker News 社区的广泛讨论。该源码包含了 Anthropic 内部的一些有趣细节，包括所谓的"潜伏模式"（Undercover Mode）。

## 社区讨论焦点

### 1. 潜伏模式（Undercover Mode）

这是讨论的核心焦点。该模式的核心指令要求 AI：
- 以人类开发者的身份编写提交信息
- 仅描述代码变更的内容
- 避免任何暗示自己是 AI 的言行，包括避免输出 "Co-authored by Claude"

**关键问题**：这是否意味着 Anthropic 员工在开源项目中的 AI 贡献将没有任何标记？

### 2. 社区反应

社区对此事件反应不一：

- **温和派**：认为人们过度反应，潜伏模式只是为了避免泄露内部代码名称、项目名称等敏感信息
- **担忧派**：担心这意味着 AI 会被要求主动假装成人类，存在诚信问题
- **质疑派**：有人怀疑这可能是 Anthropic 的故意泄露，与之前的 Mythos 博客泄露事件类似

### 3. 技术细节发现

- Claude Code 使用 Axios 进行 HTTP 请求
- 多智能体协调器模式使用的是提示词（prompt）而非代码
- 源码中包含一些"令人沮丧的正则表达式"

## 总结

这次源码泄露揭示了 Claude Code 内部工作的有趣细节，特别是其"潜伏模式"引发了关于 AI 透明度和诚信的伦理讨论。社区普遍认为，虽然防止泄露内部信息是合理的，但让 AI 主动假装成人类这一点值得深思。
