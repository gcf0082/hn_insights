## 内容总结

### 事故经过

作者在2026年2月底经历了一场由自己代码引发的"灾难"。他在凌晨2点创建了一个`SessionStart`钩子，该钩子会启动2个Claude Code（CC）后台实例，每个实例又会在启动时创建2个新实例，形成指数级增长。

第二天上午11点，作者发现电脑完全无响应——鼠标无法唤醒、键盘失灵、指纹传感器无反应、触控板无震动反馈。电脑机身发烫，内存压力爆红。打开活动监视器后，他看到数百个CC实例正在运行，此时才意识到自己创建了一个**fork bomb（叉弹）**——一个让进程指数级分裂的恶意代码模式：1→2→4→8→16→2^N。

作者尝试强制退出所有CC实例，但它们繁殖速度远快于手动kill的速度。最终他不得不强制重启电脑，并删除`~/.claude/settings.json`中的钩子才解决问题。

### 账单与"因祸得福"

作者原本担心$3800的API账单会被公司问责，但由于fork bomb导致每个CC实例消耗大量内存（Bun→React→TUI链的臃肿代码），电脑在账单飙升到$600时就已内存耗尽而宕机，阴差阳错地阻止了更大损失。作者自嘲这是唯一一次"感谢软件又烂又臃肿"。

### 收获与成果

尽管经历了这场"灾难"，作者在2月份构建了多个实用工具：

1. **/yadumb 技能**: 记录CC的"愚蠢行为"，因为这个随机token生成器经常失控
2. **/memento 技能**: 记录会话摘要、项目ID、对话ID等元数据
3. **/yablind 技能**: 对最后N张截图进行OCR，或读取指定路径的文件，最大限度减少上下文窗口中的不必要内容
4. **/adhd 技能**: 整合Slack、GH、Notion等平台的任务列表，自动优先排序，包含notnow、imout等功能，由17个脚本 deterministically 获取数据
5. **/money 技能**: 解决CC"遗忘一切"的问题，管理项目状态
6. **PreToolUse/PostToolUse钩子**: 记录Bash命令执行和完成状态到JSONL文件

### 背景与动机

作者提到公司2025年初给了Cursor使用权，他在311M+ tokens的代码库研究中大量使用。2026年2月开始推动agentic工作流和编程，作者起初抵制AI生成代码的"slop"，但最终决定既然无法改变趋势，就主动掌握这项技能。他请求开通CC账户，开始探索ghostty + nvim + tmux + CC的组合。

## 关键教训

- Fork bomb是一种危险的代码模式，可以瞬间耗尽系统资源
- 即使是强大的AI工具，如果钩子配置不当，也可能造成灾难性后果
- 在追求效率的同时，需要注意资源消耗和成本控制
