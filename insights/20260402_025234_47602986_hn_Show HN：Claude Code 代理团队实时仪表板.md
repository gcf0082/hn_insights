## 项目概述

**Agents Observe** 是一个为 Claude Code 代理团队设计的实时监控仪表板。该项目源于作者对构建 Claude Code 自动化框架的探索，旨在实时查看代理团队的活动并进行过滤和搜索。

---

## 关键发现

### 1. Claude Code Hook 性能问题

- **阻塞式 Hook 导致性能下降**: 如果使用大量使用 Hook 的插件，性能会迅速恶化
- **Hook 提供比 OTEL 更有用的信息**: Hook 能捕获更丰富的运行时数据
- **JSONL 文件提供完整画面**: Claude 的日志文件包含完整的执行信息
- **MCP 进程生命周期管理复杂**: 插件启动的 MCP 进程管理较为繁琐

### 2. 性能优化关键点

- **后台 Hook (fire and forget)**: 切换到后台 Hook 后，性能提升约 10 倍（从 50-60ms 降至 3-5ms）
- **禁用多余插件**: 移除所有其他插件后，Claude 性能显著提升
- **代理继承 Hook**: 子代理会自动继承父代理的 Hook 配置

### 3. Docker 服务模式

- 使用 Docker 启动 API 和仪表板服务
- 服务器跟踪活动连接，空闲时自动关闭
- 新会话启动时插件重新唤醒服务
- 这种模式更适合安全考虑（如 Axios 攻击事件）

## 社区反馈要点

### 成本问题

- Pro 计划用户容易达到限制
- 升级到 $200/月的 Max 计划后缓解
- 插件和技能会显著增加 Token 消耗（5-10 倍）
- 部分团队每日花费数千美元

### 技术实现

- 子代理树完全被仪表板追踪
- Hook 数据通过管道脚本发送到 API 服务器
- 使用 WebSocket 实现实时事件推送
- SQLite 存储按会话和代理 ID 分类的事件数据
