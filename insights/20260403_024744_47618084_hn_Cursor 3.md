## 内容摘要

Cursor 3 是 Anyscale 推出的 AI 编程工具重大更新，其界面从传统的 IDE 风格转向了类似于 Claude Code 和 Codex 的以聊天为中心的 UX。此举引发了社区对 AI 编程工具发展方向的广泛讨论。

## 核心讨论主题

### 1. 产品定位转变

许多用户对 Cursor 从"开发者主导的 IDE"转变为"agent 驱动的平台"表示担忧。开发者担心这种变化会使他们逐渐失去对代码的控制权，沦为工具的配角。

### 2. 定价策略变化

社区观察到整个 AI 编程领域的定价模式正在转变：
- Cursor 从 IDE 逐步向 agent 系统演进
- Windsurf 取消积分系统，改用每日配额
- Copilot 从无限使用改为计费模式
- Claude Code 推出 Auto Mode，让 agent 自主决定安全操作

### 3. 开发者 vs Agent 的角色定位

有评论指出：AI 实验室认为他们在构建替代软件工程师的自主系统，而软件工程师将这些系统视为辅助工具。即使是最强大的模型，也需要资深工程师的引导才能发挥最大效用。

### 4. 身份困惑问题

有用户表达了对产品定位模糊的困惑："我不知道这个工具实际上是什么了。"Cursor 现在同时是 IDE、agent 界面、CLI 工具和云平台，支持数十种不同模型，这种多功能性反而让部分用户感到迷失。

### 5. 技术问题

有用户报告 Cursor 3 在 Linux Wayland 上存在崩溃问题。

## 官方回应

Cursor 工程师 leerob 在讨论中确认：
- 公司仍然相信"开发者驱动、agent 辅助"的理念
- 新的 Cursor 3 界面是独立窗口，旧的 Cursor 2 风格仍然保留
- 旧界面也在持续改进中

## 社区情绪

总体而言，社区对 Cursor 3 的态度较为复杂：
- **担忧者**：担心失去对代码的控制，产品定位偏离初心
- **期待者**：愿意尝试新功能，希望旧界面保留
- **批评者**：认为 Cursor 正在变成另一个"vibe coding"应用，失去了专业优势
