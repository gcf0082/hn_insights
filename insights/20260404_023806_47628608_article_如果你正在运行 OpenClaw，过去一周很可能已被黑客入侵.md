# OpenClaw 安全漏洞洞察报告

## 基本信息

- **洞察链接**: https://old.reddit.com/r/sysadmin/comments/1sbdw29/if_youre_running_openclaw_you_probably_got_hacked/
- **发布日期**: 2026年3月
- **漏洞名称**: ClawJacked
- **影响产品**: OpenClaw (开源AI Agent框架)
- **漏洞类型**: 高危 - 远程代码执行/数据窃取
- **CVE编号**: 待确认

## 漏洞概述

安全研究人员发现OpenClaw存在高危安全漏洞"ClawJacked"，允许恶意网站通过浏览器静默劫持本地运行的AI Agent实例，从而窃取敏感数据甚至完全控制用户主机。

## 技术细节

### 漏洞成因

1. **本地信任机制**: OpenClaw的WebSocket网关默认绑定到localhost，并假设本地流量是可信的
2. **速率限制豁免**: 网关的速率限制器豁免了localhost连接，允许暴力破解密码
3. **自动设备配对**: 本地设备配对无需用户确认，自动批准

### 攻击链

1. 用户在浏览器中访问恶意网站
2. 网站嵌入的JavaScript代码向localhost上的OpenClaw网关发起WebSocket连接
3. 由于浏览器跨域策略不阻止到localhost的WebSocket连接，攻击成功建立连接
4. 攻击者以每秒数百次的速率暴力破解网关管理密码（本地连接豁免速率限制）
5. 密码破解后，攻击脚本自动注册为可信设备（无需用户确认）
6. 攻击者获得管理员权限，可完全控制AI Agent

### 影响范围

- 可读取配置文件和日志
- 可枚举已连接的设备
- 可执行任意shell命令
- 可搜索消息历史中的敏感信息
- 可从连接设备窃取文件
- 最终导致工作站完全被控制

## 修复措施

OpenClaw已在2026年2月26日发布的版本2026.2.26中修复此漏洞：

- 加强WebSocket安全检查
- 增加额外保护机制防止localhost连接被滥用

**建议所有用户立即更新到版本2026.2.26或更高版本。**

## 安全建议

1. **立即更新**: 将OpenClaw更新到最新版本
2. **AI工具管理**: 企业应识别开发者机器上运行的AI工具，许多工具可能在没有IT监督的情况下部署
3. **权限审计**: 审计AI Agent持有的权限和凭证，限制访问权限仅限必要范围
4. **AI治理**: 对AI Agent作为非人类身份进行治理，需要严格的策略控制、监控访问和完整审计跟踪
