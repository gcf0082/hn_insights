## 洞察摘要

Anthropic 研究员 Nicholas Carlini 在 [un]prompted AI 安全会议上展示了 Claude Code 如何发现一个在 Linux 内核中隐藏了 23 年之久的漏洞。这是一个位于 Linux 内核 NFSv4 服务器中的远程堆缓冲区溢出漏洞，可被利用进行远程代码执行。

### 主要发现

1. **历史性漏洞**：Claude Code 发现了一个自 2002 年以来就存在于 Linux 内核 NFSv4 服务器中的远程可利用漏洞，该漏洞已隐藏长达 23 年。

2. **AI 安全研究能力**：Nicholas Carlini（拥有超过 67,000 次 Google Scholar 引用）承认 Claude 是比他更好的安全研究员。

3. **实际影响**：这不是理论演示，而是在真实环境中发现的可实际利用的零日漏洞。

### 相关安全事件

在同一时期，Claude Code 还发生了其他安全事件：

- **源码泄露**：2026 年 3 月 31 日，Anthropic 意外将 Claude Code 的调试 JavaScript sourcemap 发布到 npm，研究人员迅速重建了约 512,000 行 TypeScript 代码。

- **权限绕过漏洞**（CVE-2026-25723）：Adversa AI 发现 Claude Code 的权限系统存在严重漏洞，攻击者可通过恶意 CLAUDE.md 文件利用超过 50 个子命令的机制绕过 deny 规则，窃取 SSH 私钥、AWS 凭证、GitHub 令牌等敏感信息。

## 意义与启示

这一发现标志着 AI 在安全研究领域的重大突破。Claude Code 能够在 90 分钟内自主发现零日漏洞，包括：
- Ghost CMS 中的盲注零日漏洞（CVE-2026-26980）
- 提取管理员 API 密钥

这表明 AI 助手不仅能够辅助开发工作，还能够成为强大的安全研究工具，帮助发现长期隐藏的系统级漏洞。
