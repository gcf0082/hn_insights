## 项目简介

这是一个名为 "caveman" 的 Claude Code 技能（skill），通过让 Claude 用简短的"穴居人"风格说话来减少 token 使用量。据作者声称，该技能可以**减少约 75% 的 token 使用**，同时保持完整的技术准确性。

## 核心功能

- 去除冗余的前言和铺垫
- 减少填充词和非必要的内容
- 保持代码和技术内容不变
- 使输出更加简洁直接

## 技术细节

- 这是一个 Claude Code 自定义技能
- 针对的是可见的完成输出，而非隐藏的思考 tokens
- Anthropic 官方文档建议更多的思考预算可以提高性能
- 技能加载会消耗上下文，即使只有技能元数据被预加载

## 社区讨论焦点

### 1. Token 与思考的关系
部分开发者认为 **tokens 就是思考单元**，减少 token 输出可能会限制模型的推理能力。Chain of Thought (CoT) 研究表明，中间 tokens 对模型性能有重要影响。

### 2. 实证证据的缺乏
批评者指出，作者的 "75% token 减少" 声称缺乏严格的基准测试，只是一个初步测试结果。

### 3. 空洞词的有效性
讨论中对于"礼貌用语"和"填充词"是否携带有用信息存在分歧：
- 一方认为 "Sure, let me help you" 这类话不携带有用信号
- 另一方引用研究证明礼貌用语确实会影响模型输出

### 4. 语言与思维
有评论提到使用更专业的语言可能会获得更好的结果，因为训练数据中专家语言与正确答案的关联更强。

## 作者的最新回应

作者 (JBrussee-2) 在讨论中澄清：
- 这更多是一个玩笑，而非严肃的研究声明
- 目标是减少可见输出中的前缀、填充词和非必要内容
- 并不试图减少隐藏的思考 tokens
- 承认 "~75%" 的数字来自初步测试，需要更严格的评估
- 真正的评估应该是端到端的：输入 tokens、输出 tokens、延迟和质量

## 结论

这个"穴居人"技能代表了一种有趣的尝试，即通过约束语言风格来减少 LLM 的 token 输出。然而，社区普遍认为需要更严格的基准测试来验证其有效性。更重要的是，关于 tokens 是否等同于"思考"，以及减少输出长度是否会削弱模型能力的讨论，反映了人们对 LLM 内部机制理解的深入。
