## 项目概述

Gemma Gem 是一个运行在浏览器中的 Chrome 扩展程序，完全在设备端通过 WebGPU 运行 Google 的 Gemma 4 模型。整个过程无需 API 密钥、无需云端、数据不会离开用户的机器。该扩展可以读取页面内容、点击按钮、填写表单、执行 JavaScript，并回答关于用户访问的任何网站的问题。

## 核心特性

1. **完全本地运行**: 利用 WebGPU 在浏览器中进行 AI 推理，无需云服务
2. **隐私保护**: 所有数据保留在本地，不上传至任何服务器
3. **浏览器操作能力**: 能够读取页面、点击元素、输入文本、滚动页面、执行 JavaScript
4. **模型选择**: 支持 Gemma 4 E2B（约 500MB）和 E4B（约 1.5GB）两种模型规格

## 技术架构

- **Offscreen Document**: 通过 @huggingface/transformers + WebGPU 运行模型
- **Service Worker**: 消息路由，处理截图和 JavaScript 执行
- **Content Script**: 注入聊天界面和 DOM 操作工具

## 可用工具

| 工具 | 功能 |
|-----|------|
| read_page_content | 读取页面文本/HTML 或指定 CSS 选择器内容 |
| take_screenshot | 捕获可见页面为 PNG 图片 |
| click_element | 通过 CSS 选择器点击元素 |
| type_text | 通过 CSS 选择器向输入框输入文本 |
| scroll_page | 按像素滚动页面 |
| run_javascript | 在页面上下文中执行 JS 代码 |

## 技术栈

- **框架**: WXT (基于 Vite 的 Chrome 扩展框架)
- **ML 推理**: @huggingface/transformers
- **Markdown 渲染**: marked
- **AI 模型**: Gemma 4 E2B / E4B (q4f16 量化，128K 上下文)

## 系统要求

- Chrome 浏览器（支持 WebGPU）
- 约 500MB 磁盘空间（E2B 模型）或约 1.5GB（E4B 模型），首次运行后缓存

## 总结

Gemma Gem 代表了浏览器端 AI 助手的一种创新实现方式，通过 WebGPU 技术实现了完全本地化的 AI 推理能力。其架构设计将模型推理、消息路由和 DOM 操作分离，使得系统更加模块化和可维护。对于注重隐私的用户来说，这是一个值得关注的 Chrome 扩展选择。
