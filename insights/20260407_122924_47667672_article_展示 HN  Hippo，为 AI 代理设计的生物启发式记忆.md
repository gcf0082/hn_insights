## 项目概述

Hippo 是一个受生物学启发的 AI 智能体记忆系统，模拟人类海马体的七种记忆机制。它实现了记忆衰减、检索强化、整合巩固等核心功能，零运行时依赖，支持 Node.js 22.5+。

## 核心问题

AI 智能体在会话之间会遗忘所有内容。现有解决方案只是保存所有内容然后稍后搜索，这相当于文件柜而非大脑。此外，用户记忆被锁定在特定工具中——ChatGPT 知道的 Claude 不知道，Cursor 规则无法转移到 Codex。

## 适用人群

- **多工具开发者**：周一用 Claude Code，周二用 Cursor，周三用 Codex，跨工具上下文无法传递
- **团队中智能体重复犯错**：智能体上周踩了同样的部署坑，本周又踩，Hippo 的错误记忆和衰减机制能让教训更深刻
- **CLAUDE.md 混乱者**：指令文件膨胀到 400 行混合规则和过期方案，Hippo 提供结构化存储
- **需要便携 AI 记忆者**：无供应商锁定，Markdown 文件存储，可跨工具导入导出

## 核心特性

### 记忆衰减机制
每条记忆都有半衰期，默认 7 天。通过 `hippo sleep` 命令定期清理衰减的记忆。

### 检索强化
每次召回记忆都会延长半衰期 2 天，模拟记忆再巩固过程。

### 错误记忆强化
标记为错误的记忆自动获得 2 倍半衰期。

### 置信度分级
- **verified**：验证过的 facts
- **observed**：观察到的模式
- **inferred**：推测
- **stale**：30 天未检索自动标记为过期

### 混合搜索
结合 BM25 关键词和余弦相似度嵌入，安装 `@xenova/transformers` 后可启用语义搜索。

### 跨工具导入
支持从 ChatGPT、Claude (CLAUDE.md)、Cursor (.cursorrules) 和任意 Markdown 文件导入记忆。

### 工作记忆层
独立于长期记忆的短期便签，最大 20 条，按重要性驱逐。

### 会话交接
创建会话摘要供下次会话无缝继续。

### 冲突检测
自动检测记忆中的矛盾并保持可见。

### 神经科学基础
基于七个海马体机制：
1. 双速存储（缓冲 + 语义）
2. 记忆衰减
3. 检索强化
4. 整合巩固
5. 模式加速
6. 冲突检测
7. 多智能体传递

## 快速开始

```bash
npm install -g hippo-memory
hippo init
hippo remember "FRED cache silently dropped the tips_10y series" --tag error
hippo recall "data pipeline issues" --budget 2000
```

## 框架集成

支持 Claude Code、Codex、Cursor、OpenClaw 的自动安装，通过 `hippo init` 自动检测并配置。
