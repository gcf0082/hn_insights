# tui-use 洞察报告

## 项目概述

tui-use 是一款让 AI Agent 能够操作终端交互式程序的工具。正如 BrowserUse 为浏览器提供自动化操作能力一样，tui-use 为终端带来相同的能力。AI Agent 可以运行 shell 命令、读取文件和调用 API，但一旦程序需要用户输入就会陷入困境——因为大多数 CLI 工具是为人类而非 Agent 构建的。tui-use 填补了这一空白，通过 PTY 模拟、屏幕文本捕获和按键注入，让 Agent 能够控制任何终端交互式程序。

## 核心特性

### 1. 完整 VT 渲染支持
PTY 输出由无头 xterm 模拟器处理，ANSI 转义序列、光标移动和屏幕清除都能正确工作。screen 字段始终是干净的纯文本。

### 2. 快照模型
与终端程序交互本质上是一个循环：读取屏幕内容、决定输入什么、重复。tui-use 将这个循环显式化——无需异步流、无需猜测时间、无需重组部分输出。

### 3. 高亮检测
每个快照都包含 highlights 字段，列出屏幕上的反色区域——这是 TUI 程序指示选中项的标准方式。Agent 可以读取当前激活的菜单选项、标签或按钮，无需解析文本或猜测光标位置。

### 4. 丰富的按键支持
支持发送文本、Enter、Ctrl+C、方向键、F 键等。运行 `tui-use keys` 可查看完整列表。

## 典型应用场景

### REPL 会话
运行 Python、Node、psql 或 redis-cli，检查输出并继续操作。再也不需要一次性脚本，当需要交互式会话时就能派上用场。

### 交互式脚手架工具
逐步执行 `npm create`、`cargo new`、`create-react-app` 以及其他在执行前需要提问的 CLI 向导。

### 数据库 CLI
连接 psql 或 mysql，运行查询，检查架构，无需单独的 API 或 ORM 层。

### SSH + 远程交互程序
SSH 到服务器并继续操作另一端的交互式程序，而不仅仅是运行一次性命令。

### TUI 应用程序
操作 vim、lazygit、htop、fzf 和其他从未设计用于脚本化的全屏程序。

## 技术架构

tui-use 在后台运行一个守护进程来管理 PTY 会话：

```
┌─────────────┐     HTTP      ┌─────────────┐     PTY      ┌─────────────┐
│  tui-use   │ ◄───────────► │   Daemon    │ ◄─────────►  │   Program   │
│   (CLI)    │               │(background) │              │  (vim/htop) │
└─────────────┘               └─────────────┘              └─────────────┘
                                     │
                                     ▼
                              ┌─────────────┐
                              │  @xterm/   │
                              │  headless  │
                              │(xterm emu) │
                              └─────────────┘
```

**渲染流程**：

1. 目标程序输出 ANSI 转义序列（颜色、光标移动、屏幕清除）
2. `@xterm/headless` 将它们渲染成完整的终端屏幕状态
3. `snapshot` 返回干净的纯文本 screen 内容，以及元数据如 highlights（反色区域）、title（窗口标题）和 is_fullscreen（备用缓冲区检测）

## CLI 接口

### 核心命令
```bash
tui-use start <cmd>                           # 启动程序
tui-use start --cwd <dir> <cmd>                # 在指定目录启动
tui-use start --label <name> <cmd>             # 带标签启动
tui-use start --cols <n> --rows <n> <cmd>      # 自定义终端大小
tui-use use <session_id>                      # 切换会话
tui-use type <text>                            # 输入文本
tui-use paste "<text>\n<text>\n"                # 粘贴多行
tui-use press <key>                            # 按键
tui-use snapshot                              # 获取当前屏幕
tui-use snapshot --format json                 # JSON 输出
tui-use scrollup <n>                          # 向上滚动
tui-use scrolldown <n>                        # 向下滚动
tui-use find <pattern>                         # 搜索屏幕内容
tui-use wait                                  # 等待屏幕变化
tui-use wait --text <pattern>                 # 等待屏幕包含内容
tui-use list                                  # 列出所有会话
tui-use kill                                  # 杀死当前会话
```

### 守护进程命令
```bash
tui-use daemon status                         # 检查守护进程状态
tui-use daemon stop                         # 停止守护进程
tui-use daemon restart                      # 重启守护进程
```

## 安装方式

### 通过 npm（推荐）
```bash
npm install -g tui-use
```

cd tui-use
npm install

npm link
```

```

```bash

```

## 局限性

- **TUI 颜色/样式信息大部分丢失** — screen 只包含纯文本，颜色和大多数格式被剥离。选中项和活动元素通过反色检测捕获在 highlights 中。窗口标题和全屏模式捕获在 title 和 is_fullscreen 中。
- **Windows 不受支持** — 需要 Unix PTY（macOS/Linux）。通过 ConPTY 的 Windows 支持正在计划中。

## 适用 Agent

非常适合 **Claude Code**、**Cursor**、**Codex**、**Gemini CLI**、**OpenCode** 和其他 AI 编码 Agent。

## 总结

tui-use 填补了 AI Agent 与终端交互的关键空白，让 Agent 能够像人类一样操作 REPL、数据库 CLI、交互式脚手架工具和 TUI 应用程序。通过快照模型和高亮检测，Agent 可以准确理解屏幕状态并做出响应，实现真正可控的终端自动化操作。
