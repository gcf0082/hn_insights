## 一、产品定位

Claude Managed Agents 是 Anthropic 提供的预构建、可配置的代理框架，运行于托管基础设施之上。该产品专为长时间运行的任务和异步工作场景而设计，与 Messages API 形成互补：

- **Messages API**：适合需要自定义代理循环和细粒度控制的场景
- **Managed Agents**：适合需要长时间执行、云基础设施和最小化运维的场景

---

## 二、核心概念

Claude Managed Agents 围绕四个核心概念构建：

| 概念 | 描述 |
|------|------|
| **Agent（代理）** | 包含模型、系统提示词、工具、MCP 服务器和技能 |
| **Environment（环境）** | 配置好的容器模板（包含软件包、网络访问等） |
| **Session（会话）** | 在环境中运行的代理实例，执行特定任务并生成输出 |
| **Events（事件）** | 应用程序与代理之间交换的消息（用户输入、工具结果、状态更新） |

---

## 三、工作流程

1. **创建代理**：定义模型、系统提示词、工具、MCP 服务器和技能，代理创建后可通过 ID 跨会话引用
2. **创建环境**：配置云容器，包含预装的软件包（Python、Node.js、Go 等）、网络访问规则和挂载文件
3. **启动会话**：启动引用代理和环境配置的会话
4. **发送事件并流式响应**：发送用户消息作为事件，Claude 可自主执行工具并通过服务器发送事件（SSE）流式返回结果
5. **引导或中断**：在执行过程中发送额外用户事件引导代理方向，或中断代理以改变任务

---

## 四、适用场景

Claude Managed Agents 最适合以下工作负载：

- **长时间执行**：需要数分钟或数小时运行的任务，包含多次工具调用
- **云基础设施**：安全的容器环境，预装软件包和网络访问
- **最小化基础设施**：无需构建自己的代理循环、沙箱或工具执行层
- **有状态的会话**：持久化文件系统和跨多次交互的对话历史

---

## 五、支持的工具

Claude Managed Agents 提供全面的内置工具：

- **Bash**：在容器中运行 shell 命令
- **文件操作**：读取、写入、编辑、glob 和 grep 容器中的文件
- **Web 搜索和获取**：搜索网页并从 URL 获取内容
- **MCP 服务器**：连接外部工具提供者

## 六、访问与限制

### Beta 访问

- Managed Agents 当前处于 Beta 阶段
- 所有端点需要 `managed-agents-2026-04-01` Beta 头
- SDK 会自动设置 Beta 头

### 速率限制

| 操作 | 限制 |
|------|------|
| 创建端点（代理、会话、环境等） | 每分钟 60 次请求 |
| 读取端点（检索、列表、流式等） | 每分钟 600 次请求 |

## 七、品牌使用指南

对于集成 Claude Managed Agents 的合作伙伴，Claude 品牌使用是可选的。

**允许的使用方式：**
- "Claude Agent"（下拉菜单首选）
- "如果已在"Agents"标签下，可使用 "Claude"
- "由 Claude 驱动的 {您的代理名称}"

**不允许的使用方式：**
- "Claude Code" 或 "Claude Code Agent"
- "Claude Cowork" 或 "Claude Cowork Agent"
- Claude Code 品牌 ASCII 艺术或视觉元素

产品应保持自身的品牌，不应呈现为 Claude Code、Claude Cowork 或任何其他 Anthropic 产品。

---

## 八、总结

Claude Managed Agents 为开发者提供了一个强大的托管代理框架，特别适合需要长时间运行、复杂工具调用和云基础设施支持的场景。其核心优势在于无需自行构建代理循环、沙箱和工具执行层，即可让 Claude 作为自主代理运行。该产品目前处于 Beta 阶段，已对所有 API 账户默认开放访问。
