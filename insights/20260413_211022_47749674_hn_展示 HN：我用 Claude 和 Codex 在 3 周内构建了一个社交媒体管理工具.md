## 项目概述

作者在3周内使用 Claude 和 Codex AI 编码工具构建了一个完整的社交媒体管理平台。该项目名为 **Brightbean Studio**，是一个生产级应用，包含以下特性：

- **12个平台集成**：Facebook、LinkedIn、Instagram、YouTube、TikTok 等
- **技术栈**：Django 5.x + HTMX + Alpine.js + Tailwind CSS 4 + PostgreSQL
- **核心功能**：多租户认证、加密凭证存储、后台任务处理、审批工作流、统一收件箱

---

## AI编码经验总结

### 成功的领域

- Django 模型、视图、序列化器、标准 CRUD 操作
- Facebook 和 LinkedIn 等文档完善的 API 提供商模块
- Tailwind 布局和 HTMX 交互
- 测试生成
- 跨文件重构（Claude 在跨模型、视图和模板的级联更改方面表现出色）

### 遇到困难的领域

- **TikTok API**：文档质量差，存在不寻常的两步上传流程，AI 工具反复生成错误的代码
- **多租户权限逻辑**：代码在单工作区运行正常，但在多工作区场景下存在数据泄露漏洞，且通过了测试
- **OAuth 边缘情况**：令牌刷新、权限撤销、平台特定错误码都需要手动处理
- **后台任务编排**：重试逻辑、速率限制退避、错误处理需要手动编写

### 关键洞察

> "80%的功能在20%的时间内完成，剩余80%用于完善和实际可用性体验"

## 社区讨论焦点

### 1. 关于"氛围编码"(Vibe Coding)的担忧

部分用户对"3周内编码"的项目表示担忧：
- 缺乏严格测试和长期维护验证
- 可能出现未维护的风险
- 但作者指出商业软件也大多采用类似方式构建，至少这个项目是诚实透明的

### 2. AI对软件开发的影响

讨论认为软件开发已进入"交付便宜"的时代：
- 任何想法都可以在一两个月内由一两个人实现 MVP
- 真正的瓶颈不再是编写代码，而是规划、设计、管理协调、寻找客户、与产品集成等"人的工作"

### 3. 技术栈选择

- 作者选择 Python/Django 是因为有数据科学背景
- HTMX 被认为是轻量级、非侵入性的选择，有助于降低 AI 代理的复杂度
- HTMX 发布于 2020 年，并非"过时"技术

### 4. 关于 X (Twitter) 的缺失

作者解释：
- X 的 API 最近改为按请求付费，价格昂贵
- 计划在未来几周内研究集成
- 评论区普遍认为 X 的参与度和推荐流量持续下降

## 开发者工作流程

```
Layer 0 - 项目脚手架
    ↓
Layer 1 — 核心功能
    ├─ Stream A: 内容管道
    ├─ Stream B: 社交平台提供商
    ├─ Stream C: 媒体库
    ├─ Stream D: 通知系统
    └─ Stream E: 设置UI
    ↓
Layer 2 — 协作与互动
    ├─ Stream F: 审批与客户端门户
    ├─ Stream G: 收件箱
    ├─ Stream H: 日历与编辑器增强
    └─ Stream I: 客户端入职
```

开发者最多同时运行 4 个代理并行工作，但由于人脑处理能力限制，这是最大并行度。
