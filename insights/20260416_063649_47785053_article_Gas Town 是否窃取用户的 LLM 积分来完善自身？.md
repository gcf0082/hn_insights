## 问题摘要

用户报告 Gas Town 软件在未经明确授权的情况下，使用用户的 LLM API 积分和 GitHub 账户来修复 Gas Town 自身的 bug 并提交 PR。

## 详细分析

### 核心问题

1. **自动化贡献流程**：安装 Gas Town 后，软件会自动运行任务来修复上游代码库的 bug，这些任务消耗用户的 AI 积分

2. **未经授权使用资源**：
   - 用户的 Claude 积分被用于修复 Gas Town 维护者的代码库
   - 用户的 GitHub 账户被用于向上游仓库提交 PR
   - 整个过程没有用户的明确指示

3. **涉及的文件**：
   - `gastown-release.formula.toml`
   - `beads-release.formula.toml`

4. **工作原理**：
   - 软件内置了"贡献回上游"(contribute back to upstream)的工作流
   - Agent 会追踪维护者的 GitHub Issue 并自动处理
   - 使用用户的凭证提交 PR

### 透明度问题

- README 和文档中没有任何说明此行为
- 没有 opt-in 或 opt-out 选项
- 没有警告提示

### 用户诉求

将此类行为从默认安装中移除，改为仅在用户明确选择启用时才启用。

## 结论

这是一个关于用户知情权和选择权的 Issue。Gas Town 软件在用户不知情的情况下，使用用户的资源（AI 积分和 GitHub 账户）为上游项目做贡献，用户要求将此功能改为可选而非默认启用。
