# Cloudflare AI平台：面向Agent的推理层

**洞察链接**: https://blog.cloudflare.com/ai-platform/
**发布时间**: 2026-04-16
**作者**: Ming Lu, Michelle Chen
**标签**: AI, Agents, AI Gateway, Workers AI, Developers

## 核心摘要

Cloudflare宣布将其AI Gateway打造为统一的推理层，支持通过单一API调用来自14+提供商的AI模型。新功能包括Workers AI绑定集成和扩展的多模态模型目录。

## 关键要点

### 1. 统一目录与端点

- 开发者可以使用相同的`AI.run()`绑定调用第三方模型
- 从Cloudflare托管模型切换到OpenAI、Anthropic等提供商仅需一行代码修改
- 支持70+模型来自12+提供商，包括**Alibaba Cloud、AssemblyAI、Bytedance、Google、InWorld、MiniMax、OpenAI、Pixverse、Recraft、Runway和Vidu**
- 新增图像、视频和语音等**多模态模型**支持

### 2. 统一计费管理

- 通过单一API访问所有模型，实现**集中管理和监控AI支出**
- 支持自定义元数据，可按属性分类成本（如免费用户vs付费用户、具体工作流等）
- 大多数公司目前平均使用3.5个模型跨多个提供商

### 3. 自定义模型支持

- 引入Replicate的Cog技术，帮助用户容器化机器学习模型
- 只需编写`cog.yaml`文件和Python推理代码即可部署自定义模型
- 支持微调和针对特定用例优化的模型

### 4. 低延迟优化

- Cloudflare网络覆盖全球330个城市的数据中心
- 调用Cloudflare托管模型时无额外公网跳转，提供**最低延迟**
- 专注于优化首token时间（Time to First Token），对实时Agent至关重要

### 5. 高可靠性与自动故障转移

- 当模型在多个提供商可用时，一个提供商宕机可**自动路由**到其他提供商
- Streaming响应缓冲机制支持Agent中断后重新连接恢复响应
- 结合Agents SDK的内置检查点功能，确保用户体验无缝

### 6. Replicate团队整合

- Replicate团队正式加入Cloudflare AI平台团队
- 即将可以通过AI Gateway访问Replicate上的所有模型
- 可以在Workers AI上托管在Replicate上部署的模型

## 技术细节

### Workers AI绑定示例

```javascript
const response = await env.AI.run('anthropic/claude-opus-4-6', {
  input: 'What is Cloudflare?',
}, {
  gateway: { id: "default" },
});
```

### 自定义元数据示例

```javascript
const response = await env.AI.run('@cf/moonshotai/kimi-k2.5', {
  prompt: 'What is AI Gateway?'
}, {
  metadata: { "teamId": "AI", "userId": 12345 }
});
```

### Cog配置示例

```yaml
build:
  python_version: "3.13"
  python_requirements: requirements.txt
predict: "predict.py:Predictor"
```

## 行业影响

Cloudflare的AI平台代表了AI基础设施的关键演进：
- **降低多提供商管理的复杂性**：单一API访问所有模型
- **成本可见性**：集中监控AI支出
- **可靠性增强**：自动故障转移减少服务中断
- **Agent友好**：低延迟和高可靠性对Agent工作流至关重要

随着AI模型的快速迭代，灵活的推理层将成为构建AI应用的基础设施。
