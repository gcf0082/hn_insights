## 洞察摘要

本文介绍了作者将 Claude Code 应用于硬件开发的实验经验，特别是将 Claude 与示波器和 SPICE 模拟器结合使用的工作流程。

## 核心内容

### 背景

作者在过去几周尝试将 Claude Code 用于硬件开发。尝试了类似软件开发的方式：用自然语言写提示词让 Claude 创建电路。但发现这种方法存在问题：
- 用简单的英语表达复杂电路设计比较困难
- 对于 trivial 电路效果不错，但复杂设计变得困难

作者意识到 Claude Code 的优势在于能够获得即时反馈。

### 新方法

赋予 Claude 访问示波器和 SPICE 模拟器的能力。这种方法在以下方面变得非常有价值：
- 验证 SPICE 电路和模型
- 嵌入式编程
- 数据分析（之前非常繁琐：规范化时间轴、对齐数据等，通常只能大致估计）

### 演示案例

作者展示了一个故意简单的演示，包含了电路和 MCU。目的不是电路本身，而是展示这种方法，可以扩展到更复杂的电路和实际嵌入式项目。

### 经验教训

**示波器相关**:
- Claude 看不到物理设置，不要让它猜测连接情况
- 确保 Claude 永远不会获得过时的测量数据
- 不要将原始数据直接输入 Claude 的上下文，保存到文件让 Claude 间接交互

**微控制器相关**:
- 明确给 Claude 提供引脚分配/引脚复用映射
- 准备一个 Makefile，暴露 build、flash、ping、erase 等功能，鼓励 Claude 依赖它，Claude 不应该自己构建这些命令

**仓库相关**:
- lecroy-mcp: LeCroy 示波器的 MCP 服务器
- spicelib-mcp: 封装 spicelib 的 MCP 服务器
- rc-filter-demo-files: 演示设置
