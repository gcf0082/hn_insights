# Amazon AI泡沫：重复工具与数据的乱象

## 核心洞察

Amazon内部AI工具和数据的重复问题只是一个表象，真正的问题在于：每个人都在各自为政地构建自己的解决方案，缺乏统一的愿景和协调。这种现象在AI时代更加严重——1000个工程师各自重复造轮子，比让1000人聚在一起讨论需求更容易。

## 关键问题

### 1. 工具泛滥（Tool Proliferation）

- 重复的工具和数据是Amazon的老问题了，甚至在"领导力原则"中都有提及
- AI时代让这个问题变本加厉：每个人都能快速生成代码，却缺乏统一的协调机制
- 只有当所有人都朝着同一个目标努力并共同承担责任时问题才能解决，否则只是在重复处理表象

### 2. 代码重复

- AI智能体不断重复编写相同的utility方法，却不检查现有代码
- 甚至在同一个PR中出现两次相同的逻辑（如图片MIME类型转扩展名的转换）
- 在一定规模下仍可能被发现并提取，但随着规模扩大会变得不可控
- 微服务可能更适合作为团队的隔离边界

### 3. 质量与价值主张不同

- AI生成的代码质量与传统手写代码的价值主张确实不同
- AI擅长快速生成初版代码，但6个月或1年后会显现问题
- 虽然短期对速度和稳定性没影响，但长期会积累技术债

## 新常态

- 生成式AI技术大部分时候既糟糕又昂贵
- AI让重复造轮子变得更容易，而不是促进协作
- 需要适应这个新常态，重新思考工程实践
- 领导者需要思考如何让AI生成代码与传统代码质量标准共存
