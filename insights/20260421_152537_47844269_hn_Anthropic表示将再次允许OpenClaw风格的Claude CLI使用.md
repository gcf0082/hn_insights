## 核心事件

OpenClaw 宣布 Anthropic 确认 OpenClaw 风格的 Claude CLI 使用再次被允许。这一消息是在 Anthropic 于约3天前宣布 OpenClaw 需要额外使用费用后恢复的。

## 社区反应分析

### 质疑与不信任

社区普遍对 Anthropic 的政策反复表示困惑和不满：

- **政策模糊性**：用户指出 Anthropic 员工在 Twitter 上的声明相互矛盾，官方没有发布正式声明
- **可靠性问题**：有用户将其比作 "PayPal 模式"——随时可能因任何原因封禁账户
- **缺乏透明度**：很多人表示不会再轻易相信 Anthropic 的承诺

### 用户流失趋势

多个用户表示已取消订阅并转向其他提供商：

- **转向 Codex**：许多用户转向 OpenAI 的 Codex ($20/月)，认为性价比更高
- **GPT-5.4 表现更优**：部分用户认为 GPT-5.4 xhigh 在编码任务上优于 Opus
- **中国模型备选**：有用户提到使用中国模型已能满足需求

### 对比观点

并非所有用户都持负面态度：

- **继续支持 Anthropic**：部分用户表示仍会继续使用，认为问题被过度放大
- **模型质量**：有用户认为 Anthropic 模型生成的代码质量更高、更易理解
- **功能迭代**：用户认可其持续推出的新功能如 Dispatch、Routines 和 Design

## 技术层面讨论

### Claude Code vs Codex

- **Claude Code**：被限制与第三方工具集成，限制较多
- **Codex CLI**：无繁复的确认提示，支持自动审查工具运行
- **使用限制**：Codex $200 计划一周内未耗尽，Claude Max $100 计划限制更严

### 设计工具评价

- **Claude Design**：在 UI 设计方面表现优于 Codex
- **Figma + Opus 4.6**：有用户尝试但效果不如 Claude Design
- **信息架构优先**：建议先构建 IA 再使用 agent 生成代码

## 启示与建议

1. **多元化策略**：建议在自动化工具中设置模型切换选项，降低单一提供商依赖风险
2. **订阅需谨慎**：当前 AI 领域变化快速，一年订阅可能相当于"十年"
3. **关注商业模式**：Harness 是护城河，而非模型本身，这是理解 AI 公司的关键
