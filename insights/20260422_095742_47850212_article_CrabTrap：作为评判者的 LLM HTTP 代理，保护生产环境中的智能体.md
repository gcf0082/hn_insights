## CrabTrap：生产环境中的安全代理

### 产品概述

CrabTrap 是 Brex 开源的 LLM 即判断代理安全工具，作为 HTTP 代理运行，能够 Intercept（拦截）AI 代理发出的每个请求。它根据预设策略对请求进行实时评估，并决定是否允许或阻止该请求。

### 核心功能

- **LLM 即判断（LLM-as-a-judge）**：利用 LLM 来判断请求是否符合安全策略，而不仅仅依赖静态规则匹配
- **实时拦截**：在请求到达目标系统之前进行过滤，确保只有合规的请求能够通过
- **决策日志**：记录每个决策是静态规则匹配还是 LLM 判断结果，便于审计和分析
- **快速部署**：声称可在 30 秒内完成安装配置
- **开源可用**：已在 GitHub 上开源，用户可以自行部署和使用

### 产品定位

CrabTrap 解决的问题是在生产环境中运行 AI 代理时的安全防护需求。当企业将 AI 代理部署到生产环境时，需要确保这些代理的行为符合企业安全策略，防止它们访问敏感数据或执行不当操作。传统的方法依赖静态规则，但 AI 代理的请求往往是动态的、上下相关的，静态规则难以覆盖所有场景。CrabTrap 引入 LLM 作为判断者，能够更智能地评估请求的意图和风险。

### 技术架构

从产品描述来看，CrabTrap 是一个 HTTP 代理层，位于 AI 代理和目标服务之间。它拦截所有出站请求，根据配置的策略进行评估。对于静态规则可以直接匹配，对于更复杂的场景则调用 LLM 进行判断。这种架构允许企业在不修改 AI 代理代码的情况下为其添加安全防护层。

### 市场意义

Brex 作为一家金融服务公司（提供企业信用卡和银行业务），开发这样一款安全工具反映了对 AI 代理安全问题的重视。随着 AI 代理在企业中的应用越来越广泛，如何确保它们的行为安全可控将成为一个重要的工程挑战。Brex 选择自己解决这个问题并开源，体现了其技术驱动企业文化。
