## 内容摘要

Qwen 发布了一款 270 亿参数的大语言模型，官方称其在编程任务上达到旗舰级水平，引发社区热烈讨论。

## 主要讨论要点

### 1. 运行硬件要求

- **27B 原始模型**：需要 Mac 或 Strix Halo 128GB 系统、高端消费级 GPU 或 RTX 6000 专业卡
- **量化版本**：Q4/Q5 量化可在 24GB VRAM 显卡（如 RTX 4090/5090）上运行
- **AMD 显卡**：Radeon R9700 32GB 是个性价比不错的选择，约 20 tokens/s
- **Mac 统一内存**：M4 Pro + 64GB 内存勉强运行，速度约 12-15 tokens/s

### 2. 性能表现（实测数据）

| 硬件配置 | 量化方案 | 速度 |
|----------|----------|------|
| M5 Pro 128GB | 量化版 | ~54 tokens/s (读取), ~26 tokens/s (生成) |
| M4 32GB | Q4_K_M | ~5 tokens/s |
| RTX 7900 XTX 24GB | IQ4_XS | ~33-36 tokens/s |
| AMD R9700 32GB | Q8_0 | ~20 tokens/s |
| 2x A6000 48GB | 全精度 | ~25 tokens/s |

### 3. 量化方案选择困惑

社区普遍反映量化选择复杂：
- Q3 有四个版本：UD-IQ3_XXS, Q3_K_S, Q3_K_M, UD-Q3_K_XL
- Q4 有七个版本：IQ4_XS, Q4_K_S, IQ4_NL, Q4_0, Q4_1, Q4_K_M, UD-Q4_K_XL
- **建议**：4-bit 量化几乎无损，可用于 24GB+ 显存的消费级显卡

### 4. 与闭源模型的对比

- **乐观派**：Qwen 3.6 35B 已经能满足 95% 的日常需求，社区认为可能接近 Opus 4.5 水平
- **谨慎派**：Benchmark 数据容易"刷分"，实际使用体验差距仍然明显
- **共识**：对于简单编码任务，本地模型已足够好；但复杂系统和深层推理仍需 Claude Opus

### 5. 实用建议

- **新手推荐**：使用 Unsloth Studio，自动选择最佳量化方案
- **显存判断**：用参数 B 前数字除以 2 即为 Q4_K_M 所需显存（GB），另加 KV cache 空间
- **本地运行**：14GB VRAM 可用 4-bit 量化运行此模型
