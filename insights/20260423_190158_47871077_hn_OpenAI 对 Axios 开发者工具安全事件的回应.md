## 事件概述

2026年3月31日，广受欢迎第三方开发者库 Axios 被攻击者攻陷，成为更广泛软件供应链攻击的一部分。OpenAI 在其 macOS 应用签名流程中使用的 GitHub Actions 工作流下载并执行了恶意版本的 Axios（1.14.1版）。该工作流拥有用于签名 macOS 应用程序的证书和公证材料，包括 ChatGPT Desktop、Codex、Codex-cli 和 Atlas。

## 影响范围

- **受影响应用**: ChatGPT Desktop, Codex, Codex-cli, Atlas
- **暴露的敏感材料**: 代码签名证书和 macOS 应用公证材料
- **未受影响**: 用户数据、密码、API 密钥、内部系统、知识财产、软件本身

## 根本原因

**GitHub Actions 工作流配置错误**：
1. 使用了浮动标签（floating tag），自动拉取最新版本的依赖而非固定的已验证版本
2. 缺少 minimumReleaseAge 设置，无法延迟采用新发布的包版本，使安全社区有时间发现异常

这种配置允许恶意 Axios 版本在发布后立即被自动拉取到构建流程中。

## OpenAI 采取的措施

1. **证书撤销与轮换**: 将签名证书视为已泄露并予以撤销和轮换
2. **安全认证更新**: 刷新安全认证，使用新签名材料
3. **工作流修复**: 已修复 GitHub Actions 工作流的配置问题
4. **用户通知**: 要求所有 macOS 用户更新到最新版本的 OpenAI 应用
5. **失效日期**: 2026年5月8日起，旧版本将不再接收更新或支持，可能无法使用

## 风险评估

OpenAI 分析认为，由于以下缓解因素，签名证书很可能未被成功外泄：
- 恶意负载执行的时机
- 证书注入作业的时间
- 构建作业本身的顺序

但出于高度谨慎，OpenAI 仍将证书视为已泄露并采取了预防性措施。

## 相关背景

该攻击被归因于与朝鲜有关联的威胁行为者，是同一时期针对多个组织的广泛供应链攻击的一部分。
