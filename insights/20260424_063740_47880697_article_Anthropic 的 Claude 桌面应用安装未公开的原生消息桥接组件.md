# Claude Desktop 隐私安全事件洞察

## 基本信息

- **洞察链接**: https://letsdatascience.com/news/claude-desktop-installs-preauthorized-browser-extension-mani-4064fb1a
- **标题**: Claude Desktop Installs Preauthorized Browser Extension Manifests
- **来源**: Let's Data Science
- **发布日期**: 2026-04-21
- **作者**: Let's Data Science

## 事件概述

安全研究员 Alexander Hanff 在调试其他应用程序时发现，Anthropic 的 Claude Desktop for macOS 在用户不知情的情况下，自动向用户计算机上的所有 Chromium 内核浏览器（包括未安装的浏览器）写入一个名为 `com.anthropic.claude_browser_extension.json` 的 Native Messaging 清单文件。

该文件预先授权了三个 Chrome 扩展 ID，允许这些扩展与本地可执行文件进行通信，从而实现浏览器自动化功能，而无需用户同意。

## 技术分析

### Native Messaging 机制

Native Messaging 是 Chromium 浏览器的一种API，允许浏览器扩展与本地应用程序进行通信。通过在浏览器配置目录中写入清单文件，extension 可以调用指定的本地二进制程序。

### 预授权的扩展

该清单文件预先授权了三个扩展 ID，包括 Claude for Chrome 扩展。这意味着：

1. 任何基于 Chromium 的浏览器（如 Chrome、Edge、Brave、Arc、Vivaldi、Opera 等）安装后，将自动获得这些扩展的通信权限
2. 扩展可以在浏览器沙箱之外以用户权限运行本地二进制程序
3. 用户不会收到任何权限提示或通知

### 安全风险

1. **攻击面扩大**: Noah Kenney（数字安全咨询公司 Digital 520 创始人）指出，这大大扩展了系统的攻击面
2. **权限提升**: 本地二进制程序以用户权限运行，绕过了浏览器沙箱保护
3. **持久化**: 每次启动 Claude Desktop 时，文件会被重写，且没有任何通知或退出选项
4. **潜在滥用**: 如果恶意扩展或通过提示注入攻击结合此机制，可能导致：
   - 读取网页内容和表单数据
   - 捕获屏幕内容
   - 访问已认证的会话（包括银行、医疗、企业账户）
   - 数据外泄

## 隐私与法律问题

该行为引发了严重的隐私和法律问题：

1. **未经同意的数据处理**: 用户未被告知或询问许可，违反了知情同意原则
2. **ePrivacy 指令第 5(3) 条**: 可能违反欧盟 ePrivacy 指令关于未经用户同意存储或访问用户设备信息的规定
3. **透明度缺失**: 没有向用户提供通知或选择退出的选项

## 行业反应

- **Anthropic**: 尚未发布公开回应或承认此发现
- **安全社区**: 广泛关注该发现的隐私和安全影响
- **建议**: 如果用户担心此行为，全面卸载 Claude Desktop 是唯一的解决方法

## 总结

此发现揭示了 Claude Desktop 应用程序存在的一个重大隐私和安全问题。虽然 Anthropic 可能意在实现「Chrome 中的 Claude」功能，但预安装清单文件到未安装的浏览器配置目录、且未告知用户的行为，引发了对用户信任和数据保护的严重担忧。
