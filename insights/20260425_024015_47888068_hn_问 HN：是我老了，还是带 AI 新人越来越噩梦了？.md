**标题**: Ask HN: Am I getting old, or is working with AI juniors becoming a nightmare?
**发帖人**: MichaelRazum

**分数**: 31分
**评论数**: 43条

## 核心问题

发帖者观察到来自顶尖大学的初级开发者过度依赖AI工具，导致代码质量下降、难以进行代码评审，引发对软件行业未来的担忧。

## 主要观点

### 1. AI Junior带来的困境

- 初级开发者过度依赖ChatGPT等工具，难以解释自己提交的代码
- 代码变得臃肿（100行替代原本可读的简短代码），但无人关心可读性
- 代码评审变得更加困难，开发者对自己代码的理解不足
- 难以说服团队成员改变现状，因为所有内容都经过AI交叉验证

### 2. 对未来的两种预测

**悲观派**：
- AI生成的代码将污染互联网，未来AI可能被迫在自身生成的低质量代码上训练
- 初级开发者将失去编写优质代码的能力
- 当资深开发者退休后，技术传承可能断裂

**乐观派**：
- AI技术会持续进步，最终可能不再依赖人类编写的代码
- "prompt engineer"可能成为新的职业方向
- 软件开发最终可能像使用计算尺一样过时

### 3. 资深开发者的核心价值

- 能够正确审查和评估AI生成的内容
- 具备判断代码是否符合业务需求的能力
- 理解代码背后的逻辑和架构
- 知道何时应该让AI介入，何时应该手动处理

### 4. 实用建议

- 严格控制使用场景：部分公司已明确禁止使用AI进行开发
- 增加规划和文档工作，进行更多测试
- 让AI专注于减少认知负担，而非炫耀技术
- AI适合处理重复性任务，但复杂系统仍需人工把关

### 5. 争议与质疑

- 部分用户质疑AI厂商的数据可信度
- 对"AI代码审查员"等新概念持保留态度
- 有人认为这是技术进步的自然过程，无需过度担忧

## 关键引用

> "AI systems look at code on the internet that was written by humans... What they produce — unreadable spaghetti code — is the maximum they can squeeze out of the best code written by humans." — baCist

> "If you don't understand the code the AI agent outputted, you shouldn't let other people run it in production." — tracker1

## 结论

社区普遍认为AI工具本身无害，但初级开发者缺乏批判性评估AI输出的能力，这是核心问题所在。关键在于找到人类判断力与AI效率之间的平衡点，而"AI编排能力"可能成为未来开发者的核心竞争力。
