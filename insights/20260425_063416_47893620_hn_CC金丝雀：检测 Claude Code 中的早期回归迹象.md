## 工具概述

cc-canary 是一个用于检测 Claude Code 性能回归（drift）的工具，包装为两个可安装的 Agent Skills。它读取 Claude Code 已经写入 `~/.claude/projects/` 的 JSONL 会话日志，检测模型是否在用户自己的工作中出现漂移，并生成可分享的取证报告。

### 核心特性

- **完全本地化**: 无网络、无账户、无遥测、无后台守护进程
- **运行在磁盘数据上**: 仅读取现有 JSONL 文件
- **输出格式**: Markdown 取证报告或 HTML 仪表板
- **时间窗口**: 支持 7d / 14d / 30d / 60d / 90d / 180d

### 追踪指标

- **读写比率**: 每次编辑前的文件阅读量
- **写入占比**: Write / (Edit + Write)
- **推理循环**: "let me try again"、"oh wait"、"actually"等短语
- **挫败率**: 提示中的挫败词汇比率
- **思维可见性**: 思维块被编辑vs可见的比例
- **API轮次**: 每个用户消息的API调用次数

## 社区讨论要点

### 1. 自我测量的困境

评论者指出："让被测量的事物自己去测量自己，可能不是最好的方法。"这类似于 "我们调查了自己，没发现问题" 的讽刺。

### 2. "Drift"的含义模糊

- "Drift" 是 LLM 领域流行但含义模糊的词汇之一
- 原始含义：分布随时间的变化（如训练数据分布与实际使用分布的漂移）
- 在商业语境中，"drift" 几乎等同于 "变化"

### 3. 漂移的多种来源

- 可以通过 harness 修改系统提示
- 可以修改底层模型
- 可以在服务器端根据变量处理请求
- 可以动态修改计算分配
- 用户无法察觉这些变化，也没有法律义务要求告知

### 4. 评估的成本问题

对于 solo 开发者来说，运行评估的成本过高。有建议提到 MarginLab 的 Claude Code 历史性能追踪器作为更传统的方法。
