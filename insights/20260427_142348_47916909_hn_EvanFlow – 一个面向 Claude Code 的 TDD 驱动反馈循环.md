## 项目概述

EvanFlow 是一个针对 Claude Code 的 TDD（测试驱动开发）驱动反馈循环工具。该项目是一个基于证据和实践的固执己见的开发流程。

## 核心功能

EvanFlow 提供一个单一的 TDD 驱动循环，说 "let's evanflow this" 即可启动以下流程：**头脑风暴 → 计划 → 执行 → TDD → 迭代 → 停止**。该流程在设计和计划阶段设有真实的检查点，从不自动提交、自动暂存或自动提出集成。

## 三大核心特性

### 1. 垂直切片 TDD
- 一个失败的测试 → 最小实现 → 下一个测试
- 在编写实现之前先观察每个测试失败
- 根据 HumanEval 研究，约 62% 的 LLM 生成的测试断言是错误的，因此 TDD 纪律比实现纪律更重要

### 2. 决策点嵌入式"拷问"
- 在锁定计划前问三个问题：
  - 如果用户做 X 会发生什么？
  - 回滚方案是什么？
  - 什么明确不在范围内？
- 在设计缺陷还便宜时将其捕获

### 3. 迭代直到清洁（硬性上限 5 轮）
- 重新阅读 diff，检查死代码、命名、删除的测试、断言正确性
- 五种失败模式检查：
  - 幻觉行为
  - 范围蔓延
  - 级联错误
  - 上下文丢失
  - 工具误用
- 对于 UI：通过无头 Chromium 截图

## 大型计划处理

对于包含 3 个或更多共享类型独立单元的大型计划，会分叉为并行 coder/overseer 编排，集成测试作为接触点的聚合契约。
