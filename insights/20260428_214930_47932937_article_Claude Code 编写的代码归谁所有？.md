# 洞察报告：谁拥有 Claude Code 编写的代码？

## 基本信息

- **洞察链接**: https://legallayer.substack.com/p/who-owns-the-claude-code-wrote
- **报告生成时间**: 2026-04-28
- **主题**: AI生成代码的版权归属问题——以 Claude Code 源代码泄露事件为切入点

---

## 事件背景

2026年3月31日，Anthropic 在一次例行 npm 更新中，意外将 Claude Code 的完整 sourcemap 文件（包含约512,000行 TypeScript 源代码）打包进了发布的 npm 包中。安全研究员 Chaofan Shou 在 X 上首次公开披露了这一泄露事件。短短几小时内，完整的源代码被归档至多个公开 GitHub 仓库，其中一个镜像仓库在一天内获得了近30,000个 star 和40,300个 fork。

Anthropic 随即向各镜像仓库发送了 DMCA 删除通知。然而，一个名为 `claw-code` 的仓库采用了"净室重写"（Clean Room Rewrite）策略——使用 OpenAI 的 Codex 将 TypeScript 代码从零开始重写为 Python 版本，从而规避了版权主张。该仓库在两小时内获得了50,000个 GitHub star，目前仍在运行中。

---

## 核心法律问题

### 1. AI生成内容的版权空白

根据美国版权局和哥伦比亚特区巡回法院的明确立场：**只有人类创作的作品才有资格获得版权保护**。如果代码完全由 AI 生成，且人类对代码的具体表达形式没有施加任何创造性控制，则该代码不受版权保护。

Claude Code 负责人 Boris Cherny 曾在公开场合表示："过去30天内，我100%的 Claude Code 代码贡献都是由 Claude Code 自己编写的"，他甚至"不做任何小的编辑"。如果法院采纳 Thaler 案确立的原则，认定 Claude Code 的相当一部分代码完全由 AI 生成而缺乏足够的人类创造性贡献，则这些部分将不受版权保护。

### 2. "undercover.ts" 文件揭示的双重标准

泄露的源代码中包含一个名为 `undercover.ts` 的文件（约90行），该文件实现了一种"隐身模式"：当 Anthropic 员工使用 Claude Code 为外部开源项目贡献代码时，系统会指示 AI 不要添加"Co-Authored-By"署名，也不要在 commit 消息中提及 Claude 或 Anthropic。

这一发现暴露了一个显著矛盾：Anthropic 一方面通过 DMCA 主张其 Claude Code 源代码的版权，另一方面却使用其工具系统性地抹去 AI 参与的证据——而这些证据恰恰是法院评估人类与 AI 贡献比例时最具价值的材料。

### 3. DMCA 通知的法律基础存疑

如果 Claude Code 的部分代码是由 Claude 本身生成的（Anthropic 首席执行官已公开暗示），那么 Anthropic 依据版权法对这些代码发出的 DMCA 删除通知，在法律上存在根本性的疑问。人类开发者无疑对代码的架构、集成和表达元素做出了创造性选择，但任何由 AI 在缺乏有意义的人类创造性贡献的情况下生成的代码部分，都无法为版权主张提供依据。

### 4. 净室重写的法律地位

Clean Room  doctrine 的先例（Phoenix/IBM 案、Sega v. Accolade 案、Sony v. Connectix 案）均涉及人类团队做出独立创造性选择的情况。尚无法院裁决过 AI 驱动的净室重实现是否符合相同标准��

更重要的是，这一事件引出了一个关键的法律悖论：如果"AI 从受版权保护的作品中学习并生成新输出"属于合理使用（这是 Anthropic 在训练数据诉讼中的立场），那么"Codex 从 Claude Code 中学习并生成 Python 重写版"在逻辑上应适用相同原则。

## 对行业的启示

### 版权归属的不确定性

AI 生成代码的版权归属问题尚无定论。根据 Anthropic 的商业服务条款，公司将代码权利转让给用户并承诺为客户辩护版权索赔。但有一个重要的但书：他们承认，纯 AI 生成的代码部分可能缺乏版权保护，因为 Anthropic 无法授予本身不存在的权利。

### 合同协议的盲区

一个真实的案例说明了这一问题的现实影响：一位营销代理商雇佣承包商使用 AI 生成网站文案，但合同中使用了几十年来一直沿用的标准语言——"承包商创建的所有工作成果归代理商所有"。由于协议未解决 AI 的角色问题，代理商试图许可该内容给第三方时遭遇了法律困境，最终导致了 £400,000 的损失。

### 许可协议的效力存疑

AI 生���的代码可能缺乏版权保护这一事实，对 MIT、GPL、Apache、BSD 等许可协议的效力提出了根本性疑问。如果 AI 可能没有法律地位授予许可，而人类也可能没有地位批准该许可，那么这些许可协议实际上可能正在授予无人有权授予的权限。

## 总结

Claude Code 源代码泄露事件不仅仅是一个安全漏洞，更是一面镜子，映照出 AI 编程时代法律框架的滞后与空白。核心问题在于：当代码的原始"作者"是一个 AI 系统时，"所有权"、"许可"和"侵权"这些传统法律概念是否仍然适用？

目前，法律界、技术界和政策制定者都在等待这一问题的答案。在那之前，任何使用 AI 生成代码的组织都应谨慎审视其合同协议和许可声明，以避免潜在的法律风险。
