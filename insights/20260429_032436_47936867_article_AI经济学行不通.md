# AI经济学为何不合理

> **洞察来源**: [AI's Economics Don't Make Sense](https://www.wheresyoured.at/ais-economics-dont-make-sense/)
> **作者**: Ed Zitron
> **发布日期**: 2026年4月28日
> **阅读量**: 37分钟

## 核心观点摘要

本文揭示了AI行业普遍存在的经济模式缺陷，从GitHub Copilot转向按token计费说起，深入剖析了AI订阅制为何本质上是一个不可持续的商业模式，以及整个AI数据中心行业面临的严峻财务现实。

---

## 1. 订阅制模式的根本缺陷

### 1.1 用户成本与订阅价格严重倒挂

**核心问题**: AI公司长期以远低于实际成本的价格提供服务，每用户每月亏损超过20美元。

- 个别用户每月消耗高达80美元的算力成本
- 标准订阅仅收取10美元（GitHub Copilot）、20美元（ChatGPT）、100美元（Claude Max）或200美元
- Anthropic曾允许用户每花费1美元订阅就消耗8美元的算力
- 这种模式本质上是在烧钱获客，期望未来能降低推理成本

### 1.2 新一代模型反而增加成本

讽刺的是，随着"推理模型"（reasoning models）的出现，token消耗不降反升：

- 推理模型需要更多的计算资源
- 推理成本实际上随时间推移而增加
- 这与行业预期的成本下降完全相反

> **翻译一下微软的说辞**: "我们已经无法继续补贴GitHub Copilot用户了，否则Amy Hood就要开始用棒球棍打人了。"

## 2. 企业级AI成本真相

### 2.1 Claude Code的实际使用成本

根据Anthropic官方文档更新后的数据：

| 成本指标 | 数值 |
|---------|------|
| 每开发者日均成本 | 13美元 |
| 月均成本 | 150-250美元 |
| 90%用户日均上限 | 30美元 |
| 年均成本（按21工作日） | 2,736-6,300美元 |

### 2.2 企业实际支出示例

假设一个10人开发团队：

- **保守估计**: 7,560美元/年
- **中等估计**: 88,200美元/年（平均50美元/天）
- **高强度使用**: 756,000美元/年（300美元/天）

> 原文指出："对于一个真正关注成本的企业来说，很难证明每年在这些服务上花费五到六位数的费用是合理的，而这些服务据称能以无人能衡量的方式'提高生产力'。"

### 2.3 大公司AI支出比例

根据高盛估算：

- 部分公司AI token支出已达**人力成本的10%**
- 预计未来几个季度可能达到**人力成本的100%**

Uber的CTO透露，公司在2026年的AI预算几个月内就花光了。

## 3. 数据中心经济的崩溃

### 3.1 数据中心建设成本分析

**100MW数据中心的经济账**:

| 项目 | 成本 |
|------|------|
| IT设备（GPU等） | 31.26亿美元 |
| 每年电费 | 5,540万美元 |
| 运维成本 | 1.28亿美元/年 |
| 托管费用 | 1.33亿美元/年 |
| 折旧（6年） | 5.21亿美元/年 |
| **年收入预期** | **10.63亿美元** |

**结果**: 即使100%满负荷运转，年利润率仅约**5.19%**，尚需6年才能收回投资。

### 3.2 实际运营的艰难现实

- 数据中心需要1-2个月安装GPU和客户入驻
- 期间零收入但仍需支付全部费用
- **每天亏损约327万美元**
- 如果使用债务融资（80% LTV，6%利率），处境更加艰难

### 3.3 建设进度严重滞后

截至2028年计划建设的114GW数据中心中：

- **实际在建仅15.2GW**
- "在建"可能仅指"地上有个坑"
- 更关键的是，**50%以上是为OpenAI和Anthropic两家公司建造**
- 如果这两家无力支付，整个系统存在系统性风险

## 4. Stargate Abilene项目：最大灾难

### 4.1 项目概况

- **规模**: 8栋建筑，1.2GW容量
- **位置**: 德克萨斯州阿比林
- **总投资**: 约528亿美元
- **唯一客户**: OpenAI
- **预计年收入**: 100亿美元

### 4.2 严重延期

原计划 vs 实际进度：

| 时间点 | 原计划 | 实际 |
|--------|--------|------|
| 2025年初 | 200MW通电 | 延迟 |
| 2025上半年 | 开始入驻 | 延迟 |
| 2025年底 | 64,000 GPU | 96,000 GPU（仅2栋） |
| 2026年中 | 1.2GW全完工 | 仅206MW可用 |

### 4.3 Oracle的财务风险

Oracle为OpenAI建设的数据中心：

- 已承担约**1,150亿美元债务**
- 还需要**1,500亿美元**完成项目
- 上季度自由现金流为**-247亿美元**
- 其他业务线增长停滞

> 文章警告："如果OpenAI无力支付这些计算费用，Oracle就会死亡。"

## 5. OpenAI的财务黑洞

### 5.1 收入与支出预测（2026-2030）

| 指标 | 金额 |
|------|------|
| 预计总收入 | 6,730亿美元 |
| 预计总支出 | 8,520亿美元 |
| **净亏损** | **1,790亿美元** |

### 5.2 离谱的增长要求

为支付所有计算合同，OpenAI需在4年内：

- 筹集或赚取**8,520亿美元**
- 收入从当前水平增长**250%以上**，每年翻倍
- 到2030年收入需达到**3,000亿美元**（与微软持平）
- 且必须实现现金流为正

> "一家新闻机构如果不指出这些数字有多愚蠢，就应该感到羞耻。"

### 5.3 CFO的担忧

据Wall Street Journal报道，OpenAI CFO Sarah Friar已告知管理层：

- 公司可能无法按计划支付未来计算费用
- 财务内控尚不符合上市公司标准
- 最近刚错过用户和收入目标

## 6. 行业的欺骗性做法

### 6.1 故意隐藏成本

AI公司故意 obfuscate（混淆）真实成本：

- 使用"token"、"消息"、"百分比"等模糊概念
- 用户无法准确知道每次任务消耗多少
- 故意设置复杂的费率限制，使用户难以计算

### 6.2 先低价吸引，再大幅涨价

典型的"诱饵换岗"策略：

1. 先提供无限量或低价订阅吸引用户
2. 让用户产生依赖
3. 用户习惯后再大幅涨价
4. 用户已无法轻易迁移

### 6.3 媒体配合宣传

主流科技媒体（如The Atlantic）撰写Claude Code评测时：

- 使用20美元月费订阅进行体验
- 对模型的小错误"非常宽容"
- 完全忽视背后的实际token消耗成本

> "这是骗局的一部分——非常重要的一点是，写AI报道的主流媒体记者确实不知道这些服务的真实成本。"

## 7. 未来展望与预警信号

### 7.1 不可避免的全面转向

**两大警示**:

1. Microsoft（资本最雄厚）已放弃补贴，转向按token计费
2. Anthropic和OpenAI企业客户已采用类似模式

> "一旦所有订阅都转向按token计费，我就不确定我们还能看到那么多关于生成式AI的宣传了。"

### 7.2 真正的"灰马骑士"风险

作者预计的下一个重大事件：

> "真正的警示——一匹真正的灰马——将是一家大型AI实验室（如Anthropic或OpenAI）将所有订阅转向按token计费。那一天到来时，就是关门的时候。"

### 7.3 ROI困境

每项关于AI投资回报的研究都发现：

- 几乎没有证据证明AI带来了实际ROI
- 企业对AI的投入与回报严重脱节
- 投资者终将质疑持续增长的R&D成本

## 结论

AI经济学不合理的根本原因在于：

1. **商业模式错误**: 月度订阅制不适合基于token消耗的服务
2. **补贴不可持续**: 每用户都在亏损，且亏损额随使用量增加
3. **成本认知错位**: 用户被故意引导远离真实成本
4. **基础设施过度投资**: 数据中心建设远超实际需求
5. **客户集中度过高**: 绝大多数数据中心依赖1-2家AI公司
6. **财务预测离谱**: OpenAI需要的天文数字资金在商业上不可行

这篇文章的核心洞察是：**整个AI行业建立在补贴和幻觉之上，当补贴结束时，一切都会崩溃。**
