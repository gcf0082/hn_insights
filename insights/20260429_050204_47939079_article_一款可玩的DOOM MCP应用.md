## 项目概述

Chris Nager 创建了一个可玩的 DOOM MCP 应用，能够在兼容的 AI 客户端（如 ChatGPT 和 Claude）中内联运行，同时也支持在任何浏览器中通过 URL 回退运行。

## 技术架构

该方案包含以下核心组件：

- **一个 MCP 工具**：用于创建 DOOM 会话
- **一个 MCP 工具**：用于返回普通的启动 URL
- **一个浏览器路由**：`/doom/play`
- **签名令牌流程**：在两种环境中均可工作

### 部署方案

- 轻量级的 MCP 服务器
- 浏览器 DOOM 运行环境
- 通过启动 URL 传递的签名令牌
- Netlify 部署用于 `/doom/*` 和 `/doom/mcp` 路径

## 核心功能

### 1. 创建 DOOM 会话

第一个目标是暴露一个创建 DOOM 会话并返回签名启动 URL 的工具，这成为 `create_doom_session`。

### 2. 获取启动 URL

后来添加了 `get_doom_launch_url` 作为更简单的回退方案，适用于无法渲染内联应用界面的客户端。

### 3. 签名令牌设计

重要的设计决策是保持启动路径自包含。URL 中的签名令牌足以启动游戏，这意味着浏览器路由不需要依赖服务器端会话持久化来开始游戏。

## 路由结构

- `/doom/play` - 浏览器播放入口
- `/doom/mcp` - MCP 端点

## 当前功能

当前版本可以：

- 在支持内联视图的 MCP 应用主机中内联启动 DOOM
- 在其他任何环境中返回正常的浏览器 URL
- 在内联和外部视图两种情况下使用相同的签名令牌会话模型
- 保持足够的自包含性，浏览器路径无需持久存储即可开始游戏

## 意义

这个项目证明了 ChatGPT 和 Claude 可以运行 DOOM。DOOM 部分运行在 ChatGPT 和 Claude iOS 应用中。

这是一个有趣的探索，展示了 AI 客户端的潜力，不仅仅是文本交互，还可以运行交互式的游戏体验。
