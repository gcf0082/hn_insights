## 项目概述

Cua Driver 是一个后台运行的计算机操作驱动工具，专为 macOS 设计，允许 AI 代理在后台点击、输入、滚动和读取原生应用程序，同时保持用户的鼠标光标、最前面的应用程序和 Space 位置不变。

## 核心问题

当 UI 自动化进程控制桌面应用程序时，通常会接管人类的会话：光标移动、键盘焦点被夺走、窗口跳到前台，用户必须停止工作直到代理完成。这也是为什么历史上倾向于避免让用户直接在主机上运行这些进程，而是依赖虚拟机或 GUI 容器来实现并发和后台执行。

## 技术挑战

1. **CGEventPost**: 通过硬件输入流路由，会移动光标
2. **CGEvent.postToPid**: 不会移动光标，但 Chromium 将这些事件视为不受信任，在渲染器边界静默丢弃点击
3. **激活目标应用**: 首先会提升窗口并获取焦点，违背了后台执行的目的

## 解决方案

**SkyLight** 是关键突破：`SLEventPostToPid` 是公共 per-PID 调用的兄弟函数，但它通过 Chromium 接受的 WindowServer 通道传输。结合 yabai 的 focus-without-raise 模式，以及在 (-1, -1) 位置的离屏初始点击，点击可以落地而无需提升窗口。

## 应用场景

- 委托演示录制：用 Claude Code 驱动应用，同时用 cua-driver recording start 捕获轨迹、截图、操作和点击标记
- 替换 Vercel 的 agent-browser 和其他浏览器操作 CLI
- 开发循环 QA 代理：在编辑器保持前端的情况下重现视觉 bug、编辑代码、重新构建并验证 UI
- 个人助理流程：使用 iMessage
- 从 Chrome、Figma、Preview 或 YouTube 窗口拉取视觉上下文

## 技术要点

正确的寻址模式取决于应用程序：
- 原生 macOS 应用程序通常有丰富的 AX 树
- Chromium 系列应用程序通常需要 AX 和截图的混合
- Blender 或 CAD 等工具可能几乎不暴露有用的 AX 表面

错误的方式是在所有地方默认使用像素，或在所有地方默认使用 AX。
