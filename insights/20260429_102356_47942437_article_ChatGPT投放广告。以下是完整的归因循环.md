## 执行摘要

本报告详细分析ChatGPT广告投放系统的技术实现，涵盖广告注入机制、投放策略和归因追踪完整链路。

## 一、广告注入机制

### 1.1 技术架构

ChatGPT后端通过SSE（Server-Sent Events）流在对话响应中注入广告单元。关键端点为`chatgpt.com/backend-api/f/conversation`。

### 1.2 广告数据结构

广告以`single_advertiser_ad_unit`类型结构化数据发送，包含以下核心字段：

- **ads_request_id**: 请求唯一标识
- **advertiser_brand**: 广告主品牌信息（名称、URL、favicon、账户ID）
- **carousel_cards**: 广告卡片数组（标题、描述、图片、跳转链接）

### 1.3 关键发现

- 广告主创意内容托管在OpenAI的`bzrcdn.openai.com`CDN
- 链接默认在ChatGPT应用内WebView打开（非外部浏览器）
- 每个广告包含4个Fernet加密令牌，用于不同阶段的归因追踪

## 二、广告投放策略

### 2.1 上下文定向

测试账户在同一会话中接收到6个不同主题的广告，证明投放采用**对话上下文定向**：

| 对话主题 | 投放广告主 |
|---------|-----------|
| 北京行程规划 | Grubhub |
| 北京旅游预订 | GetYourGuide |
| 北京航班 | Axel |
| NBA季后赛 | Gametime |
| 春季时尚 | Aritzia |
| 演示文稿 | Canva |

### 2.2 待确认问题

目前无证据表明是否整合历史对话记录进行个性化投放。

## 三、归因追踪体系

### 3.1 四令牌归因链

每个广告携带4个Fernet加密令牌，各司其职：

| 令牌名称 | 位置 | 用途 |
|---------|------|------|
| ads_spam_integrity_payload | SSE数据内 | 服务端防作弊校验 |
| oppref | 点击URL + Cookie | 前向归因令牌（30天TTL） |
| olref | 点击URL | 出站链接引用日志 |
| ad_data_token | SSE载荷 | 服务端对账使用 |

### 3.2 Fernet令牌解析

Fernet令牌前9字节为公开信息（版本字节+8字节时间戳），可无需密钥提取mint时间：

```python
import base64, struct, datetime
b = base64.urlsafe_b64decode("gAAAAABp7fdA" + "==")
print(datetime.datetime.utcfromtimestamp(struct.unpack(">Q", b[1:9])[0]))
# → 2026-04-26 11:30:08 UTC
```

测试点击延迟：95秒（mint到页面加载）

### 3.3 商户端SDK

OAIQ SDK（`bzrcdn.openai.com/sdk/oaiq.min.js` v0.1.3）工作流程：

1. 读取URL中的`oppref`参数
2. 写入第一方Cookie `__oppref`（720小时TTL）
3. 设置探测Cookie `__oaiq_domain_probe`
4. 通过`measure`调用上报事件至`bzr.openai.com/v1/sdk/events`

## 四、隐私与安全影响

### 4.1 需要屏蔽的域名

- `bzrcdn.openai.com`（广告资源）
- `bzr.openai.com`（事件上报）

### 4.2 需要监控的Cookie

- `__oppref`
- `__oaiq_domain_probe`
