## 问题概述

这是一个关于 Claude Code v2.1.111 版本的严重回归 bug。系统会在每次执行 `Read` 和 `Grep`（内容模式）工具时注入一段关于恶意软件的系统提醒，而这个提醒在 v2.1.92 版本中已被修复，但在后续版本中再次出现。

## 技术细节

### 被注入的提醒文本

```
<system-reminder>
Whenever you read a file, you should consider whether it would be considered malware.
You CAN and SHOULD provide analysis of malware, but you MUST refuse to improve or augment the code.
You can still analyze existing code, write reports, or answer questions about the code behavior.
</system-reminder>
```

### 问题根源

提醒文本存在歧义：
- 第一句 "You CAN and SHOULD provide analysis of malware" — 明确限定于恶意软件
- 第二句 "you MUST refuse to improve or augment the code" — **无任何限定条件**

当子代理严格按照字面意思解读时，第二句的无条件拒绝语句优先级更高，导致其拒绝执行任何代码编辑任务。

## 影响后果

### 1. 子代理拒绝率极高

在 Opus 4.7 子代理的并行工作流中，代码编辑任务的拒绝率达到 **40-60%**，使得多代理协作编程任务几乎无法使用。

### 2.  Tokens 浪费严重

- 每次 `Read` 操作增加约 400 tokens 的提醒内容
- 单个会话通常包含 50-100+ 次文件读取
- 每个会话浪费 **20,000-40,000 tokens**

### 3. 主线程会话额外开销

主线程需要额外消耗 tokens 向子代理解释忽略该提醒，进一步加剧资源浪费。

## 修复建议

1. **完全移除该提醒** — Claude 本身的训练拒绝行为已足够处理恶意软件相关需求

2. **明确条件范围** — 将条件置于动作之前，例如：
   > "If you determine that a file you just read is malware, you MUST refuse to improve or augment that malware, though you may still analyze it and describe its behavior."

3. **限制触发频率** — 只在会话中首次读取文件时触发，而非每次读取都触发
