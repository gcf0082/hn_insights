## 核心洞察

Mendral 公司通过采用分层代理架构，成功将 LLM 成本降低了超过 50%。这一方案的核心在于：**让廉价模型处理绝大多数任务，仅在必要时才调用昂贵的 frontier 模型**。

## 架构设计

### 三层代理系统

```
┌─────────────────────────────────────────────────────────────┐
│                    Opus (昂贵模型)                          │
│              仅处理 20% 需深度调查的复杂问题                 │
└─────────────────────────────────────────────────────────────┘
                              ▲
                              │ 升级
                              │
┌─────────────────────────────────────────────────────────────┐
│               Haiku (廉价模型-triager)                       │
│         负责 80% 的问题分类和重复检测                        │
└─────────────────────────────────────────────────────────────┘
                              ▲
                              │ 下沉
                              │
┌─────────────────────────────────────────────────────────────┐
│            Haiku (廉价模型-workers)                        │
│              执行具体的搜索和验证任务                        │
└─────────────────────────────────────────────────────────────┘
```

### 分层成本对比

| 模型 | 处理内容 | 成本比率 |
|------|----------|----------|
| Haiku (triager) | 重复检测、问题分类 | 1x |
| Haiku (workers) | 具体搜索任务 | 1x |
| Opus | 复杂调查、规划 | 25x+ |

## 关键实践

### 1. 分类器模式 (Triager Pattern)

- 使用 Haiku 作为分类器，决定问题是否需要升级
- **80% 的失败请求never reach Opus**
- 仅需 25x 低于完整调查的成本

### 2. 语义搜索检测重复

- 使用 pgvector 进行相似错误匹配
- 精确匹配 + 语义搜索双重验证
- 示例：`operator does not exist bigint character varying` 和 `migration type mismatch` 虽然字符串不同，但根源相同，语义搜索能识别

### 3. 拉取而非推送上下文

- 不将日志push到prompt中
- 给代理 SQL 接口，让其自主查询
- **避免提前预设答案，干扰调查方向**

### 4. 分层规划与执行

- **Opus 负责规划**：形成假设、分配子任务
- **Haiku 负责执行**：执行具体搜索任务
- 子任务仅一层深度，避免无限制的成本扩散

### 5. 上下文清洁

- 每个子代理独立上下文，执行完毕即丢弃
- 只传递结构化摘要，不传递原始日志
- 避免上下文污染导致决策错误

## 成本效益

| 指�� | Haiku | Opus |
|------|-------|------|
| 处理输入 token 占比 | ~65% | ~35% |
| LLM 支出占比 | ~36% | ~64% |
| 输入输出 token 比率 | 86:1 | 50:1 |

- Haiku 吸收数据，Opus 不必直接读取
- 在不使用分层架构的情况下，账单会翻倍倍以上

## 核心经验

> 最困难的不是让代理变得更聪明，而是构建阻止它不应该运行时的防护层。

## 可推广场景

此模式适用于任何高事件量场景：
- 安全日志分析
- IoT 遥测数据
- 金融交易监控

**核心原则**：大多数事件是噪音或重复，昂贵的模型应该只看到有意义的新问题。
