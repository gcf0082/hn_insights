## 洞察内容

### 核心发现

Theo 发现了一个关于 Claude Code 的严重问题：如果用户的代码库中最近有提交包含 "OpenClaw" 关键词（即使只是在 JSON 配置中），Claude Code 会：

1. **拒绝用户的请求** - 直接拒绝处理
2. **额外收费** - 向用户收取额外费用

值得注意的是，这是一个空仓库，Theo 只是直接调用 Claude Code，并没有实际使用任何第三方工具。这一行为被 Theo 形容为"疯狂"（Insanity）。

### 社区反应

这条推文引发了广泛讨论，许多开发者表达了强烈不满：

- **Raj Pundkar**: "Anthropic 就像吃醋的前任，在 git 历史和 JSON blob 中搜索 'OpenClaw'，然后要么拒绝工作，要么收取额外的'使用费'。OpenClaw 只是开源的代理工具，实际上在你的机器上运行，可以与任何模型配合。提到竞争对手的元数据就能破坏他们的'编码代理'，这充分说明了他们的护城河有多脆弱。"

- **Curtis**: "你的代码编辑器现在在读取你的 package.json 来决定你是不是竞争对手。这是什么时代啊！"

- **Master_Kush**: "我刚取消了 Claude Code 的 max 订阅，转向 Codex。用多少都可以，棒极了。"

- **Nish**: "有人说是为了安全离开 OpenAI，这就是他们所说的'安全'原因。"

- **Maciej Litwiniuk**: "亲自测试了一下，太疯狂了。Anthropic 做得太过分了。"

### 问题分析

1. **过度竞争防御**: Anthropic 通过检测用户代码库中是否提及竞争对手（OpenClaw）来限制服务，这种做法被认为过于激进。

2. **对开源生态的影响**: OpenClaw 是一个开源的代理工具框架，可以在用户本地运行并支持任何模型。Anthropic 的这种行为被解读为对开源竞争对手的打压。

3. **用户信任危机**: 社区普遍认为 Anthropic 正在"摧毁"其辛苦建立的用户好感度。有用户提到 Dario（Anthropic CEO）四处宣扬"软件工程已死"，进一步加剧了负面印象。

4. **潜在的Prompt注入风险**: 有开发者指出，Anthropic 将用户 commit 历史放入 agent 系统 prompt 中，可能存在 prompt 注入的安全隐患。

### 替代方案讨论

社区建议的替代方案包括：

- **Codex**: 一些用户转向 OpenAI 的 Codex，认为它更少限制
- **开源模型**: 有用户表示开源模型已经足够好用，不再怀念 Opus、Sonnet 或 Haiku
- **本地 LLM 路由工具**: 如 Dario 等工具可以同时使用多个 LLM 提供商

## 结论

这条推文揭示了 AI 编码工具领域日益激烈的竞争态势。Anthropic 的这一做法被认为是"自杀式"的竞争策略，通过过度限制来保护市场份额而非通过产品优势吸引用户。社区普遍认为这种做法损害了用户信任，并开始考虑转向更开放的替代方案。
