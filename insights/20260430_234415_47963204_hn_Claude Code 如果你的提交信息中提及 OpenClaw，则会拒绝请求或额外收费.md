## 事件概述

Anthropic 公司的 Claude Code 被指控对包含 "OpenClaw" 字符串的提交拒绝服务或收取额外费用。OpenClaw 是一个开源的 AI 代理库。这一事件引发了开发者社区的广泛讨论和不满。

## 主要争议点

### 1. 指纹识别机制粗糙
- 用户报告称，只要在博客文章中提及 "OpenClaw"，Claude Code 就会拒绝响应
- 用户提供了 OpenClaw 的官网链接后，聊天立即结束并触发了5小时的使用限制
- 批评者指出，这种指纹识别方式过于简单粗糙，作为拥有极高系统权限的工具来说是不合适的

### 2. 涉及用户范围广泛
- 有用户在 NixOS Qemu 虚拟机中本地运行 OpenClaw，使用自己的配置沙盒
- 这些用户质疑：为什么 Claude Code 要阻止或对这些项目收取额外费用？
- 项目仓库中包含 Hermes 或 OpenClaw 的指令也会受到影响

### 3. 与"道德 AI 公司"形象不符
- 评论者将 Anthropic 与 OpenAI 进行比较，认为两者都在快速消耗自己的声誉
- 播客节目揭露了 Anthropic 声称的"道德"形象与其实际行为之间的差距
- 实际上 Anthropic 深度参与了美国军事项目，自2024年6月起获得机密权限

## 社区反应

### 批评观点
- 认为 Anthropic 的行为是"快速走向垃圾化"（enshittification）
- 对定价策略透明度不足表示失望
- 呼吁坦诚说明任何非按量计费的定价模式都是不可持续的

### 辩护观点
- 部分开发者认为对资源进行 QoS 限制是合理做法
- 认为应该为 OpenClaw 等工具设置单独的定价层级

## 技术细节

用户发现 Claude Code 的封禁机制似乎是简单的正则表达式匹配，而非更智能的识别方式。评论者建议至少应该对 Claude Code 本身进行白名单处理，避免误伤正常用户。

## 结论

这一事件反映了 AI 开发工具在资源管理和用户限制方面面临的挑战。开发者社区对这种粗糙的指纹识别和潜在的不公平定价策略表达了强烈不满，同时也开始关注开源模型和本地部署作为替代方案。
