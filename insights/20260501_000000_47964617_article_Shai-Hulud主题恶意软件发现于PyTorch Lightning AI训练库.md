# 洞察报告：Shai-Hulud 主题恶意软件发现于 PyTorch Lightning AI 训练库

**洞察链接**：https://news.ycombinator.com/item?id=47964617

**来源**：Hacker News

**发布日期**：2026年4月30日

---

## 事件概述

2026年4月30日，PyPI 包 `lightning` 在发布的 2.6.2 和 2.6.3 版本中遭到供应链攻击。恶意代码隐藏在一个名为 `_runtime` 的隐藏目录中，当该模块被导入时会自动执行。

## 攻击手法分析

### 恶意代码执行机制

恶意代码被嵌入在 `_runtime` 目录中，通过模块导入触发自动执行。这种技术在供应链攻击中较为隐蔽，普通用户难以察觉。

### 数据窃取目标

攻击者窃取的敏感信息包括：

- GitHub 访问令牌
- AWS 凭证
- Azure 密钥
- GCP（谷歌云平台）密钥
- 环境变量
- 认证令牌

### 数据泄露渠道

该攻击使用四个平行渠道进行数据外泄：

1. **HTTPS POST 到 C2 服务器**：直接向攻击者控制的命令控制服务器发送窃取的数据
2. **GitHub 提交搜索 dead-drop**：利用 GitHub 提交功能作为数据中转站
3. **攻击者控制的公开 GitHub 仓库**：将数据推送到攻击者自己的仓库
4. **直接推送到受害者仓库**：试图将恶意代码注入受害者的 GitHub 仓库

### 持久化机制

恶意代码通过以下方式进行持久化：

- Claude Code 的 hook
- VS Code 的 hook

这种双重持久化策略使得攻击者可以在多种开发环境中维持控制。

## 影响范围

- **受影响版本**：lightning 2.6.2 和 2.6.3
- **发布平台**：PyPI
- **攻击时间**：2026年4月30日

## 社区应对建议

1. 立即降级到 2.6.1 版本
2. 等待 2.6.4 版本发布后再进行升级
3. 检查是否使用了受影响的版本，并排查潜在的数据泄露
4. 轮换可能已被窃取的凭证和密钥

## 总结

这是一起典型的软件供应链攻击，攻击者通过入侵 PyPI 包来窃取开发者的敏感凭证。事件提醒我们：

- 开源依赖的安全性至关重要
- 自动化工具链可能成为攻击目标
- 定期审计和及时响应是防御供应链攻击的关键

社区建议所有使用 `lightning` 包的用户立即检查并采取补救措施。
