## 核心摘要

OpenAI 发布 GPT-5.4 mini 和 GPT-5.4 nano，这是迄今为止最强大的小型模型。它们将 GPT-5.4 的许多优势带到更快、更高效的模型中，专为高吞吐量工作负载而优化。

## 主要产品

### GPT-5.4 mini
- 在编码、推理、多模态理解和工具使用方面显著优于 GPT-5 mini
- 运行速度比 GPT-5 mini 快 **2倍以上**
- 在多项基准测试中接近 GPT-5.4 的性能
- 支持功能调用、文件搜索、计算机使用和技能

### GPT-5.4 nano
- GPT-5.4 系列中最小、最便宜的版本
- 适用于**分类、数据提取、排名**等任务
- 专为速度优先、成本敏感的场景设计
- 非常适合编码子代理

## 性能亮点

| 能力 | GPT-5.4 mini 表现 |
|------|-------------------|
| 编码能力 | SWE-bench Pro: 64.40% |
| 终端操作 | Terminal Bench 2.0: 65.70% |
| 多模态理解 | 强大的计算机使用能力 |
| 工具使用 | 接近旗舰模型表现 |

## 可用性

- **GPT-5.4 mini**: API、Codex、ChatGPT (现已上线)
  - API 支持 400k 上下文窗口
  - ChatGPT: Free 和 Go 用户可通过 "Thinking" 功能使用
  
- **GPT-5.4 nano**: 仅 API 可用

## 适用场景

1. **编码工作流**: 快速迭代开发
2. **代理系统**: 与 GPT-5.4 配合使用作为子代理
3. **多模态应用**: 实时图像推理
4. **高吞吐任务**: 分类、数据提取、排名
5. **成本敏感项目**: 需要在性能和成本间平衡

## 关键洞察

- 小型模型正在接近旗舰性能，为实际应用提供更优选择
- OpenAI 战略重点转向**智能代理(Agents)**工作负载
- 定价差异显著：nano 成本仅为旗舰的 **8-10%**
- 速度提升使这些模型适合延迟敏感型产品体验
