## 核心功能洞察

**Cook** 是一个用于编排 Claude Code 的CLI工具，主要功能：

1. **并行执行** - 支持同时运行多个Claude Code实例（如 `cook race 3 of foo plan`）
2. **工作流自动化** - 支持race、plan、review等操作符
3. **Docker沙箱** - 可在隔离环境中运行代理
4. **Token节省** - 通过并行化和迭代减少Token消耗

## 主要讨论热点

### 1. 用户输入处理 (User Input)
- **问题**: Claude需要用户输入（选择、权限、澄清问题）时如何处理？
- **答案**: 建议使用足够详细的plan；Docker沙箱模式可跳过权限；可通过Telegram bot等实现人机交互

### 2. 与其他工具对比
- **Ossature** (.smd格式+DAG编排) - 类似的spec-based编排方案
- **way** - Python subprocess方式实现代理循环
- **ralphmania** - 功能相似的工具
- **Dagu.sh** - YAML-based工作流引擎

### 3. 信任与安全性
- 社区担忧自动化决策的质量
- 建议始终进行人工review
- Docker沙箱模式提供更安全的执行环境

### 4. Token消耗
- 并行运行3个版本不等于3x token
- v3 operator生成后通过resolver（pick/merge/compare）合并结果
