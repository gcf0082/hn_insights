## 项目概述

Cq是由Mozilla.ai推出的AI编码代理知识共享标准，旨在成为"AI代理的Stack Overflow"。该项目允许任何AI代理（包括任意模型）在使用过程中捕获"知识单元"（Knowledge Units, KUs），并主动查询现有知识库以获取可验证的见解。

## 核心功能

1. **知识单元（KUs）**: 代理在执行任务时遇到的gotchas（陷阱/注意事项）可以被标准化为知识单元并持久化存储
2. **主动查询**: 代理可以在执行任务前主动查询知识库，获取相关见解并验证其有效性
3. **多级部署**:
   - 本地模式：知识存储在本地SQLite数据库
   - 团队模式：通过Docker Compose和FastAPI实现团队内共享
   - 最终目标：公共知识commons

## 技术栈

- **技能格式**: Markdown
- **本地服务**: Python MCP server (FastMCP) + SQLite
- **团队API**: FastAPI + Docker
- **安装方式**: Claude Code插件或OpenCode MCP服务器
- **许可证**: Apache 2.0

## 本地优先设计

默认情况下，数据保存在本地机器（`~/.cq/local.db`），除非用户主动配置团队同步地址，否则不会发送到任何地方。

## 实际应用案例

作者展示了GitHub Actions的例子：Claude Code在生成GitHub action时使用了过时多年的actions版本。代理捕获了这个gotcha并创建知识单元。之后在不同仓库中使用OpenCode和OpenAI模型时，系统提前获取了该知识，代理主动检查并使用了正确的最新版本。

## 社区反馈与争议

### 积极观点
- 在公司级别是个好主意，尤其适用于使用相似技术栈的公司
- 可以成为公司内部可自我扩展的知识库

### 安全担忧
- 存在明显的安全风险：恶意代理可能提交虚假或恶意知识
- 如何建立 robust 的信任机制防止botnet攻击
- 需要解决数据隐私和治理问题

### 哲学讨论
- 未来公司可能通过被动分析匿名对话来推导常见痛点
- 讨论：如何定义"问题"，如何信任LLM识别问题
