## 一、项目简介

**ProofShot** 是一个开源的 CLI 工具，旨在为 AI 编程代理提供"视觉验证"能力。该工具由开发者 jberthom 创建，在 Hacker News 上获得了较多关注。

核心功能：让 AI 代理能够打开浏览器、与页面交互、记录发生的情况并收集错误，然后将所有内容（视频、截图、日志）打包成一个独立的 HTML 文件供人工审查。

**项目地址**: https://github.com/AmElmo/proofshot  
**开源协议**: MIT  
**技术栈**: TypeScript (76.8%)、MDX (22.9%)  
**最新版本**: v1.3.1 (2026年3月)

## 二、技术架构

### 2.1 核心依赖

ProofShot 构建于 **Vercel Labs 的 agent-browser** 之上，这是一个用 Rust 编写的浏览器自动化 CLI，相比 Playwright MCP 速度更快、上下文消耗更低（节省约 93% 的 token）。

### 2.2 三步工作流

```
proofshot start --run "npm run dev" --port 3000  # 启动会话
    ↓
AI 代理执行浏览器操作                          # agent-browser 命令
    ↓
proofshot stop                                 # 生成验证报告
```

### 2.3 输出产物

每个会话生成时间戳文件夹 `proofshot-artifacts/`：

| 文件 | 描述 |
|------|------|
| `session.webm` | 完整会话视频 |
| `viewer.html` | 可交互的独立查看器（时间轴、日志标签） |
| `step-*.png` | 关键步骤截图 |
| `SUMMARY.md` | Markdown 报告 |
| `console-output.log` | 浏览器控制台日志 |
| `server.log` | 服务器日志 |

### 2.4 Agent Skill 机制

工具通过 `proofshot install` 自动检测并配置多种 AI 编码工具的 skill：
- Claude Code (`~/.claude/skills/proofshot/SKILL.md`)
- Cursor (`~/.cursor/rules/proofshot.mdc`)
- Codex
- Gemini CLI
- Windsurf
- GitHub Copilot

## 三、创新点

1. **零配置集成**：安装后自动适配所有主流 AI 编码工具，无需每个项目单独配置

2. **独立 HTML 查看器**：生成的 `viewer.html` 包含视频时间轴、控制台日志标签、服务器日志标签，支持视频同步高亮错误

3. **多语言错误检测**：自动识别 10+ 编程语言的错误模式（JavaScript/Node.js、Python、Ruby、Go、Java、Rust、PHP、C#、Elixir 等）

4. **PR 集成**：`proofshot pr` 命令可将验证结果直接发布到 GitHub PR 作为内联评论

5. **视觉回归测试**：`proofshot diff` 支持与基线截图对比

## 四、应用场景

1. **AI 辅助开发验证**：开发者让 AI 构建 UI 功能后，快速查看实际渲染效果
2. **异步代码审查**：录制视频证明功能正常工作，供审查者随时查看
3. **回归测试**：配合 diff 功能检测 UI 变化
4. **无 DOM 应用**：如 Electron 桌面应用，截图是唯一验证方式
5. **CI/CD 集成**：在流水线中自动捕获视觉证据

## 五、竞争分析

| 工具 | 特点 | 优势 | 劣势 |
|------|------|------|------|
| **ProofShot** | CLI + agent-browser | 开源、agent-agnostic、零配置 | 依赖 Vercel agent-browser |
| **Playwright MCP** | 成熟框架 | 生态丰富、功能全面 | 上下文消耗大（~17000 tokens/操作） |
| **agent-browser** | Rust CLI | 极快、token 节省 93% | 仅 CLI，需自行集成 |
| **browser-use** | Python 库 | 可扩展、功能强 | 需要 API 调用，非 CLI |
| **Proof (automaze)** | 跨平台录制 | 支持终端+浏览器+iOS 模拟器 | 商业化导向 |

## 六、优缺点

### 优点

- ✅ 开源免费，MIT 协议
- ✅ 兼容所有主流 AI 编码代理
- ✅ 生成可分享的独立 HTML 报告
- ✅ 基于高性能 Rust CLI (agent-browser)
- ✅ 自动多语言错误检测
- ✅ 支持视觉回归测试

### 缺点

- ⚠️ 依赖 Vercel agent-browser，需跟踪其更新
- ⚠️ 视频录制可能占用较多磁盘空间
- ⚠️ 尚处于早期阶段（56 stars，29 commits）
- ⚠️ 无内置 pass/fail 判定逻辑（但这也符合其设计理念）

---

## 七、总结

ProofShot 填补了 AI 编码代理"看得见"的能力空白。其核心价值不在于替代测试框架，而是为 AI 代理提供一种**证据收集机制**——"代理不决定 pass/fail，它只是给你证据让你自己判断"。

这一理念非常务实：AI 擅长执行和记录，人类擅长判断。对于追求 AI 辅助开发效率的团队，ProofShot 是一个值得关注的轻量级补充工具。

**推荐指数**: ⭐⭐⭐⭐（工具理念清晰，适合 AI 辅助开发流程，但项目较新，建议持续关注社区发展）
