## 项目概述

SentrySearch 是一个基于人工智能的语义搜索工具，专门用于在行车记录仪（dashcam） footage 中进行语义搜索。用户可以通过输入文字描述，快速找到对应的视频片段并自动裁剪出来。

## 核心功能

### 1. 视频语义搜索
- 用户输入自然语言查询（如"red truck running a stop sign"）
- 系统返回最匹配的视频片段
- 自动裁剪并保存匹配的视频

### 2. 视频分块处理
- 将视频分割为重叠的片段（默认30秒）
- 使用 Gemini Embedding 2 将视频直接嵌入到向量空间
- 支持自定义分块时长和重叠时间

### 3. 向量存储
- 使用 ChromaDB 本地向量数据库存储
- 768维向量空间，支持高效相似度匹配

### 4. 智能优化
- **预处理**：自动降采样到480p/5fps，减少上传大小
- **静止帧跳过**：自动跳过无画面变化的片段，节省API调用成本

## 技术架构

### 核心技术栈
| 组件 | 技术 |
|------|------|
| 向量嵌入 | Google Gemini Embedding 2 |
| 向量数据库 | ChromaDB |
| 视频处理 | ffmpeg / imageio-ffmpeg |
| 命令行工具 | Python Click |

### 工作原理
Gemini Embedding 2 原生支持视频嵌入——原始视频像素被投影到与文本查询相同的768维向量空间中。无需转录、无需帧标注，文本查询可以直接与视频片段进行向量级别比较。

## 使用方式

### 初始化
```bash
sentrysearch init
# 输入 Gemini API 密钥
```

### 索引视频
```bash
sentrysearch index /path/to/dashcam/footage
```

### 搜索
```bash
sentrysearch search "red truck running a stop sign"
```

### 可调参数
- `--chunk-duration`: 分块时长（秒）
- `--overlap`: 重叠时间（秒）
- `--target-resolution`: 目标分辨率
- `--target-fps`: 目标帧率
- `--no-skip-still`: 嵌入所有片段（包括静止帧）

### 成本优化建议
- 使用 `--chunk-duration 60 --overlap 10` 可减少API调用
- 启用静止帧跳过可显著降低成本
- 预处理功能主要提升速度而非降低成本

## 适用场景

- Tesla Sentry Mode 视频搜索
- 行车记录仪 footage 检索
- 任何 mp4 格式视频的语义搜索

## 局限性

1. **静止帧检测是启发式的**：基于JPEG文件大小比较，可能偶尔跳过有细微运动的片段
2. **搜索质量依赖分块边界**：跨两个分块的事件可能无法完美匹配
3. **Gemini Embedding 2 处于预览阶段**：API行为和定价可能变化

## 总结

SentrySearch 是一项创新的技术应用，利用 Gemini Embedding 2 的原生视频嵌入能力，实现了在行车记录仪 footage 上的实时语义搜索。该项目展示了多模态AI在实际场景中的应用潜力，特别适合需要从大量视频中快速定位特定事件的用户。
