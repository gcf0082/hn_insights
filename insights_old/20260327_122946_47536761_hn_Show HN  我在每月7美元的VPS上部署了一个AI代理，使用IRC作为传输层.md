## 项目概述

这是一个将 AI 智能体部署在每月 7 美元的 VPS 上并使用 IRC 作为传输层的创新项目。项目采用双智能体架构设计。

## 技术架构

### 双智能体设计

项目包含两个独立的智能体：

- **公开智能体 (nullclaw)**: 一个 678 KB 的 Zig 二进制程序，仅使用约 1 MB 内存，连接到 Ergo IRC 服务器。访客可通过嵌入在网站中的 gamja Web 客户端与之对话。
- **私有智能体 (ironclaw)**: 处理电子邮件和日程安排，仅可通过 Tailscale 网络访问，使用 Google 的 A2A 协议进行通信。

### 分层推理策略

项目采用了分层推理策略来优化成本：

- **Haiku 4.5**: 用于日常对话响应，速度快（亚秒级）、成本低廉
- **Sonnet 4.6**: 仅在需要使用工具时调用

每日 API 费用硬性上限为 2 美元。

### A2A 直通设计

私有端智能体借用网关自己的推理管道，这意味着无论谁发起请求，都只有一个 API 密钥和一个计费关系。

## 社区反馈

### 积极评价

- 用户认为这是一个很棒的想法，展示了 IRC 在实际应用中的潜力
- IRC 作为传输层的做法被认为具有创新性，房间即上下文（rooms-as-contexts）的模式被评价为被低估
- 该模式提供了免费的命名空间隔离，无需构建任何会话管理

### 成本优化讨论

社区成员提出了更经济的替代方案：

- **MiniMax M2.7**: 输入 tokens 0.30 美元/百万，输出 tokens 1.20 美元/百万
- **Kimi K2.5**: 输入 tokens 0.45 美元/百万，输出 tokens 2.20 美元/百万
- **Xiaomi Mimo v2-Flash**: 成本极低，0.3 cents 可完成复杂任务
- **Gemini 3.1 Flash Lite Preview**: 0.25 美元/百万输入 tokens

### 质疑与讨论

1. **关于使用 Claude API**: 有评论指出既然依赖 Claude API，并非真正"拥有整个技术栈"
2. **安全隔离测试**: 有人提议进行 CTF 测试，尝试访问私有智能体上的数据
3. **角色扮演问题**: 用户提到 nully（公开智能体）的态度有点居高临下
4. **IRC 消息长度限制**: 讨论了 IRC 协议本身的限制

## 技术亮点

1. **极低成本运行 AI 智能体**: 仅需 7 美元/月的 VPS 即可运行 AI 代理
2. **IRC 作为传输层**: 利用 IRC 的固有特性（如回溯功能）实现持久化对话
3. **客户端无关性**: 可从终端、移动 IRC 客户端或 Web 客户端连接同一个智能体
4. **分层推理**: 根据任务复杂度动态选择合适的模型
