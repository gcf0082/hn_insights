## 项目概述

这是一个 Claude Code 提示词优化项目，通过在项目根目录放置一个 `CLAUDE.md` 文件来控制 Claude 的输出行为，从而减少不必要的 tokens 消耗。

### 核心特性

1. **即插即用**：只需将 `CLAUDE.md` 放入项目根目录即可生效
2. **无需代码修改**：不影响现有代码逻辑
3. **模型无关**：规则基于上下文读取，理论上适用于任何模型
4. **可覆盖**：用户指令始终优先于文件规则

## 适用场景与局限性

### 最佳适用场景

- 高输出量的自动化管道（恢复机器人、代理循环、代码生成）
- 重复的结构化任务（跨数百次调用的冗余会累积）
- 需要一致、可解析输出格式的团队

### 不适用场景

- 单次短查询（文件加载开销大于节省）
- 偶发使用（低量时开销不划算）
- 深度失败模式修复（需要钩子、门禁、机械强制）
- 每次任务使用新会话的管道（新鲜会话不携带 CLAUDE.md 开销收益）
- 大规模解析可靠性（使用结构化输出更稳健）
- 探索性或架构性工作（辩论、反驳、 alternatives 是重点时）

## 社区建议

1. **针对实际失败模式定制规则**：通用规则如 "be concise" 有帮助，但真正收益来自针对实际遇到的特定失败。例如："当步骤失败时，立即停止并报告完整错误及回溯后再尝试修复。"

2. **利用 CLAUDE.md 组合**：
   - 全局文件 (~/.claude/CLAUDE.md)：保持通用偏好（语气、格式、ASCII 规则）
   - 项目级文件：保持项目特定约束
   - 子目录文件：保持任务特定规则

## 结论

`claude-token-efficient` 项目提供了一种简单有效的方式，通过提示词优化来减少 Claude 的输出 tokens 消耗。基准测试显示可实现约 63% 的输出 tokens 减少，但需要注意：

1. CLAUDE.md 文件本身会在每次消息时消耗输入 tokens
2. 只有当输出量足够高以抵消持久输入成本时，净节省才为正
3. 适合自动化、高频使用场景，不适合低频偶发使用

该项目是 Claude 社区对输出冗余问题的重要贡献，已获得 504 个星标和 23 个分支。
