## 内容摘要

该项目提供了一套 Claude 提示词配置，旨在将 Claude 的输出 token 减少 63%。核心规则包括：

- **答案优先**：答案始终在第一行，推理放在后面
- **禁止冗余**：不重复会话中已确立的信息
- **直接执行**：不要解释将要做什么，直接去做
- **无建议**：不主动提供建议，只做被要求的事
- **简洁输出**：移除 "Great question!" 等恭维语

## 社区讨论焦点

### 支持观点

1. **节省成本**：输出 token 价格为输入的 5-10 倍
2. **减少认知负担**：冗长输出增加阅读负担
3. **实用工具**：有人开发了 handoff、checkpoint 等会话交接系统

### 反对观点

1. **影响推理质量**：强制短响应可能损害推理和思维链
2. **训练目标冲突**：模型经过 RL 训练以在质量和长度间取得平衡
3. **基准偏差**：测试主要针对单次解释任务，而非代码生成等代理任务
4. **Transformers 本质**：自回归模型需要更多 token 来推理

### 中间观点

1. **区分高价值和低价值输出**：移除谄媚语和格式噪音，但保留有用推理
2. **输入成本占比更高**：实际数据显示 93.4% 是输入 token，4% 是输出
3. **提示缓存**：缓存可将输入成本降低 90%

## 技术细节

### 建议的替代方案

- **Headroom**：开源代理，压缩上下文，节省 34%
- **RTK**：Rust CLI 工具，压缩 shell 输出 60-90%
- **MemStack**：持久化记忆，避免重复读取代码库

### 会话交接系统

用户 sillysaurusx 开发了 `/handoff` 命令，在会话结束前生成永久性 Markdown 记录，包含：
- 完成的工作内容
- 关键决策
- 可供未来会话参考的上下文

## 结论

该项目的目标用户是处理大量数据、需要并行运行的用户。但对于实际的代理编码任务，社区普遍认为默认的详细输出模式更有利于保持长期一致性和推理质量。强制精简输出可能导致模型偏离训练分布，反而影响性能。
