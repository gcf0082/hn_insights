## 核心摘要

Cursor 3 是 Cursor 推出的全新统一工作空间，专门为构建软件代理而设计。该版本从零开始构建了以代理为中心的新界面，将开发者提升到更高的抽象层次，同时保留了深入探索的能力。新版本支持多仓库布局、本地和云端代理之间的无缝切换，并可随时切回传统 Cursor IDE。

---

## 主要更新内容

### 1. 所有代理集中管理

新界面本质上是多工作区的，允许人类和代理跨不同代码仓库协同工作。所有本地和云端代理都显示在侧边栏中，包括从移动端、Web端、桌面端、Slack、GitHub 和 Linear 启动的代理。

### 2. 并行运行多个代理

与代理协作变得更加简单。云端代理会生成演示和截图供用户验证，这与 cursor.com/agents 上的体验相同，现已集成到桌面应用中。

### 3. 本地和云端代理无缝切换

在云端和本地之间移动代理会话变得非常快速：
- **云端转本地**：当需要编辑和测试时，可以将代理会话从云端移到本地桌面。Composer 2（Cursor 自家的前沿编码模型，使用限额很高）非常适合快速迭代。
- **本地转云端**：可以将代理会话从本地移到云端，以便离线时保持运行，或切换到下一个任务。这对长时间运行的任务特别有用。

### 4. 从提交到合并 PR

新的差异视图允许使用更简单的 UI 更快地编辑和审查更改。准备好后，可以暂存、提交和管理 PR。

## 继承 Cursor 最佳特性

### 代码理解文件

可以随时深入查看文件，使用完整的 LSP 在编辑器中跳转到定义。

### 集成浏览器

Cursor 可以使用内置浏览器打开、导航本地网站并进行提示。

### Cursor Marketplace 插件

浏览数百种插件，通过 MCP、技能、子代理等扩展代理功能，一键安装，或设置团队私有插件市场。

---

## 展望未来

Cursor 3 已经奠定了基础——模型、产品和运行时——以构建更自主的代理和更好的团队协作。Cursor 还将继续投资 IDE，直至代码库实现自动驾驶。

这不会是软件构建界面最后一次改变。更强大的编码模型将解锁新的交互模式。Cursor 将继续构建、简化和转型，成为使用 AI 编程的最佳方式。

---

## 如何体验

升级 Cursor 后，输入 `Cmd+Shift+P -> Agents Window` 即可尝试新界面，或在[文档](https://cursor.com/docs/agent/agents-window)中了解更多。
