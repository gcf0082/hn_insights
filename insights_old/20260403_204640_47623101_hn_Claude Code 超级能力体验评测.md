## 内容摘要

本文作者 Evan Schwartz 对 Claude Code 的 Superpowers 插件进行了热情推荐。他与插件作者无任何关联，但认为使用 Superpowers 后 Claude Code 的工作效率大幅提升，生成代码的正确性也显著提高。

## 核心观点

### 问题与解决方案

作者之前使用 Claude Code 的主要痛点是：模型有时过于快速地跳到实现方案，而这些方案可能正确也可能完全错误。Plan 模式虽然有所改进，但在实际操作中存在 UX 问题——模型会生成冗长的计划文档让用户审核，修改后又是全新版本，体验不佳。

Superpowers 解决了这些问题，其完整工作流程为：**头脑风暴 → 评估选项与权衡 → 计划草稿 → 设计文档 → 实现计划 → 实施步骤**。

### 详细工作流程

1. **头脑风暴 (Brainstorming)**: Superpowers 几乎总是从头脑风暴模式开始，探索代码库、提出大量问题，并形成一些通用方向供用户选择。

2. **选项与权衡 (Options and Tradeoffs)**: 头脑风暴的关键环节是展示多个选项及其权衡利弊，帮助用户在深入细节前做出明智选择。最近还新增了视觉设计技能，可构建简单的 UI 改版模型并在本地启动开发服务器预览。

3. **计划草稿 (Plan Sketch)**: 确定方向后，Superpowers 给出高层次的计划描述（十余条要点或稍长的说明），比默认 Plan 模式更容易审阅和反馈。

4. **设计文档 (Design Doc)**: 批准高层次计划后才会编写完整设计文档，此时用户已认同大方向，审阅和评论更加高效。

5. **实现计划与实施 (Implementation Plan and Steps)**: 批准设计文档后，Superpowers 编写实现计划并用子代理审查，之后启动子代理实施计划各部分，并自动审查其工作是否符合实现计划。

## 社区反馈要点

- **UX 改进需求**: 有用户指出 Claude Code 原生 Plan 模式过于线性僵化，缺少"部分同意/部分修改"的灵活选项。
- **工具对比**: 有用户推荐 Plannotator 工具，可在网页上高亮和添加注释后传回 Claude Code。也有用户将其与 gsd-2 进行对比。
- **使用体验**: 部分用户认为 Superpowers 对非资深开发管理者更有价值，资深开发者可能更喜欢直接使用原生 Claude Code。
