# 使用 OpenCode 构建 100 个 API 集成的经验总结

## 基本信息

- **原始链接**: https://news.ycombinator.com/item?id=47579818
- **文章链接**: https://nango.dev/blog/learned-building-200-api-integrations-with-opencode/
- **Hacker News 评分**: 71 分
- **评论数**: 17 条
- **作者**: rguldener

## 核心观点

### AI 驱动 API 集成的价值

文章探讨了 Nango 团队使用 OpenCode AI 智能体构建 100 个 API 集成的经验。关键发现是：AI 可以读取 API 文档、组装请求、迭代调试直到正常工作。虽然这听起来简单，但对人类来说既耗时又容易出错。

### 实验方法

团队选择了"少限制、多学习"的策略，而非一开始就设置大量护栏。他们希望通过实验了解模型的实际行为，而不是过度拟合自己的假设。

### 成本效益

文章声称 AI 能在一周内完成约一周的开发工作，成本仅需 20 美元 token 费用。

## 社区讨论焦点

### 质疑观点

1. **重复造轮子问题**: 有评论指出，每次为不同客户重新生成代码而非使用现成库的价值在哪里？
2. **开源版本限制**: Nango 声称完全开源，但自托管版本似乎只是功能子集，引发对宣传真实性的质疑。
3. **效率问题**: 有观点认为用 AI 生成一切是浪费 token 的行为，更好的做法是用 OpenAPI/Swagger 规范生成代码生成器。

### 反驳观点

支持者认为，实验的意义在于探索 AI 智能体的边界，而非 immediate 优化。用 20 美元和几分钟就能取得很大进展，这本身就是有价值的证据。

### 关键矛盾

TL;DR 与文章内容存在不匹配：文章声称 AI 可靠地完成一周开发工作，但随后列出了各种故障模式和调试工作，最后却结论是"智能体尚未准备好端到端自主完成所有集成"。

## 总结

这是一篇关于 AI 辅助 API 集成实践的经验分享，反映了当前 AI 编程助手的实际能力和局限性。社区讨论围绕 AI 生成代码的质量、可维护性以及与传统开发方式的对比展开。
