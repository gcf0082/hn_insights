## 核心观点

本文将2008年金融危机前的次贷危机与当前AI行业进行类比，指出AI行业正面临类似的"次贷AI危机"。作者认为，整个AI行业建立在不可持续的经济模式之上，通过大量补贴吸引用户，但最终将被迫涨价或削减服务，导致用户流失和行业崩溃。

## 次贷危机历史回顾

### 2008年金融危机的原因

1. **宽松信贷环境**：次贷机构发放了约190万份次贷贷款，其中许多是可调利率抵押贷款（ARM）
2. **误导性营销**：贷款机构忽视借款人的实际还款能力，以低利率吸引客户
3. **价格泡沫**：房价基于不可持续的抵押贷款被虚假抬高
4. **监管缺失**：监管机构对泡沫视而不见

### 关键教训

- 危机并非仅由低收入借款人造成，实际上是中高等收入群体也被低利率诱惑
- 每个人都参与了这场"游戏"，信用在最热门房产市场扩张最严重
- 问题的根源是价值幻觉和信贷易得性

## AI行业的"次贷"模式

### 资金来源

1. **数据中心**：从银行、私募股权或"商业发展公司"举债建设
2. **AI实验室**：从风险投资、超级科技公司、主权财富基金获得资金
3. **关键金主**：Blue Owl、MUFG、高盛、摩根大通、SMBC等

### 经济链条

```
NVIDIA（硬件销售）→ 数据中心开发商 → 超级科技公司（Google、Meta、Amazon、Microsoft）→ AI实验室（OpenAI、Anthropic）→ AI初创公司 → 消费者
```

### 补贴真相

- **NVIDIA是唯一真正盈利的环节**
- 数据中心几乎完全靠债务融资（约4200万美元/兆瓦）
- CoreWeave（最大GPU提供商）运营利润率为-6%，净亏损率达-29%
- Anthropic：营收50亿美元，计算成本100亿美元
- OpenAI：截至2025年9月，营收43亿美元，仅推理成本就达86.7亿美元

## "次贷AI危机"的触发点

### 2025年6月起的变化

1. **OpenAI和Anthropic推出"优先服务层级"**
   - 涨价要求3-12个月的前置token吞吐量保证
   - AI初创公司成本大幅增加

2. **具体影响案例**
   - **Cursor**：从按请求收费改为"模型价格+20%手续费"模式
   - **Replit**：转向"工作量定价"，用户订阅20美元/月，实际燃烧至少25美元
   - **Augment Code**：改为按消息计费，有用户在同一计划下花费15,000美元token
   - **Perplexity**：激进削减速率限制，20美元/月的订阅从300+搜索/天降至每周"平均使用限制"

3. **用户习惯养成问题**
   - 用户习惯于"无限"或"部分无限"的服务
   - 订阅费用与实际token消耗严重脱节
   - Anthropic和OpenAI每收取1美元，用户可燃烧3-13美元的算力

### 即将到来的后果

- AI实验室被迫大幅削减用户的token使用量
- 用户愤怒离去
- AI初创公司死亡（无法承受成本）
- AI实验室自身也面临计算资源预订过剩的问题
- 数据中心建设停摆，债权人蒙受损失

## 与次贷危机的类比

| 次贷危机 | 次贷AI危机 |
|---------|-----------|
| 宽松信贷→房价泡沫 | 风险投资补贴→用户增长泡沫 |
| 可调利率抵押贷款（ARM） | 更消耗token的新型模型 |
| 贷款机构赚取手续费 | AI实验室收取API费用 |
| 最终利率上升导致违约 | 成本上升导致涨价和限制 |
| 房价下跌→资不抵债 | 用户流失→收入下降→无法盈利 |

## 结论

作者认为AI行业与次贷危机的相似之处在于：

1. **基于不可持续的经济模式**：一切都建立在"未来会更便宜"或"用户会付更多钱"的假设上
2. **价值幻觉**：AI初创公司的估值依赖于被收购或上市的预期，而非实际盈利能力
3. **短期思维**：为吸引用户而提供巨额补贴，但这些补贴最终无法持续
4. **监管缺失**：没有人愿意承认问题的严重性

"次贷AI危机"已经开始，当最大的玩家被迫面对其腐烂的经济现实时，整个链条的下游后果将随之而来。
