## 核心洞察

### 1. 高层架构

整个测试流程包含四个关键步骤：

1. **教程脚本**：从包含命令块的 `tutorial.sh` 脚本开始
2. **pytest 转换**：为每个命令块生成一个或多个 pytest 测试函数
3. **代理执行**：为每个测试函数启动一个代理来运行、调试、修复和改进
4. **结果整合**：将所有代理的结果整合在一起

### 2. 教程脚本编写

Imbue 团队采用了一种巧妙的"人机协作"方式：
- 编写注释如 `# Managing snapshots`
- 让编码代理填充具体示例
- 人工审核保留优质示例

这种方法的额外收益：如果代理生成示例质量差，说明接口设计不够清晰，可以此改进 mngr 本身。

### 3. 教程块到 pytest 函数的转换

关键挑战：
- **一对一映射不足**：一个教程块可能需要多个测试用例，覆盖成功和失败路径
- **保持关联**：要求代理在生成的测试函数中声明对应的教程块
- **验证机制**：使用脚本检查每个教程块是否至少有对应测试

端到端测试的三大困境（对代理和人类同样适用）：
- **Arrange**：设置尽可能少的依赖，但需要足够的隔离性
- **Act**：忠实还原实际命令，但需要适当调整以适应测试
- **Assert**：尽可能精确测试效果，但过于具体会导致脆弱或 flaky 测试

### 4. 测试框架

Imbue 在 Python `subprocess` 模块基础上构建了薄层抽象，提供：
- `e2e.write_tutorial_block()`：记录教程块
- `e2e.run()`：运行命令并获取结果
- `expect(result).to_succeed()`：断言成功
- `expect(result.stdout).to_contain()`：断言输出包含特定内容

额外功能：
- 生成 CLI 转录本
- 使用 asciinema 录制 tmux 会话
- 网页端统一查看所有产物

### 5. 测试编排

使用 mngr 原语实现自动化流程：

1. **收集测试**：`pytest --collect-only`
2. **启动测试代理**：使用 `mngr create`
3. **状态轮询**：使用 `mngr list`
4. **拉取结果**：使用 `mngr pull`
5. **停止代理**：使用 `mngr stop`
6. **整合变化**：创建"整合者"代理合并所有 PR

### 6. 变更整合策略

- 将变更分为"实现修复"和"非实现修复"（如测试改进、文档修复）
- 非实现修复通常直接合并
- 实现修复按重要性排序，保持独立提交，最后合并为单一线性分支
- 无法整合的代理标记为"blocked"状态

### 7. 本地到远程的无缝扩展

- **本地起步**：先在本地机器构建工作流，快速迭代
- **远程扩展**：通过简单修改 `mngr create foo` 为 `mngr create foo@.modal`，即可将代理部署到 Modal 运行
- **一致性**：本地和远程的交互方式完全一致，`mngr list`、`mngr pull`、`mngr stop` 等命令行为相同

## 软件可组合性

Imbue 构建的本质上是一个小型 MapReduce 管道：
1. **Map**：收集测试列表 → 每个测试分配一个代理
2. **Reduce**：将结果合并为单一分支

关键洞察：mngr 本身不了解 MapReduce，它只是提供了"maps"和"reduces"这样的函数式编程原语，让用户自由构建简单的或复杂的多代理管道。

---

## 软件可扩展性

### 向上扩展（Scale Up）
- 从本地 10 个代理平滑扩展到数百个 Modal sandbox
- 增量成本低

### 向下扩展（Scale Down）
- 同样可以轻松缩回本地机器
- 受限于 RAM，但可以夜间运行数百代理（以时间换空间）

### 核心观点
业界通常关注"向上扩展"和"增量成本"，但往往忽视"向下扩展"和"前置成本"。对于小团队：
- 快速启动、本地运行、无需大规模前期投入 → 更具价值
- mngr 让这种灵活扩展成为可能

## 总结

本文展示了 Imbue 如何利用 mngr 工具进行自我改进的测试，核心经验包括：
1. **自动化测试生成**：从教程脚本自动生成 pytest 测试
2. **并行代理执行**：利用百个代理并行测试和修复
3. **无缝扩展**：本地到远程的平滑过渡
4. **可组合原语**：提供基础原语而非固定框架，赋予用户最大灵活性

这代表了 AI 辅助软件开发的新范式：不是提供封闭的解决方案，而是提供可组合的工具，让开发者构建适合自身需求的自动化工作流。
