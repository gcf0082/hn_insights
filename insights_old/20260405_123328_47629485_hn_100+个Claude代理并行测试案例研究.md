## 内容摘要

本文是Imbue公司关于其`mngr`工具的案例研究，展示了如何利用100多个Claude代理并行运行来测试和改进该工具自身。

### 核心架构

整个流程包含以下步骤：
1. **教程脚本编写**: 从`tutorial.sh`脚本开始，包含命令块
2. **转换为pytest函数**: 将每个命令块转换为一个或多个测试函数
3. **代理执行**: 为每个测试启动一个代理来运行、调试、修复和改进
4. **结果整合**: 将所有代理的结果合并到一个PR中

### 技术要点

- **测试框架**: 基于Python的`subprocess`模块，构建了轻量层来处理CLI命令执行
- **远程执行**: 使用Modal平台运行远程代理，本地只需更改`mngr create foo`为`mngr create foo@.modal`
- **Git集成**: 本地使用Git worktree模式，远程使用Git mirror模式

### 软件可组合性

本质上构建了一个小型MapReduce管道：
- Map: 对每个测试启动一个代理
- Reduce: 将所有结果合并到单个分支

值得注意的是，`mngr`本身不了解MapReduce，而是通过其提供的原语来实现这个管道。

### 软件可扩展性

- **向上扩展**: 从本地10个代理扩展到远程数百个代理
- **向下扩展**: 可以轻松回退到本地机器运行
- **低成本**: 增量成本低，前置成本也低

## 评论区观点

1. **产品定位**: 有评论指出这是推销代理编排产品和服务的软文。

2. **成本考量**: 
   - Token预算的主要消耗不在代理数量本身，而在于理解orchestrating的成本
   - 每个代理运行需要20-50k tokens用于上下文（仓库结构、相关文件、近期更改）
   - 100个代理每小时运行，跨越10-20个仓库，每天已达数百万tokens

3. **可观测性挑战**:
   - 单个代理可以读日志理解问题，100个代理需要聚合、模式检测
   - 如果3个代理静默失败，是真实问题还是限速？
   - 如果40个代理同时超时，是依赖问题还是基础设施饱和？

4. **并发策略**: 异步模式不是"尽可能多地运行"，而是"运行API和预算支持的精确数量"。

5. **知识产权讨论**: 
   - 讨论AI生成内容是否可版权化
   - 有评论指出使用AI提供商时没有秘密可言，因为提供商跟踪所有交互
   - 长期来看，Anthropic可能成为竞争对手

## 总结

本文展示了Imbue如何利用其`mngr`工具进行大规模代理测试的实践。通过将教程脚本转换为测试用例，然后并行运行代理来执行和改进测试，实现了软件开发的自动化循环。关键启示包括：构建可组合的工作流原语、保持前后端成本低廉、以及在大规模和局部之间灵活切换的能力。
