## 项目概述

GuppyLM 是一个小型语言模型,专门模拟一条名为 Guppy 的小鱼说话方式。该项目的核心理念是展示训练自己的语言模型并非高不可攀,无需博士学位,也不需要大规模的GPU集群。

## 技术架构

| 参数项 | 数值 |
|--------|------|
| 参数量 | 8.7M |
| 层数 | 6 |
| 隐藏维度 | 384 |
| 注意力头数 | 6 |
| FFN维度 | 768 (ReLU) |
| 词表大小 | 4,096 (BPE) |
| 最大序列长度 | 128 tokens |

采用标准Transformer架构,没有使用GQA、RoPE、SwiGLU等优化技术,保持最简单的实现方式。

## 训练数据

- **数据集**: arman-bd/guppylm-60k-generic
- **规模**: 60,000条对话 (57K训练/3K测试)
- **话题数**: 60个话题
- **生成方式**: 合成模板组合

训练话题涵盖:问候、感受、温度、食物、光线、水、噪音、孤独、气泡、反射、游泳、颜色、味道、 plants、 algae、 snails、恐惧、兴奋、无聊、好奇、快乐、疲倦、猫咪、雨、季节、音乐、访客、儿童、生命意义、时间、记忆、梦境、未来、过去、名字、天气、睡觉、朋友、笑话、恐惧、爱、年龄、智慧、健康、唱歌、电视等。

## 模型特点

Guppy (角色设定):
- 说短小、低沉的句子
- 通过水、温度、光线、振动和食物来感受世界
- 不理解人类抽象概念
- 友好、好奇、有点傻
- 经常想食物

## 设计决策

1. **无系统提示词**: 9M模型无法条件遵循指令, personality已固化在权重中
2. **单轮对话**: 128-token上下文窗口限制了多轮对话能力
3. **合成数据**: 保证角色性格一致性
4. **简化架构**: 在小参数规模下,标准attention+ReLU FFN表现相同

## 快速开始

### 聊天(无需训练)
直接使用预训练模型,通过Colab运行。

### 本地运行
```bash
pip install torch tokenizers
python -m guppylm chat
```
