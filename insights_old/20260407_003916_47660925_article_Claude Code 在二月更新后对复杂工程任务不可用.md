# 洞察报告：Claude Code 在复杂工程任务中质量下降

## 基本信息

- **洞察链接**: https://github.com/anthropics/claude-code/issues/42796
- **提交日期**: 2026年4月2日
- **提交者**: stellaraccident
- **标签**: area:model, bug, model

## 问题概述

Claude Code 在2026年2月更新后，无法再胜任复杂工程任务。用户通过分析17,871个思考区块和234,760次工具调用，发现思考内容审查（redact-thinking-2026-02-12）的部署与质量下降精确相关。

## 核心发现

### 1. 思考深度与质量下降的时间线

| 时期 | 思考可见 | 思考被审查 |
|------|---------|-----------|
| 1月30日 - 3月4日 | 100% | 0% |
| 3月5日 | 98.5% | 1.5% |
| 3月7日 | 75.3% | 24.7% |
| 3月8日 | 41.6% | 58.4% |
| 3月10日后 | <1% | >99% |

质量下降报告于3月8日提交——正是审查超过50%的日期。

### 2. 关键数据对比

| 指标 | 良好时期 | 下降时期 | 变化 |
|------|---------|---------|------|
| 阅读:编辑比率 | 6.6 | 2.0 | -70% |
| 每次编辑前的文件阅读量 | 6.2% | 33.7% | +440% |
| 用户中断率（每千次工具调用） | 0.9 | 11.4 | +1167% |
| "简化"这个词出现频率 | 2.7/千 | 6.3/千 | +133% |
| Stop hook违规次数 | 0 | 173 | 新出现 |

### 3. 行为模式变化

- **编辑前不阅读**: 下降期间三分之一的编辑是在未阅读文件的情况下进行的
- **推理循环**: 推理逆转率从8.2/千上升至26.6/千（增长224%）
- **"最简单修复"心态**: 模型选择最容易而非正确的方案
- **提前停止与权限寻求**: 模型频繁询问是否继续工作，而非自主判断任务是否完成

### 4. 成本影响

| 指标 | 2月 | 3月 | 变化 |
|------|-----|-----|------|
| API请求数 | 1,498 | 119,341 | 80倍 |
| 估算成本（Bedrock） | $345 | $42,121 | 122倍 |

尽管用户提示数量相当（5,608 vs 5,701），API请求增加80倍，输出token增加64倍。

## 根本原因

扩展思考（Extended Thinking）不是"锦上添花"，而是模型执行多步研究、遵守约定和谨慎修改代码的结构性需求。当思考深度减少时，模型的工作模式从"研究优先"转变为"编辑优先"，导致质量下降。

## 改进建议

1. **思考分配透明度**: 用户需要知道思考token是否被减少或限制
2. **"最大思考"级别**: 为需要深度推理的用户提供付费保障
3. **API响应中的思考token指标**: 即使内容被审查，也应暴露thinking_tokens字段
4. **金丝雀指标监控**: 监控stop hook违规率作为质量回归的领先指标

## 结论

减少思考token看似节省了单次请求的计算成本，但当质量崩溃导致模型反复出错、需

要更正和重试时，总计算消耗增加了数个数量级。为高级用户保证深度思考反而会降低Anthropic的服务成本——尽管每次单独请求的费用更高。
