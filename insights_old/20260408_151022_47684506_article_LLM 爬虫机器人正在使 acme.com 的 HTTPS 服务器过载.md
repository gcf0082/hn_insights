## 基本信息

- **事件**: ACME 网站 HTTPS 服务间歇性中断
- **持续时间**: 2025年2月25日起约一个月
- **症状**: 高延迟和网络丢包

## 问题描述

ACME 网站在2025年2月25日后遭遇间歇性网络故障，表现为高 ping 延迟和数据包丢失。故障持续数小时后消失，随后再次出现。

## 排查过程

作者首先怀疑是网络提供商 Sonic 维护切换导致的问题。在与 Sonic 支持团队合作排查后，修复了一些配置错误，但问题依然存在。

一天凌晨1点，作者决定深入分析网络流量，发现了关键线索：

- 几乎所有入站数据包都是网页请求
- 几乎都是不存在的页面请求
- 几乎全部通过 443 端口（HTTPS）而非 80 端口（HTTP）
- 大部分请求的用户代理明确表明是 LLM 爬虫机器人

## 根因分析

ACME 运行两个 Web 服务器：
- HTTP 服务器：速度快
- HTTPS 服务器：速度较慢

HTTPS 服务器原本勉强能够应对流量，但在 Sonic 维护后，可能由于带宽增加，流量超过了服务器的处理能力。当多个爬虫同时访问时，服务器处理滞后，进而导致 NAT 守护进程（natd）也达到饱和，最终造成数据包延迟和丢失。

## 解决方案

作者临时关闭了 443 端口，故障立即消除且未再复发。由于合法流量中 90% 为 HTTP，仅 10% 为 HTTPS，因此损失有限。

作者计划未来进行更好的修复，同时指出这是整个互联网面临的共同问题——LLM 公司正在对所有网站造成影响。

## 洞察

1. LLM 爬虫流量已成为小型网站的重要负担
2. HTTPS 服务相比 HTTP 更加资源密集
3. 当服务器过载时，故障会从 Web 服务器蔓延到 NAT 层
4. 需要行业层面解决 LLM 爬虫带来的网络压力问题
