## 核心内容

Anthropic 发布了 **Claude Managed Agents**，这是一项允许用户定义成果和成功标准，然后由 Claude 自我评估和迭代直到达成目标的服务（目前处于研究预览阶段）。该服务还支持传统的提示词-响应工作流模式，以便用户需要更紧密的控制。

## 主要观点

### 1. 代理框架的早期阶段
社区普遍认为我们正处于代理框架的早期阶段，类似于 PHP 出现之前的 web 开发时代。每周都有全新的模式和方法被重新发明，包括 evals、ReACT、DSPy、RLM、记忆模式、动态上下文、沙箱策略等。锁定某个框架可能会让试图保持竞争力的开发者陷入被动。

### 2. 供应商锁定担忧
**核心担忧**是被锁定在单一模型提供商（尤其是 Anthropic）是一个 Deal Breaker。许多开发者倾向于能够比较不同的模型，并根据任务类型（前沿模型用于设计和规划，自托管模型用于简单的操作任务）做出选择。有评论指出依赖 Anthropic 的方式处理复杂事务是"自杀式"的做法。

### 3. Anthropic 的战略定位
Anthropic 正在从纯模型 API 提供商向更复杂的部署产品转型。有评论将其比作"Claude Code 的网页版"，认为这是向平台化发展的重要一步。Anthropic 需要成为平台而非单纯的 token 管道来实现大规模 IPO。

### 4. AaaS (Agent as a Service) 商业模式
有评论指出 **2026 年的下一个 100B 商业模式是 AaaS**。这可能会对现有的 AI/Agent SaaS 公司造成冲击，传统 SaaS 可能会被代理服务简化甚至取代。

### 5. 对现有 SaaS 的影响
关于 Anthropic/OpenAI 是否会抢占现有 AI/Agent SaaS 的市场份额，社区存在分歧：
- **乐观方**：认为 SaaS 价格低廉（$20-$100/月），对于软件工程师来说不到 1 小时的工资，不会被替代
- **悲观方**：认为随着模型能力提升，代理服务将变得更简单高效，可能会取代大部分 SaaS

### 6. 与现有方案的比较
有评论指出 LangChain 的 Smith 平台早已提供类似功能，Anthropic 并非第一家。AWS 和 GCP 等传统云平台可能会被代理服务"从第一性原理重新设计"。
