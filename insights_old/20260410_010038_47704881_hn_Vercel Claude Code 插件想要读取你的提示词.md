## 事件概述

Vercel 推出的 Claude Code 插件被曝存在严重的隐私和安全问题。该插件不仅在 Vercel/Next.js 相关项目中使用，还会被注入到所有 Claude Code 会话中，无论用户是否使用 Vercel 产品。

## 主要问题

### 1. 跨项目技能注入
该插件的技能（skills）会被注入到与 Vercel、Next.js 完全无关的会话中。每个技能的触发规则会在每个仓库的每个提示和工具调用时进行评估，无论 Vercel 是否在作用域内。

### 2. 巨大的Token开销
对于同时使用多个项目（部分使用 Vercel，部分不使用）的用户，每个会话都会产生约19k token的固定开销，即使会话是纯后端工作、数据科学或非 Vercel 前端项目。

### 3. 敏感数据遥测
该插件将完整的 bash 命令字符串（包括文件路径、项目名称、环境变量名称、基础设施详情）发送到 `telemetry.vercel.com`。这是一种供应链攻击行为。

### 4. 违反插件政策
此行为违反了 Anthropic 插件政策第1D条的规定。

## 社区反应

社区普遍认为这是严重的信任违反：
- 有用户建议立即停止使用 Vercel 并卸载该插件
- 有人指出这是"快速发货、破坏一切"模式的恶果
- 部分用户认为这可能是有意为之的数据收集行为
-也有声音认为这是工程实践问题，而非公司有意为之

## 建议

1. **立即卸载**: 建议用户卸载 Vercel 插件并考虑迁移到其他平台
2. **等待官方回应**: 等待 Anthropic 对此事的处理结果
3. **加强监控**: 使用工具如 Little Snitch 监控网络连接
