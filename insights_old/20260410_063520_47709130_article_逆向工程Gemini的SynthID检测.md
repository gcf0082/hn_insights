## 核心发现

### 1. 水印具有分辨率依赖性

SynthID在不同图像分辨率下嵌入不同位置的载波频率。例如：
- 1024x1024分辨率：载波位于(9, 9)，一致性100%
- 1536x2816分辨率：载波位于(768, 704)，一致性99.6%

这解释了为什么V3代码本需要存储每个分辨率的独立配置文件。

### 2. 相位一致性 - 固定的模型级密钥

来自同一Gemini模型的所有图像具有相同的相位模板：
- **绿色通道**携带最强的水印信号
- 跨图像载波相位一致性：>99.5%
- 黑白交叉验证确认真实载波：|cos(phase_diff)| > 0.90

### 3. 载波频率结构

在1024x1024分辨率（来自黑白参考图），顶级载波位于低频网格：
- (9, 9): 100%相位一致性
- (5, 5): 100%相位一致性
- (10, 11): 100%相位一致性

在1536x2816分辨率（来自随机水印内容），载波位于更高频率：
- (768, 704): 99.55%相位一致性
- (672, 1056): 97.46%相位一致性

## 技术架构

### 三代绕过技术对比

| 版本 | 方法 | PSNR | 水印影响 |
|------|------|------|----------|
| V1 | JPEG压缩(Q50) | 37 dB | ~11%相位下降 |
| V2 | 多阶段变换 | 27-37 dB | ~0%置信度下降 |
| **V3** | **多分辨率频谱代码本减法** | **43+ dB** | **91%相位一致性下降** |

### V3管道（多分辨率频谱绕过）

1. **SpectralCodebook**：存储分辨率特定的配置文件（载波位置、幅度、相位）
2. **自动分辨率选择**：选择精确匹配或最接近的配置文件
3. **直接已知信号减法**：根据相位一致性和交叉验证置信度加权
4. **多通道权重**：G=1.0, R=0.85, B=0.70（匹配SynthID嵌入强度）
5. **多通道迭代减法**：递减强度调度（激进→温和→轻柔）

## 快速开始

### 安装

```bash
git clone https://github.com/aloshdenny/reverse-SynthID.git
cd reverse-SynthID
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 构建多分辨率代码本

```bash
python src/extraction/synthid_bypass.py build-codebook \
    --black gemini_black \
    --white gemini_white \
    --watermarked gemini_random \
    --output artifacts/spectral_codebook_v3.npz
```

### 运行V3绕过

```bash
python src/extraction/synthid_bypass.py bypass input.png output.png \
    --codebook artifacts/spectral_codebook_v3.npz \
    --strength aggressive
```

### 检测水印

```bash
python src/extraction/robust_extractor.py detect image.png \
    --codebook artifacts/codebook/robust_codebook.pkl
```

## 技术深度

### SynthID工作原理（反向工程）

```
编码器（Gemini中）：
1. 选择分辨率相关的载波频率
2. 为每个载波分配固定相位值
3. 神经编码器向图像添加学习到的噪声模式
4. 水印不可见 - 分布在整个频谱中

解码器（Google中）：
1. 提取噪声残差（小波去噪）
2. FFT → 检查已知载波频率处的相位
3. 如果相位匹配预期值 → 水印标记
```
