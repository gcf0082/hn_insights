## 摘要

本文讨论了 AI 代理开发中 MCP（Model Context Protocol）与 Skills 两种方案的优劣对比引发了开发者社区的广泛讨论。

## 核心观点

### MCP 优势论

- **标准化接口**: MCP 提供了统一的工具调用规范，便于跨平台使用
- **企业友好**: 在受限环境中（如企业聊天机器人、移动端）无法运行 CLI 时，MCP 仍有优势
- **安全性**: 可在代理运行前启动服务器，避免密钥暴露在环境变量中
- **渐进式发现**: 现代应用（ChatGPT、Claude 等）支持按需加载工具，节省上下文窗口

### Skills/CLI 优势论

- **本地优先**: 开发者更倾向于使用本地 CLI 工具，无需额外配置
- **组合性更强**: CLI 工具可以像 Unix 管道一样组合使用，LLM 可灵活调用
- **简洁性**: MCP 增加了不必要的中间层，CLI 本身已是高效的 API
- **成本效益**: 无需维护远程服务器，降低使用门槛

### 综合观点

多数开发者认为 MCP 与 Skills 是互补关系，适用于不同场景：
- **本地工具** → 使用 Skills
- **远程服务** → 使用 MCP

## 技术细节

- MCP 本质上是一种 JSON-RPC 协议
- Skills 可包含脚本代码，支持 API 调用
- 远程 MCP 可通过工具如 `murl` 调用本地 CLI
- 渐进式披露（Progressive Disclosure）对两者都很重要

## 结论

两者并非零和博弈。Anthropic 官方也指出 Skills 和 MCP 是互补的。选择取决于具体使用场景：本地开发优先 CLI/Skills，企业受限环境优先 MCP。
