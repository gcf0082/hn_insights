## 项目背景

MemPalace 是由女演员米拉·乔沃维奇（Milla Jovovich）主导开发的 AI 记忆系统。该项目在多个基准测试中取得了最高分，一度引发广泛关注。然而，随着讨论深入，项目基准测试结果的真实性受到了质疑。

## 讨论要点

### 1. 基准测试争议

多位评论者指出项目基准测试存在问题：

- **LongMemEval 基准**：声称获得 100% 分数，但实际上是对特定失败的 3 个问题进行了针对性修复，加上 LLM 重排序。如果没有这些修复，实际得分仅为 98.4%
- **LoCoMo 基准**：100% 分数使用了 top-k=50，这实际上超过了会话数量。使用诚实的 top-10 且无重排序仅获得 88.9%
- 有人将此与 "OpenCLAW" 事件相比，指出这是"工程化基准测试结果以使其看起来完美，然后作为突破进行营销"

### 2. 实际价值

尽管存在争议，部分评论者认为项目仍有实际价值：

- 便捷的 ChromaDB 导入管道，支持 Claude Code、ChatGPT 和 Slack 导出
- 语义搜索功能可跨越数月或数年的对话历史
- 完全本地化、MIT 许可、无需 API 密钥
- 独立的时间知识图谱模块可独立使用

### 3. 不同技术路线

讨论还涉及 AI 记忆系统的不同设计思路：

- **存储一切**：MemPalace 的方式是保留所有内容，确保可检索性
- **合成路线**：Lodestar 等项目选择在会话结束时由 LLM 合成摘要，存储在代码库而非本地向量数据库

### 4. 社区反应

- 有人制作了相关 meme，如 "Missed the chance to call this Resident Eval"
- 也有人戏称其为 "Leeloo Dallas Memory Palace"（出自《第五元素》）
- 还有人提到 Multi-Pass 的梗

## 结论

MemPalace 项目展示了 AI 记忆系统的一个有趣方向，但其宣称的"史上最高分"需要谨慎看待。基准测试结果显示该项目在技术上可能有一定价值，但营销手法存在夸大成分。社区普遍认为，即使基准测试存在争议，该项目仍可能为开发者提供实用的记忆解决方案。
