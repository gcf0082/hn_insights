## 文档概述

本文档为使用AI工具协助Linux内核开发提供了明确的指导方针。AI工具在参与内核开发时必须遵循标准的内核开发流程，包括参与开发流程文档、编码风格指南以及补丁提交规范等。

---

## 核心要点总结

### 1. 标准开发流程要求

AI辅助工具在进行Linux内核开发时，必须遵循既定的内核开发规范：

- **开发流程文档**: Documentation/process/development-process.rst
- **编码风格规范**: Documentation/process/coding-style.rst  
- **补丁提交指南**: Documentation/process/submitting-patches.rst

### 2. 许可协议与法律要求

所有贡献必须符合内核的许可协议要求：

- 所有代码必须与GPL-2.0-only兼容
- 使用适当的SPDX许可标识符
- 详细信息参阅Documentation/process/license-rules.rst

### 3. Signed-off-by与开发者来源证书（DCO）

这是一个关键的法律要求：

- **AI代理禁止添加Signed-off-by标签**
- 只有人类开发者才能合法认证开发者来源证书（DCO）
- 人类提交者负责：
  - 审核所有AI生成的代码
  - 确保符合许可协议要求
  - 添加自己的Signed-off-by标签以认证DCO
  - 对贡献承担全部责任

### 4. 属性标注规范

当AI工具参与内核开发时，正确的属性标注有助于追踪AI在开发流程中不断演变的角色。贡献应包含Assisted-by标签，格式如下：

```
Assisted-by: AGENT_NAME:MODEL_VERSION [TOOL1] [TOOL2]
```

其中：
- `AGENT_NAME`: AI工具或框架的名称
- `MODEL_VERSION`: 使用的具体模型版本
- `[TOOL1] [TOOL2]`: 可选的专项分析工具（如coccinelle、sparse、smatch、clang-tidy）

注意：基本的开发工具（git、gcc、make、编辑器）不应列出。

**示例**：

```
Assisted-by: Claude:claude-3-opus coccinelle sparse
```

## 重要意义

这份文档体现了Linux内核社区对AI辅助开发工具的务实态度：

1. **明确责任归属**: 人类开发者对AI生成的代码承担全部责任
2. **保持透明度**: 通过Assisted-by标签追踪AI参与情况
3. **合规性保障**: 确保所有贡献符合开源许可协议要求
4. **质量控制**: AI工具必须遵循内核编码规范，不能降低代码质量

这一政策的核心在于：AI是辅助工具而非决策者，最终的责任和控制权始终掌握在人类开发者手中。
