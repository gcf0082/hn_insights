## 核心议题

### Claude Code 性能下降引关注

近日，多位用户反映 Claude Code 在近期的更新后出现了明显的性能下降问题。主要表现为：

1. **任务完成度不足** - 大型编程任务中，约35%的请求被完全忽略或草率完成
2. **主动延迟任务** - Claude 会主动询问用户是否"留到明天再做"，称其为"大型重构"
3. **思维深度缩减** - 有用户指出模型运行时的思维深度约为原来的1/10

### 社区热议焦点

- **计算资源问题**：有用户猜测 Anthropic 为了节省计算资源，主动降低了模型的思考深度
- **订阅限制**：愤怒的用户指出这是"订阅计划的静默限制"，导致即使付费也无法获得应有的服务质量
- **可持续性问题**：有评论指出当前的用户令牌消耗率在 without VC funding 的情况下不可持续
- **行业趋势**：部分用户认为这是每个新模型发布的普遍现象，可以预测

### 用户行动

- 有用户在使用 Codex 进行质量检查时，发现了7处明显的未完成部分
- 部分用户已取消订阅，认为服务质量不值得每月$200的费用

## 洞察总结

此次讨论反映出 AI 编程助手用户对服务质量的期望与实际体验之间存在较大落差。社区普遍认为 Anthropic 为了控制成本而牺牲了用户体验，这也引发了对 AI 服务可持续商业模式的思考。
