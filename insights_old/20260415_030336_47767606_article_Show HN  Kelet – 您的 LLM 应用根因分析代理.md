# Kelet - AI Agent 自动化调试工具洞察报告

## 基本信息

- **产品名称**: Kelet
- **官方网站**: https://kelet.ai/
- **产品定位**: 自动化调试 AI Agent 和 LLM 应用的生产故障
- **核心功能**: 追踪生产故障、定位根因、提供修复方案

## 产品概述

Kelet 是一款专为 LLM 应用和 AI Agent 设计的自动化调试工具。它能够追踪生产环境中的故障，精确定位问题根因，并提供可验证的修复方案，让开发者无需再手动翻阅大量追踪数据。

## 核心功能

### 1. 自动化根因分析
- 读取生产环境中的所有追踪数据
- 自动聚类失败模式
- 提供带有证据的根因分析，而非猜测

### 2. 智能问题分类
- Dashboard 显示待处理问题、Agent 健康评分和 AI 生成摘要
- Issues 功能展示每个失败模式的分类、证据和可直接应用的提示词修复方案
- Sessions 检查器可查看任意追踪，精确判断哪个步骤出了问题

### 3. 修复验证
- 从根因分析到提示词修复的完整流程
- 在真实会话上运行修复方案
- 展示修复前后的可靠性对比

### 4. 持续学习机制
- 自动收集追踪和信号
- 调查根因
- 验证修复效果

## 支持的技术栈

Kelet 支持与以下主流框架和工具集成：
- OpenTelemetry、Langfuse、Mixpanel
- OpenAI、Anthropic
- LangChain、pydantic
- AI SDK、CrewAI、Strands、Agno、Mastra
- PostHog

## 关键数据指标

- **33,000+** 个会话已分析，发现故障
- **73%** 的团队存在未被注意的故障（Kelet 发现）
- **14.3 分钟** 中位数：从追踪数据摄入到提示词修复

## 工作原理

1. **数据收集**：连接到 Agent 栈，几分钟内完成集成，所有交互、信号和用户反馈自动流入
2. **根因定位**：Kelet 阅读每条追踪数据，分钟级定位根因，基于证据而非直觉
3. **修复交付**：通过真实会话验证修复效果，确保部署后正常运行

## 信号机制

信号是帮助 Kelet 定位问题的概率性提示，包括：
-  thumbs-down 评分
- 用户编辑 AI 输出
- 放弃的对话
- 配置的合成 LLM 判断检查

## 安全与隐私

- SOC 2 认证
- 数据按组织隔离，行级安全
- 永不将用户数据用于训练公共模型
- 自动为每个子 Agent 微调私有模型，仅服务该组织

## 适用场景

Kelet 适用于：
- Agentic 循环
- 多步骤工作流
- RAG 管道
- 聊天机器人
- 自主 Agent

不适用于：
- 使用他人构建的 AI 工具（Cursor、Claude Code 等）
- 在现有 Agent 平台上构建技能/插件

## 总结

Kelet 将传统工程师手动进行的根因调查过程自动化，以规模化的方式同时处理每个故障。它不仅是一个观测工具，更是能够诊断根因并生成针对性提示词修复的"AI 医生"。通过解决 AI Agent 调试的痛点，帮助团队将工程时间从盲目排查转移到真正的问题解决上。
