# Claude Code 源码泄露揭示的AI工程文化

**洞察链接**: https://news.ycombinator.com/item?id=47772282

**基本信息**:
- 标题: What Claude Code's Source Revealed About AI Engineering Culture
- 来源: Hacker News
- 发布时间: 14小时前
- 评论数: 22
- 得分: 37

## 核心洞察

2025年12月27日，Anthropic首席工程师Boris Cherny在X上发布声明："过去三十天，我100%的Claude Code贡献都是由Claude Code编写的。"259个pull requests，497次提交，40,000行新增代码，130万次浏览。科技界为之鼓掌。

三个月后，一次打包错误暴露了512,000行代码。泄露会发生，公司会恢复。泄露不是故事本身，代码才是。

## 关键发现

### 1. 惊人的代码结构

Anthropic的Claude Code包含64,464行核心TypeScript代码服务于付费客户：
- `print.ts`：单个函数长达3,167行，486个分支点，12层嵌套
- `QueryEngine.ts`：46,000行
- `Tool.ts`：29,000行
- `commands.ts`：25,000行
- `main.tsx`：785 KB

一个HN评论员指出，这个3,167行的函数应该被拆分为8-10个独立的模块。

### 2. 讽刺的Regex情感分析

在世界顶级LLM公司中，检测用户沮丧情绪使用的是正则表达式：
```regex
/\b(wtf|shit|fuck|horrible|awful|terrible)\b/i
```
这被评论为"就像货运公司用马匹运输零件"。虽然 defenders辩称regex比推理调用更快更便宜，但这反映了工程文化的核心：便宜优先于正确，快速优先于良好，直接发布。

### 3. 已知但未修复的Bug

泄露的源码中有一个注释成为象征："1,279个会话连续失败50+次（高达3,272次），每天浪费约250,000次API调用。"修复只需要3行代码。某个人知道这个问题，写了注释记录它，然后仍然发布了。

### 4. 自动化的沉默

问题追踪器本身被自动化到沉默：
- Claude Sonnet驱动的去重机器人处理每个新问题
- 扫地机器人在14天后将问题标记为陈旧，再过14天关闭
- 锁定机器人在7天后禁止对已关闭问题发表评论
- 分析估计49-71%的26,792个问题关闭是由机器人驱动的
- Issue #38335有201个赞成票，零团队回应，标签为"无效"

### 5. 八次服务中断与重复泄露

一个月内八次服务中断。源码泄露发生了两次（第一次在2025年初悄悄修补）。同一天，Axios依赖在供应链攻击中被入侵。74个npm依赖用于一个本质上是API包装器的CLI工具。

##  Ouroboros：自我吞噬的蛇

Anthropic的团队哲学是什么？代码审查AI生成的代码没有意义。更新规格并重新生成。

这是 ouroboros。蛇吞自己的尾巴。AI编写代码。AI审查代码。AI检查部署。当它坏了，答案是更多的AI。循环没有退出条件。

CEO Dario Amodei的预测从"90%的代码由AI编写"到"70-80-90%"的范围内移动，最终达到100%。但90%意味着什么？提交的行数？工程工作？键入的字符？Anthropic从未澄清。模糊是目的。

## 舒适区之外

工程文化没有开关。团队用12层嵌套 shipping `print.ts`，在编写模型训练代码时不会突然变得纪律严明。同样的过程，同样的代码审查，或者缺乏它们。

当Anthropic用数十亿收入和无限计算时，他们用资源而不是修复来喂养技术债务。功能是3,167行？不要重构，添加更多RAM。autoCompact bug烧掉250,000次API调用？利润率吸收它。模型回归？将更多GPU小时投入到训练中。

这在资金流动时有效。Anthropic是一家规模扩张快于建设工程实践的创业公司。AI-writes-AI-checks-AI-fixes的递归循环掩盖了基础缺失。但计算变得昂贵。收入周期轮转。技术债务变成债务陷阱，没有出口。

## 关于这个讨论

社区讨论揭示了不同的观点：

**支持"快速行动"观点**：这种"坏"软件在商业上如此成功，对于经验丰富的审查者如此令人厌恶，意味着快速是正确的工程选择，而不是"正确地做事情"强调代码质量。

**反对观点**：这种草率的vibe coding只有在没有竞争时才会有效。当竞争带来更有效的东西时，如pi-dev，低效的应用将被淘汰。Claude Code立即打开新市场，但因为它仍然是一团糟，现在有更好的替代品。

**用户体验观点**：Claude Code运行良好，在可用性层面没有问题。可以看到他们正在快速发货，在市场时间和代码质量之间取得了正确的平衡。

## 结论

出售AI编码工具的公司无法用其自己的AI编码工具构建质量产品。百分比始终是宣传，而不是产品。80、90、95、100。在有人问100%实际产生什么之前，源代码为他们回答。

AI放大了已经存在的任何东西。良好的纪律变成伟大的输出。没有纪律变成机器速度的技术债务。Anthropic选择了一个方向。去更快。让Claude检查Claude。当它坏了，仍然去更快。

如果这是从引领我们行业向前的公司看到的新质量标准，那么我不确定我想去哪里。

"做得好，或者根本不做。简单规则。质量不是功能，它是地板。那地板已经 gone。质量是遗物。现在没有人想要它。没有人支付它。没有人测量它。指标是速度。指标是生成的代码百分比。指标是你能多快发布一个每天烧掉250,000次API调用的3,167行函数并称其为100%AI编写。
