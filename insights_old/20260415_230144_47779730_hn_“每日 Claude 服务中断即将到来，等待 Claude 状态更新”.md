## 内容摘要

这是一篇关于 Claude Code（Anthropic 的 AI 编程助手）每日定期宕机的 Hacker News 讨论帖。用户报告在每日大约 14:30 UTC 时刻，Claude Code 开始出现持续的 API Error: 500 内部服务器错误，导致无法正常使用。

## 主要讨论点

### 1. 宕机问题的规律性

多位用户指出，这种宕机现象非常规律，几乎每天都在同一时间发生。有用户戏称："Once AGI is achieved, they'll reach the fabled superhuman 'two nines' of uptime."（一旦实现 AGI，他们将达到传说中的 "两个九" 的正常运行时间）

### 2. Status 页面与实际状态不符

用户讽刺地指出：
- Claude 返回错误："API Error: 500... check status.claude.com"
- 但 status.claude.com 显示："All Systems Operational"（所有系统正常运行）

这种状态页面的滞后更新让用户感到困惑和不满。

### 3. 用户对 Anthropic 的批评

一位用户详细列举了对 Anthropic 的不满：
- 支持服务很差，提交工单后一周没有回复
- claude.ai 账号和 Console 账号的分离令人困惑
- 旧组织删除后需要等待 7 天才能加入新组织
- API 额度充值功能曾故障约一周
- Claude 聊天质量下降，给出简单问题的答案很糟糕
- 桌面应用是伪装的网页应用
- MCP 集成调试困难，只有通用的 "发生错误" 提示

### 4. 替代方案讨论

用户纷纷提出替代方案：

- **OpenAI Codex**: 规划能力强，执行的代码质量略差，但能创建可用的产品。配额度比 Claude 慷慨得多。
- **Qwen 3.6 Plus**: 有用户推荐配合 opencode 使用
- **z.ai 的 GLM 模型**: 曾被推荐，但近期价格上涨超过一倍
- **OpenRouter**: 可以尝试多种模型
- **本地模型**: 有用户认为开源模型已经 "足够好"，每月支付 100-200 美元不再有意义
- **GitHub Copilot**: 价格便宜，提供几乎无限的自动补全和慷慨的月度配额

### 5. 服务可用性质疑

有用户指出过去 30 天 Claude Code 的正常运行时间只有 91%，对于一个声称 "Coding is solved"（编程已解决）的服务来说，这个可用性数字低得可怜。

### 6. 与早期 Twitter 的类比

有用户将 Claude 的频繁宕机与早期 Twitter 的 "fail whale" 相比，尽管故障频繁，但服务仍然是不可阻挡的。

---

## 结论

这次讨论反映了用户对 Claude Code 频繁宕机的强烈不满。尽管 Anthropic 声称 "编程已解决"，但其服务的可用性和稳定性仍存在严重问题。用户开始积极寻找替代方案，OpenAI Codex 成为最常被提及的选择。随着 AI 编程助手市场的竞争加剧，Anthropic 需要认真对待其服务质量问题，否则可能面临用户流失的风险。
