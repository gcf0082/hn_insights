## 项目概述

Libretto 是一个 Skill+CLI 工具，旨在让编码代理能够生成可确定性调试的浏览器自动化脚本。其核心思路是从"运行时提示代理"转变为"开发时生成可检查、可运行、可调试的实际代码"。

## 核心特性

1. **开发时 AI**: 在运行前生成实际代码，而非运行时黑盒行为。代码可检查、修改、版本控制和调试。

2. **混合方法**: 结合 Playwright UI 自动化与直接网络/API 请求，提高可靠性和反机器人检测能力。

3. **记录功能**: 记录手动用户操作，帮助代理生成和更新脚本。

4. **调试支持**: 支持单步调试。

5. **只读模式**: 可选只读模式，防止代理意外提交或修改数据。

6. **代码生成**: 生成遵循代码仓库现有抽象和约定的代码。

## 与现有工具的对比

与 Browseruse、Stagehand 等运行时 AI 工具相比，Libretto 的优势在于：

- 不依赖容易出错的 DOM 解析
- 减少 AI 调用次数，降低成本
- 代码可追溯，不透明
- 专门帮助生成和调试自动化失败

## 社区反馈要点

1. **HIPAA 合规性**: 有人在评论中提醒医疗场景需注意 PHI 暴露风险。开发者回应已与 Anthropic 和 OpenAI 签订 BAA。

2. **DOM 解析脆弱性**: 评论指出 DOM 解析依赖渲染 artifacts，任何样式重构、框架升级或 A/B 测试都可能破坏脚本。

3. **WebSocket 处理**: 对于复杂情况（如 WebSocket 或分块响应），Libretto 目前仅捕获 HTTP 请求，必要时回退到 Playwright DOM 操作。

4. **许可证**: MIT 许可证。

## 技术评价

Libretto 代表的"开发时 AI"思路值得关

注。网络请求优先的方法比 DOM 解析更接近测试契约本身，是一种更可靠的自动化方法。该工具特别适合需要高稳定性的医疗保健和保险门户等复杂场景。
