# 洞察报告：Firebase 浏览器密钥导致 €54,000 意外账单

## 基本信息

- **洞察链接**：https://discuss.ai.google.dev/t/unexpected-54k-billing-spike-in-13-hours-firebase-browser-key-without-api-restrictions-used-for-gemini-requests/140262
- **来源**：Google AI Developers Forum
- **发布日期**：2025年4月15日
- **标签**：gemini-api, gemini, billing

## 事件概述

用户在现有 Firebase 项目中启用了 Firebase AI Logic（生成网页片段的简单 AI 功能）后，遭遇了意外的巨额账单。在短短 13 小时内，Gemini API 产生了超过 €54,000 的费用。

## 详细情况

### 背景
- 项目创建于一年多前，最初仅用于 Firebase Authentication
- 最近添加了简单的 AI 功能（根据文本提示生成网页片段）
- 启用了 Firebase AI Logic

### 问题发生
- 启用后不久，Gemini API 使用量突然急剧飙升
- 流量与实际用户无关，明显是自动化攻击
- 发生在短暂的夜间窗口内
- 用户在禁用 API 并轮换凭证后，攻击停止

### 后续处理
- 用户设置了 €80 的预算警报和成本异常警报，但都延迟了几个小时才触发
- 做出反应时，成本已约 €28,000
- 由于成本报告延迟，最终结算金额达 €54,000+
- Google Cloud 支持将此费用归类为有效使用，因为请求来自用户的项目
- 账单调整请求最终被拒绝

## 关键教训

1. **API 密钥安全**：Google API 密钥不再是安全凭证，需要像保护密码一样保护它们
2. **警报延迟**：预算和成本异常警报可能存在显著延迟，无法及时阻止损失
3. **密钥类型风险**：Firebase 浏览器密钥（无 API 限制）容易被滥用

## 建议的防护措施

- 使用 App Check
- 设置配额限制
- 将调用移至服务器端
- 严格限制 API 密钥的使用范围
