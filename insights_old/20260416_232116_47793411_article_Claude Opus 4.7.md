## 一、核心升级要点

Claude Opus 4.7 是对 Opus 4.6 的重大改进，尤其在高级软件工程领域表现突出。该模型能够处理复杂的长时任务，严格遵循指令，并能够在报告结果前验证自己的输出。

### 1.1 编程能力显著提升

- 在93项编程基准测试中，Opus 4.7 比 Opus 4.6 提升了13%的分辨率
- 其中4个任务Neither Opus 4.6 nor Sonnet 4.6 都能解决
- 中位延迟更快，指令遵循更严格
- 特别适合复杂的多步骤编程工作流

### 1.2 视觉能力大幅增强

- 支持更高分辨率的图像输入，最长边达2,576像素（约375万像素）
- 是之前Claude模型的三倍以上
- 适用于计算机使用代理读取密集截图、复杂图表数据提取等场景

### 1.3 创造力与设计品味提升

- 在完成专业任务时更具品味和创造力
- 能够生成更高质量的界面、幻灯片和文档
- 被评价为"构建仪表板和数据丰富界面的最佳模型"

## 四、安全与对齐

- 整体安全状况与 Opus 4.6 相似
- 在诚实度和抵抗恶意提示注入攻击方面有改进

- 自动检测和阻止高风险网络安全使用请求

## 五、新功能发布

### 5.1 更高努力控制（Effort Control）

- 新增 `xhigh`（超高）努力级别
- 位于 `high` 和 `max` 之间
- Claude Code 默认努力级别已提升至 `xhigh`
- 建议编码和代理用例从 `high` 或 `xhigh` 开始

### 5.2 Claude Code 新功能

- `/ultrareview` 斜杠命令：生成专门的代码审查会话
- Auto mode 扩展至 Max 用户

### 5.3 API 平台

- 支持更高分辨率图像
- 任务预算（Task Budgets）公测版

## 六、迁移注意事项

### 6.1 Tokenizer 更新

- Opus 4.7 使用更新的tokenizer
- 相同输入可能映射到更多tokens（约1.0-1.35倍）

### 6.2 思考token增加

- 在更高努力级别下会进行更多思考
- 特别是在代理设置的后续轮次中

### 6.3 用户建议

- 可以通过调整努力参数、任务预算或提示简洁度来控制token使用
- 建议在真实流量上测量差异
