---

## 概述

Anthropic 发布了 Claude Opus 4.7 版本，该版本采用了更新的分词器（tokenizer），导致相同输入映射到的 token 数量增加约 1.0-1.35 倍，具体取决于内容类型。这一变化引发了社区的热烈讨论。

---

## 主要讨论主题

### 1. Token 数量增加问题

用户对新版分词器导致的 token 增加表示担忧。官方建议用户可以通过以下方式控制 token 使用：
- 使用 effort 参数调整任务预算
- 提示模型更加简洁

### 2. Caveman 风格提示的热议

有用户提到使用 "caveman" 风格的提示可以压缩 token 使用，但遭到其他用户的反驳：
- 这种工具大多是玩笑/恶作剧项目
- 实际节省的 token 不到 1%
- 可能导致模型推理更多 token

### 3. Claude Code 与 OpenAI Codex 的对比

许多用户表示近期 Claude 4.6 表现不佳，转向使用 OpenAI Codex：
- Codex 更加稳定，输出更一致
- Claude 有时会出现质量下降和"失灵"
- OpenAI 近期将 $100 计划的限制提升至 2 倍以争夺 Anthropic 的客户

### 4. Anthropic 计算资源问题

社区普遍认为 Anthropic 面临计算资源短缺：
- 90% 的问题与计算资源不足有关
- 相比之下，OpenAI 早期的计算资源投资策略现在成为优势
- Anthropic 采取了保守的投资策略，导致难以满足需求

### 5. Mythos 模型发布日期

关于 Mythos 模型：
- 目前仅向少数用户开放测试
- 官方表示将根据安全测试结果决定何时公测
- 社区猜测可能是因为成本过高或计算资源不足

### 6. 用户信任问题

用户对 Anthropic 的透明度表示不满：
- 员工声称没有性能下降，但用户实际体验相反
- 希望官方能诚实说明"计算资源繁忙，建议使用其他模型"
- 部分用户因质量问题取消订阅并转向 Codex

---

## 关键观点

1. **用户对 Claude 4.7 持谨慎态度**：许多人担心新版可能与之前经历的性能下降版本相同
2. **市场竞争激烈**：OpenAI 通过提升限制和推出新计划正在蚕食 Anthropic 的市场份额
3. **透明度缺失**：用户对 Anthropic 的沉默和回避态度感到失望
4. **稳定性是关键**：用户更看重工具的一致性和可靠性，而非偶尔的惊艳表现
