## 项目概述

AutoProber 是硬件黑客的自动化飞针测试堆栈，旨在让 AI 代理能够安全地从"新目标出现在测试板上"一路完成各个引脚的探测。整个系统包含 Python 控制代码、Web 仪表板、CAD 文件和完整文档，可自行构建使用。

## 核心功能

1. **代理驱动的目标发现** - AI 代理可以自主完成从目标识别到引脚探测的全流程
2. **显微镜映射** - 拍摄并拼接微观图像，自动标注 pads、pins、chips 等特征
3. **安全监控的 CNC 运动** - 使用独立示波器通道监控安全停止条件
4. **探针审查** - 在 Web 仪表板上人工审核批准的探测目标
5. **受控引脚探测** - 仅在获得批准后执行有边界的探测运动

## 技术架构

### 硬件栈

- **CNC 控制器**: GRBL 兼容的 3018 型 CNC，通过 USB 串口控制
- **显微镜**: USB 显微镜，由 mjpg_streamer 提供视频流
- **示波器**: Siglent SDS1104X-E，通过 LAN/SCPI 通信，用于安全监控 (Ch4) 和测量 (Ch1)
- **光学位 stop**: 外部 5V 供电，连接到示波器 Ch4
- **可选**: 网络控制电源插座

### 软件架构

```
apps/              操作脚本和 Flask 仪表板入口
autoprober/        可复用的 Python 包 (CNC、示波器、显微镜、日志、安全)
dashboard/         单页 Web 仪表板
docs/              架构、设备参考、操作和安全指南
cad/               可打印的 STL 文件
config/            示例配置文件
```

## 工作流程

1. 导入项目并连接所有硬件
2. 确认所有部件正常工作
3. 执行归零和校准
4. 安装定制探针和显微镜头
5. 放置新目标，系统自动定位并拍摄帧
6. 拼接图像并标注特征
7. 在仪表板审核探测目标
8. 执行批准的探测并报告结果

## 安全模型

**重要**: 这是一个机器控制系统，非普通 Web 应用。

- **忽略 GRBL 的 Pn:P** - CNC 探针引脚不是可信的终点限位
- **独立安全停止** - 来自示波器 Ch4 的光学位 stop 信号
- **持续监控** - 任何运动期间必须持续监控 Ch4
- **停止条件**: Ch4 触发、电压模糊、CNC 报警、真实 XYZ 限位
- **无自动恢复** - 停止后需人工介入，排除故障

详见 docs/safety.md 和 docs/operations.md

## 当前限制

- 显微镜到探针的 XY 偏移必须事先测量
- 校准必须在实际运行的机器上生成，不能伪造
- 仪表板是实验室控制工具，不应暴露到不可信网络

## 负责任使用
