## 核心发现

Anthropic 的 Claude Opus 4.7 迁移文档声称新分词器使用 "大约 1.0 到 1.35 倍" 的 token 数量。然而，作者实际测量发现：

- 技术文档：**1.47 倍**
- 真实的 CLAUDE.md 文件：**1.45 倍**
- 真实 Claude Code 内容加权平均：**1.325 倍**

**结论**：大多数 Claude Code 内容的实际 token 增长接近 Anthropic 文档范围的上限，而非中间值。

## 分词器变化原因

Anthropic 声称新分词器带来"更精确的指令遵循"。更小的 token 强制注意力集中在单个词汇层面，这是已知的更严格指令遵循、字符级任务和工具调用精度的机制。

### IFEval 基准测试结果

作者使用 IFEval 基准测试验证指令遵循改进：

| 指标 | 4.6 | 4.7 | 差异 |
|-----|-----|-----|------|
| 严格模式（提示级） | 17/20 (85%) | 18/20 (90%) | **+5pp** |
| 严格模式（指令级） | 25/29 (86%) | 26/29 (90%) | +4pp |
| 宽松模式（提示级） | 18/20 (90%) | 18/20 (90%) | 0 |
| 宽松模式（指令级） | 26/29 (90%) | 26/29 (90%) | 0 |

**结论**：4.7 在严格指令遵循方面有小幅但方向一致的提升。

## 实际会话成本计算

假设一个 80 轮的 Claude Code 调试/重构会话：

### 4.6 会话成本

| 项目 | 计算 | 成本 |
|------|------|------|
| 首次缓存写入 | 8K × $6.25/MTok | $0.05 |
| 缓存读取 (79轮) | 79 × 86K × $0.50/MTok | $3.40 |
| 新鲜用户输入 | 79 × 500 × $5/MTok | $0.20 |
| 输出 | 80 × 1,500 × $25/MTok | $3.00 |
| **总计** | | **~$6.65** |

### 4.7 会话成本

| 项目 | 计算 | 成本 |
|------|------|------|
| 首次缓存写入 | 10K × $6.25/MTok | $0.06 |
| 缓存读取 (79轮) | 79 × 115K × $0.50/MTok | $4.54 |
| 新鲜用户输入 | 79 × 660 × $5/MTok | $0.26 |
| 输出 | 80 × 1,500-1,950 × $25/MTok | $3.00-$3.90 |
| **总计** | | **~$7.86-$8.76** |

### 成本差异

**从 ~$6.65 增至 ~$7.86-$8.76，每轮会话增加约 20-30%**

---

## 对提示缓存的影响

1. **首次会话冷启动更贵**：切换到 4.7 会使所有缓存失效，4.7 的前缀比 4.6 大 1.3-1.45 倍

2. **缓存容量按 token 比率增长**：CLAUDE.md 部分需要 1.445 倍的 token 支付缓存写入和读取

3. **相同内容不同计数**：如果在 4.6 和 4.7 之间切换，日志将显示不同的 token 数量

## 总结

| 指标 | 数值 |
|------|------|
| Token 增加比例 | 1.3-1.45 倍 |
| 指令遵循提升 | +5pp (严格模式) |
| 每会话成本增加 | 约 20-30% |
| CJK 内容影响 | 几乎无影响 (1.01x) |

**是否值得？** 这取决于您发送的内容类型。对于英文和代码-heavy 的工作，您每轮会话多付 20-30% 的费用，获得小幅但真实的指令遵循改进。
