## 项目概述

ctx 是一个本地上下文管理器，专为 Claude Code 和 Codex 设计。它可以帮助用户精确绑定对话历史、在不同工作流之间安全分支、管理上下文记忆，而无需 API 密钥或托管服务。

## 核心功能

### 1. 精确的对话绑定
每个内部 ctx 会话都可以绑定到来源的 Claude 和/或 Codex 对话，确保后续提取始终在该绑定对话上操作，而不会跳转到磁盘上最新的聊天。

### 2. 安全的工作流分支
可以从另一个工作流的当前状态创建新工作流，而不会共享未来的对话记录或劫持源对话。

### 3. 索引检索
保存的工作流、会话和条目都经过索引，可用于快速 `ctx search` 查找。

### 4. 精选加载
可以固定条目使其始终加载，排除条目使其保持可搜索但停止传递给模型，或完全删除。

### 5. 本地优先
无需 API 密钥、无托管服务，使用纯 SQLite 和本地文件。

## 常用命令

### Claude Code
- `/ctx` - 显示当前工作流
- `/ctx list` - 列出保存的工作流
- `/ctx search <关键词>` - 搜索上下文
- `/ctx start <名称> --pull` - 创建并拉取上下文
- `/ctx resume <名称>` - 继续现有工作流
- `/ctx branch <源> <目标>` - 创建分支

### Codex
- `ctx list --this-repo` - 仅列出当前仓库的工作流
- `ctx web --open` - 打开本地浏览器界面
- `ctx curate <工作流>` - 管理保存的内存

## FAQ
