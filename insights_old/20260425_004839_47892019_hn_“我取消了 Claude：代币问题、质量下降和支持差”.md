## 帖子摘要

这是一篇关于用户取消 Claude 订阅的批评帖子，主要反映了对 Anthropic 公司产品的三个核心不满：**Token 用量问题**、**质量下降**和**客户服务不佳**。

---

## 核心观点分析

### 1. Token 用量激增问题

用户反映 Claude Code 在使用过程中出现了严重的 Token 消耗异常：
- **案例1**：用户使用 Sonnet 中等难度模式，完整消耗了会话限制，并额外支付了费用，思考了53分钟后提示 API 错误（输出超过32000 token上限）
- **案例2**：Claude Code 的 Agent 模式比直接使用 LLM API 效率低很多——相同任务，Agent 模式需要10分钟，而直接调用 llama_cpp 不到1分钟完成
- **用量不透明**：用户报告 Claude Code Max x20 计划的使用量波动极大，有时3小时高强度使用只消耗70%，有时20分钟就耗尽

### 2. 模型质量下降

用户普遍感受到 Claude 系列模型出现了明显的质量退化：
- **自适应思考（Adaptive Thinking）变更**：从 CC adaptive thinking 改为 4.7 版本后质量明显下降
- **默认推理级别上调**：Opus 4.7 默认使用 xhigh 推理，导致 Token 消耗更快
- **路由到低端模型**：有用户怀疑系统会将请求路由到更便宜/更弱的模型（伪装成 Opus 4.7）
- **用户对策**：有经验的用户会先问一个困难问题测试模型质量，如果答案不满意就重新开始会话

### 3. 定价与服务问题

- **价格高昂**：Cursor 订阅曾达到 $1,500/月，Claude Code Max 5x 约 $240/月，Max 20x 约 $2400/月
- **峰值时段限制**：在高峰时段模型响应变慢，有用户讽刺"我们应该在工作时间之外使用这个工作工具"
- **客户沟通不足**：Anthropic 在未提前通知的情况下更改了用户默认设置，引发大量不满
- **依赖风险**：开源社区担忧商业 AI 产品的不透明性和订阅模式带来的"上瘾效应"

### 4. 替代方案讨论

社区热烈讨论了多种替代方案：
- **本地/开源模型**：Kimi2.6、Qwen、GLM、MiniMax 等开源模型受到关注
- **开源 Agent 框架**：pi.dev、swival 等本地 AI Agent 框架被推荐
- **性能监控工具**：用户开发了 repogauge（https://repogauge.org）来追踪不同 AI 模型的表现
- **OpenAI Codex**：部分用户考虑迁移到 Codex

### 5. 正面反馈

尽管批评声音众多，仍有部分用户表示满意：
- 日常使用中未遇到明显问题
- 在科学应用和部署环境中仍能获得很好的价值
- 关键在于为模型提供清晰约束并持续监控其思考过程
