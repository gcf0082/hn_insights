## 核心观点

### 1. MCP协议开启个人自动化新时代

作者Matt May分享了如何利用Claude Code的routines功能实现财务自动化管理。传统的自动化方案（如Chrome DevTools MCP）存在诸多问题：浏览器渲染异常、意外的2FA验证、GPT格式变化导致脚本失效等。而Claude Code routines提供了一种全新的解决思路——无需编写agent代码，无需部署基础设施，只需声明式配置即可实现复杂的自动化任务。

### 2. 从爬虫爬取到API直连的演进

作者最初尝试使用Chrome DevTools MCP通过cron任��自动登录银行、信用卡和券商账户，获取余额和交易记录后发送每日邮件。但这种方式过于脆弱，每次网站UI变更或验证流程调整都会导致任务失败。最终作者创建了Driggsby项目——一个基于Plaid的MCP服务器，通过正规API连接金融机构，稳定性大幅提升。

### 3. 非技术人员也能使用AI自动化

文章特别强调了一个重要观点：作者的妻子也能创建自己的自动化任务。这意味着AI自动化工具的门槛已经降低到普通用户可及的程度。Claude Code routines的声明式特性使得复杂的工作流可以被普通用户理解和修改。

## 启示与思考

### 对个人用户

- **隐私优先**：使用Plaid等正规API而非爬虫，数据安全性更高
- **主动监控**：自动化让"忘记查账"成为历史
- **家庭协作**：非技术背景的家庭成员也能参与自动化配置

### 对开发者

- **API优先**：选择稳定API而非依赖网页解析，降低维护成本
- **声明式设计**：routines模式简化了复杂工作流的定义
- **MCP生态**：围绕AI工具构建的中间件生态正在快速成熟

### 对AI行业

- **自动化民主化**：从需要编程能力到自然语言描述，门槛持续降低
- **垂直领域深耕**：金融、医疗等领域的专业化AI工具需求旺盛
- **人机协作模式**：AI不是替代人类，而是增强人类的能力和效率

## 总结

Matt May的实践展示了AI驱动自动化的一个重要方向：让技术为普通人的日常生活服务。通过Claude Code routines和MCP协议的结合，复杂的金融监控任务变得触手可及。这种"零代码"的自动化范式预示着未来每个人都可能拥有自己的AI助手来管理财务、健康、家庭事务等方方面面。Driggsby项目的成功也提醒我们，在AI时代，选择稳定可靠的API接口比依赖脆弱的网页爬取更加重要。
