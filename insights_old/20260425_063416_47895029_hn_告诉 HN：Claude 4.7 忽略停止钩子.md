## 一、问题概述

Claude Code 4.7 版本存在严重的 Stop Hooks 功能失效问题。用户配置的 Hook 规则被系统性忽略，导致工作流自动化和代码质量控制机制失效。

### 1.1 核心问题

用户配置的 Stop Hook 正确返回了 `{"decision": "block", "reason": "..."}`，但 Claude Code 完全忽视该阻止信号，直接结束会话。这使得用户无法在会话结束前强制执行必要的检查或文档更新流程。

### 1.2 问题影响范围

- 代码质量门禁失效
- 文档更新强制要求无法生效
- 测试运行检查被绕过
- 依赖自定义 Hook 的工作流系统崩溃

## 二、社区反馈与问题表现

### 2.1 主要症状

| 问题类型 | 描述 | 影响程度 |
|---------|------|---------|
| Block 决策被忽略 | Hook 返回 block 但会话仍结束 | 严重 |
| 触发时机混乱 | 每轮响应后触发，而非会话终止 | 高 |
| 间歇性失效 | 某些响应触发 Hook，其他则不 | 中 |
| 静默停止检测缺失 | 工具执行后无输出时 Hook 不触发 | 高 |

### 2.2 用户调试证据

```
Decision: BLOCK (planning work detected, AGENT_NOTES not updated)
Full response:
{
  "ok": true,
  "decision": "block",
  "reason": "Planning documentation required..."
}

执行日志显示:
✅ Hook 执行成功
✅ Hook 检测到问题并返回 block
❌ Claude 忽略 block，会话仍然结束
❌ 用户从未看到阻止消息
```

### 2.3 语义混淆问题

**Stop Hook 实际行为**：在每个 Assistant 响应完成后触发（Turn-Level）

**用户预期行为**：仅在会话结束时触发（Session-Level）

这导致交互式工作流中，每次响应都会弹出门禁对话框，严重破坏用户体验。

## 三、技术分析与根因

### 3.1 Block 决策处理缺陷

Claude Code 在解析 Hook 返回的 JSON 后，未正确处理 `decision: "block"` 分支。代码可能存在：

- 决策字段解析后未传递到会话管理逻辑
- 默认行为优先于 Hook 返回值
- 并发情况下竞态条件导致决策被覆盖

### 3.2 触发时机问题

1. **Turn-Level 触发**：每次 Assistant 消息发送后即触发
2. **Session-Level 缺失**：真正的会话终止事件无对应 Hook

这表明 Hook 系统设计存在概念混淆：将「响应完成」与「会话结束」视为同一事件。

## 四、潜在解决方案

### 4.1 短期 workaround

**A. 使用 SessionEnd 而非 Stop Hook**

```json
{
  "hooks": {
    "SessionEnd": [{"hooks": [{"type": "command", "command": "..."}]}]
  }
}
```

**B. 心跳检测方案（检测静默停止）**

```bash
# 定时检查 Claude 是否停滞
TIMEOUT=120
while true; do
  if [ -f "$HOME/.claude/.heartbeat" ]; then
    LAST=$(cat "$HOME/.claude/.heartbeat")
    NOW=$(date +%s)
    DIFF=$((NOW - LAST))
    if [ "$DIFF" -gt "$TIMEOUT" ]; then
      echo "Claude stalled for ${DIFF}s"
    fi
  fi
  sleep 15
done
```

### 4.2 官方修复建议

| 优先级 | 方案 |
|--------|------|
| P0 | 修复 `decision: "block"` 被忽略的 Bug |
| P1 | 明确区分 Turn-Level 与 Session-Level 事件 |
| P1 | 添加 Hook 触发原因字段，区分不同场景 |
| P2 | 修复间歇性触发问题（文本响应/包含换行符时） |

### 4.3 文档改进

- 明确文档化 Stop 语义（在每轮响应后触发）
- 推荐 SessionEnd 用于会话级别逻辑
- 提供 Hook 触发时机对照表
